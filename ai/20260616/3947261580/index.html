AI 是不是偷偷變笨了？AIStupidLevel：24 小時監測 AI 模型退化的評測網站

#AI #ChatGPT #AIStupidLevel #AI Benchmark #LLM #AI 模型評測 #Claude #Gemini

前言

不知道大家有沒有遇過這種狀況，也就是昨天使用 AI 還感覺它很聰明，不用多講什麼就會順便把一些 Bug 給解掉，但隔天卻同樣的問題它打死都不回去主動順便把一些明顯的 Bug 給解掉，反而都要等你發現他才會處理，甚至它卻開始鬼打牆、答非所問，讓你開始懷疑「AI 是不是偷偷變笨了？」。

我相信這種感覺應該滿多人都有，但就是很難講出所以一個然。到底是 AI 廠商搞鬼，還是只是你今天的問題剛好比較難、或單純是心理作用？沒有一個客觀的數字，這就只是一個都市傳說。

所以這一篇就要來介紹一個網站，它就叫做AIStupidLevel，光看名字（直翻大概是「AI 智障程度」）就知道它走的是很直白的路線。它做的事情就是 24 小時不間斷地測試各家 AI 模型，幫你把「AI 是不是變笨了」這件事變成一條可以攤開來看的曲線。

所以我才寫一下這一篇來介紹啦～

所以 AIStupidLevel 到底是什麼？

簡單來講，AIStupidLevel 是一個「持續監測 AI 模型表現有沒有退步」的第三方評測平台，就跟我們系統在做健康檢查一樣，而它則是專門盯著 AI 模型的「健康狀態」，幫你抓出哪個模型最近偷偷變笨了。全年無休 24h 幫你監控 OpenAI、Anthropic、Google、xAI、DeepSeek、Kimi、GLM 共 7 家供應商旗下 21 個現役 AI 模型的表現，然後把每個模型當下的表現換算成一個分數攤在儀表板上。

這時候有些人可能會好奇，這個專案是不是某個 AI 監控公司做的，其實不是，這是一個開源專案。所以立場會比較偏向「第三方監督」角色，比較不容易跟廠商牽扯上關係，這點我覺得滿重要的，畢竟比較不會發生球員兼裁判的問題哩～

那麼了解了基本 AIStupidLevel 是什麼之後，讓我們繼續往下看吧。

恩？AI 為什麼會「變笨」？

我想這應該是滿多人會有的疑問，也就是…

模型出廠不是就訓練好了嗎？固定了嗎？

這個問題的答案其實原因有以下：

  1. 無聲更新（silent update）： 你如果有接過 API 的話，你可能會看到這種版本號 （例如 gpt-4.0-20240924），其實這個版本號裡面那串日期（20240924）但實際上你在使用 Codex 選模型時，你並不會看到這麼細的版本號，甚至有些模型根本就沒有版本號，只有一個 model ID（例如 gpt-4），所以你根本不知道自己實際使用的是哪一版模型，被偷偷更新也很難察覺。
  2. 量化（quantization）：許多時候，為了處理來自全世界的尖峰流量、算力，模型商可能會把模型壓縮過（量化）。
  3. 算力限流：當使用人數超過一定之後的時候，模型商會把你能夠使用的算力限制掉，這樣就會讓模型的表現變差。
  4. 算力移轉：有時候，當一個新模型即將推出的時候，模型商可能會把算力從舊模型轉移到新模型上，這樣就會讓舊模型的表現變差。以我自己長期接 API 來講，就很常遇到這種狀況，相同 Prompt 突然某一天彷彿降級到變成智障，接著沒多久模型商就推出更強的版本，接著舊版本才恢復正常。

其中「量化」這件事我之前在 想自己跑 LLM？搞懂模型參數與量化讓你不再選錯模型 那篇有比較完整聊過，簡單來講就是把模型壓縮的技術，壓縮過後雖然省資源，可是細節會少一點。

所以來畫個重點吧：

AI 模型不是一個固定不變的東西，廠商可能會在背後偷偷更新、偷偷量化、偷偷限流、偷偷轉移算力，這些都會讓模型的表現變差，但你卻很難察覺，所以才會有「AI 是不是偷偷變笨了」這種感覺。

它是怎麼測的？

講了那麼多，但似乎沒聊到 AIStupidLevel 到底怎麼測試的，其實核心測試方式很簡單，分別是：

  1. 固定題目反覆考：它準備了一套固定的題庫，定時定期丟給各個模型去考，然後把成績記錄下來。
  2. 同一題問 N 次：因為 AI 模型的輸出有隨機性，所以它同一題會跑 N 次，取中位數來當分數，並且算出 95% 的信心區間。

看起來很簡單吧？但魔鬼就藏在細節裡。

同一題問 N 次，因為 AI 每次答案都不一樣

「AI 每次產出都有隨機性不是嗎？」

沒錯，就是利用這個特性來設計測試的，如果只問一次就給分數，那絕對可信度是非長堤的，因此同一個題目至少會跑 N 次（但 N 是多少呢？AIStupidLevel 的做法是 5 次），然後取中位數來當分數，這樣就能比較穩定地反映出模型的「平均表現」，而不是被偶爾的好運或壞運給誤導。

四種測試輪流跑

AIStupidLevel 可不是只考一種題目，它可是準備了好幾套不同的測試輪流上場，每一套盯的東西不太一樣。整理成表格會比較清楚：

測試套組 多久跑一次 在測什麼
速度測試（程式碼解題） 每 4 小時 147 道程式題，看模型解程式題的綜合表現
深度推理 每天一次 5～7 turn 的多輪對話，看長對話下邏輯會不會爆掉
工具呼叫 每天一次 開真實的 Docker 沙箱讓 AI 跑 execute-command、read-file、write-file 之類的多步驟流程
快速漂移偵測 每小時 12 題輕量快篩，第一時間抓出有沒有異常

從這張表可以看出來，它的設計是「高頻快篩 + 低頻深測」搭配著用。每小時的快篩負責當哨兵，一發現苗頭不對就拉警報；每天那套比較重的深度推理跟工具呼叫，則是拿來做比較紮實的全面體檢。

其中工具呼叫那套我覺得特別有意思，它是真的開一個 Docker 沙箱（你可以想像成一個跟外界隔離的乾淨小電腦）讓 AI 在裡面實際跑指令，而不是只看它「嘴上說會不會」。這樣測出來的結果會比較貼近你實際拿 AI 來寫程式時的體感。

評分不是只看「對不對」

那一題到底怎麼打分數？這邊也是它做得比較細的地方。它不是單純看答案對不對，而是從好幾個面向一起評。

以程式碼解題來講，AIStupidLevel 目前是用 9 個面向加權算出來的，分別是正確性（40%）、複雜度（20%）、程式碼品質（15%）、穩定性（10%）、效率（5%）、邊界狀況處理（3%）、除錯能力（3%）、輸出格式（2%）、安全性（2%）。可以看到正確性佔了最大宗，但即使答案能跑，寫得又亂、邊界沒處理好、格式亂噴，分數還是會被扣。

怎麼抓出「變笨的那一刻」

光是有分數還不夠，因為分數本來就會上上下下的跳動（前面講過 AI 有隨機性）。真正的難題是：怎麼判斷這次掉分是「真的退化」，還是「正常的隨機跳動」？

這就是它最核心的技術了。它用了一個叫做 CUSUM 的演算法來偵測這件事。這個東西其實不是 AI 領域發明的，它本來是工廠在做品質管制時用的老方法，概念上就是持續累積「實際表現」跟「基準水準」之間的落差，當落差累積到一個程度，就判定「這不是抖動了，是真的出事了」。

除了 CUSUM 之外，它還會搭配統計檢定（去算這個差異在統計上是不是顯著、p 值有沒有低於 0.05）來再確認一次，盡量壓低「明明沒事卻亂報」的誤判率。

這邊真正的關鍵是：靠這套統計方法，它可以在模型開始退化的幾個小時內就抓到，而不是等到大家都在社群上哀嚎「AI 變笨了」才後知後覺。整個系統內建了 29 種不同的警告類別，對應各種不同的異常狀況。

那分數到底怎麼看？（Stupid Meter）

其實你打開網站就可以馬上看到儀表板了

每個模型旁邊都會有一個 0～100 的即時分數，分數越高代表當下表現越好、越「聰明」，你可以把它想像成 AI 模型的即時戰力錶。然後分數旁邊會掛上一個狀態標籤，告訴你它現在是 STABLE（穩定）、VOLATILE（在抖）、DEGRADED（已經降級）、還是 RECOVERING（正在回血）這四種狀態的哪一種。

除了當下的分數，它還會把每個模型的歷史曲線攤開，讓你比較「上週很穩的模型，這幾天是不是開始亂跳」這種情境，再搭配前面那 29 種警告類別，可以很快判斷現在這個時間點，哪幾個模型狀態正常、哪幾個最近怪怪的。

所以實際用起來，它比較像是看盤軟體，你不是來看一個死的排名，而是來看「此時此刻哪個模型最值得用」。如果你正在煩惱要選哪個 AI 工具，我之前寫的 2026 年 AI Coding 工具怎麼選？ 那篇可以搭配著看哩～

順便講一下 Smart Router

除了監測之外，AIStupidLevel 還附帶了一個叫做 Smart Router 的功能，這個我覺得是這個專案蠻有意思的延伸。

簡單來講，它就是一個 OpenAI 相容的 API endpoint（也就是你原本打 OpenAI 的程式碼幾乎不用改），你只要把各家 provider 的 API Key 存進去（會用 AES-256 加密），再把 base URL 換成它的網址，它就會根據當下監測到的模型狀態，自動幫你把 request 丟給最適合的模型。

它目前提供六種路由策略，你只要在打 API 的時候把 model 欄位指定成下面任何一個，它就會幫你動態挑模型：

  • auto-best：通盤考慮，挑當下綜合分數最高的
  • auto-coding：專挑當下程式題表現最好的模型
  • auto-reasoning：專挑推理任務最強的
  • auto-creative：偏向創意輸出
  • auto-cheapest：在合格門檻內挑最便宜的
  • auto-fastest：在合格門檻內挑回應最快的

如果你只填 auto，它會用你預設的策略；如果你填具體的模型名（例如 claude-opus-4-7），它就會直接打那個模型不走路由。換句話說，當某個模型偷偷變笨被它抓到時，Smart Router 會自動避開那個模型，把流量導去當下狀態還正常的兄弟模型，這對於要把 AI 真的接進產品的人來說滿實用的。

總結

聊那麼多，最後總結一下吧。

所以 AIStupidLevel 到底是什麼？

  • 它是一個獨立、開源（MIT License）的第三方評測網站，24 小時監測 OpenAI、Anthropic、Google、xAI、DeepSeek、Kimi、GLM 共 7 家供應商旗下 21 個現役 AI 模型
  • 測法是固定題目反覆考，同一題跑 5 次取中位數 + 95% 信心區間，分高頻快篩跟低頻深測四種套組輪流跑
  • 它用 CUSUM 變點偵測搭配統計檢定，能在幾小時內抓出某個模型是不是偷偷退化了
  • 另外還附帶 Smart Router，可以根據監測結果自動把 API 流量導去當下狀態最好的模型

下次你再覺得 AI 突然變笨的時候，先別急著懷疑是不是自己的問題，可以打開這個網站看一眼，搞不好真的能找到證據哩～
