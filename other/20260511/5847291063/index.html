RSS 是什麼？為什麼 2026 年了還要用 RSS？順便聊聊我自己寫的 Chrome 擴充套件 Raydar

#RSS #Atom #RSS Reader #Raydar #Chrome 擴充套件 #訂閱 #資訊管理 #OPML

前言

這篇從 RSS 是什麼、跟訂閱電子報差在哪裡、為什麼 2026 年了還值得用 RSS、跟演算法推送的差異、Atom 是什麼，一路聊到我自己寫了一個 Chrome RSS 閱讀器擴充套件 Raydar，包含 RSS 自動偵測、一鍵訂閱、三種閱讀模式、本地優先儲存、無帳戶、OPML 匯入匯出、JSON 備份、鍵盤快捷鍵等功能，最後也會說明怎麼裝跟為什麼會想自己刻一個。

事情是這樣的

這幾天我把自己寫的 Chrome 擴充套件收尾收得差不多了，是一個 RSS 閱讀器，名字叫 Raydar。

寫完之後跟朋友隨口聊了一下，結果有個朋友反問我「RSS 是什麼？」，我才意識到一件事，RSS 對工程師圈來講雖然不算冷門，可是對一般人來講，這個詞真的已經很久沒出現在主流的視野了。

再加上 2013 年 Google Reader 收掉之後，網路上有滿多文章寫說「RSS 已死」，導致很多新手會直接跳過這個東西，覺得它是「上個時代的技術」。

可是說真的，在現在這個演算法越來越霸道、IG / X 各種重新排序、新聞網站 paywall 越開越多的時代，RSS 反而是少數還能讓你「自己決定要看什麼」的方法之一，所以這一篇就想說來介紹一下 RSS 是什麼、為什麼還值得用，然後最後也順便聊聊我寫的這個小工具哩～

RSS 到底是什麼？

簡單來講，RSS 是一份網站對外公開的「最新內容清單」，格式是 XML，內容包含每篇文章的標題、連結、發佈時間、摘要等等。

什麼意思呢？舉個實際的例子比較好懂。

假設你今天打開我的部落格首頁，你看到的是一個「為了人類眼睛而排版」的頁面，有導覽列、有側邊欄、有廣告、有各種互動元素。

那如果你今天打開的不是首頁，而是 https://israynotarray.com/atom.xml，你看到的是一份這樣的東西

這就是 RSS（嚴格來講上面是 Atom，可是先不管，後面會講差別）。

它是一份「給機器讀的版本」，沒有 CSS、沒有 JavaScript、沒有廣告，就是純粹的內容清單。RSS 閱讀器就是負責去定期抓這些 XML 檔案，把所有訂閱來源的最新文章匯集到一個地方給你看。

用一個生活化的比喻

如果用比較生活面的例子來講的話，RSS 比較像「報紙的訂閱機制，可是反過來」。

傳統訂報紙是報社主動把報紙送到你家信箱，RSS 則是你主動去報社門口看「有沒有出新一期」，有的話就拿回家看。差別在於你不需要把地址（email）告訴任何人，純粹是你去拿，對方甚至不知道你是誰。

這個「你去拿、對方不知道是誰」的特性，是 RSS 在這個資料追蹤越來越嚴重的時代滿值得重提的一個賣點。

RSS 跟訂閱電子報差在哪？

這幾年 Newsletter（電子報）很紅，例如 Substack、ConvertKit 等等，所以滿多人會問 RSS 跟 Newsletter 是不是同一個東西。

答案是不一樣，差別還滿大的，這邊整理一下

面向 RSS 電子報（Newsletter）
推送方式 你主動去拉資料 對方主動寄到你信箱
需要的資訊 不需要 email 必須給 email
隱私 對方不知道你是誰 對方有你的 email、可以做開信追蹤
取消訂閱 你自己從閱讀器移除即可 要找到取消連結，有時候會被忽略
內容呈現 純內容，不含追蹤元件 通常會嵌入追蹤像素、UTM 參數
集中查看 一個閱讀器看所有來源 信箱跟個人通訊混在一起

從這張表可以看出來，RSS 跟電子報走的是兩條不同的路線。電子報的優點是被動推送、作者跟讀者距離近，可是缺點是你的信箱會被資訊洗版，而且你的個資（email）會被很多人拿到。

RSS 則是相反，你自己決定要不要去拉、什麼時候拉、用什麼工具讀，是一種更「資料主權留在自己手上」的方式。

為什麼 2026 年還要用 RSS？

接下來這部分可能比較主觀，可是我覺得有幾個原因，是這幾年讓我重新認真用 RSS 的關鍵。

演算法已經吃掉你大部分的注意力

你應該有遇過這種狀況，打開 IG 滑了 20 分鐘，發現自己都在看一些根本沒追蹤的人的影片；打開 X 想找某個人的最新動態，結果首頁全部都是被推薦的奇怪內容；打開 YouTube 想看訂閱頻道，結果首頁推薦的東西比訂閱還多。

這不是錯覺，這是平台的設計。

平台希望你停留越久越好，所以它會用各種方式塞「它認為你會看得下去」的內容給你，而你「主動訂閱誰」這件事，反而被排在很後面。

RSS 沒有這件事。你訂了誰，就只會看到誰，按發布時間從新到舊排列，乾乾淨淨。

你不會錯過你真正在意的內容

承接上面那點，正因為演算法會幫你「過濾」內容，所以你真正在追蹤的某些小眾來源，常常會被忽略掉。

以我自己來講，我有訂幾個寫得很好可是更新頻率很低的個人部落格，這種一個月可能才出一篇的作者，在 IG / Twitter 完全打不過演算法，可是在 RSS 閱讀器裡，他發了就是會出現在我的時間軸上，不會被埋掉。

一次處理所有資訊來源

技術部落格、新聞網站、開源專案的 Release、Hacker News、Reddit 的特定 subreddit、GitHub trending、甚至 Mastodon、Bluesky 都可以走 RSS，所以一旦你習慣了，你會發現不再需要這麼多 APP 各自打開查資訊，一個閱讀器全部搞定。

AI 時代資料來源反而更重要

這個比較少人聊。

AI 模型訓練的時候，其實滿依賴「結構化、語意清楚」的內容，而 RSS 友善的網站通常也是內容語意清楚的網站，所以未來 AI 在替你做摘要、整理資訊的時候，RSS feed 反而會變成一個重要的「乾淨資料源」。

換句話說，RSS 不只是給人看的，未來你想自己接一些 AI 工具來自動化資訊處理，RSS 也是最容易切入的入口。

Atom 又是什麼？跟 RSS 差別在哪？

前面有提到 Atom，這邊稍微說明一下。

Atom 跟 RSS 簡單來講是兩份不同的規格文件，可是做的事情幾乎一樣，都是把網站的最新內容用 XML 格式公開出來。差別大概是這樣

  • RSS（特別是 2.0 版）：比較老，欄位定義比較寬鬆，所以不同網站產出的 RSS 有可能格式不太一樣
  • Atom：後來才提出的標準，欄位定義比較嚴謹，例如時間欄位強制要 RFC 3339 格式

對使用者來講，這兩個其實沒什麼差，因為現在的 RSS 閱讀器通常兩種都吃，網站也常常兩種都提供。你訂的時候閱讀器會自動處理，不用煩惱是哪一種。

Note
除了 RSS 跟 Atom 之外，還有一個更老的格式叫 RDF（RSS 1.0），可是現在已經很少網站還在用，遇到的機率不高。

RSS 閱讀器有哪些選擇？

聊完規格，接下來就是工具。RSS 閱讀器市面上其實還滿多選擇的，這邊列幾個比較常見的

  • Feedly：應該是目前最知名的雲端 RSS 閱讀器，免費版有功能限制
  • Inoreader：另一個雲端 RSS 閱讀器，功能比 Feedly 多一點
  • NetNewsWire：macOS / iOS 原生 APP，開源免費
  • Reeder：Apple 平台老牌付費 APP
  • Miniflux：可以自架的開源 RSS 閱讀器，走極簡路線

這些選擇各有各的好，可是我自己用了一輪之後，發現幾個共同的問題

  1. 雲端服務都要註冊帳號、訂閱資料會跑到對方伺服器
  2. 多數要嘛是要付費才有完整功能，要嘛就是要自己架機器
  3. 桌面端的原生 APP 大多沒有 Chrome 擴充套件這種「順手在瀏覽器發現 RSS 就訂」的體驗

所以我就想說，乾脆自己寫一個。

我自己寫的 Chrome 擴充套件：Raydar

Raydar 是我這幾個月寫的 Chrome 擴充套件，目標就是做一個本地優先、不需要帳戶的極簡 RSS 閱讀器，把雲端 reader 的那些「要先註冊才能用」的步驟通通拿掉。

擴充套件已經上架 Chrome Web Store：Raydar — Chrome Web Store。

接下來介紹幾個我自己覺得比較有意思的特色。

邊瀏覽邊偵測 RSS

這是一開始就想做的功能。你應該有遇過一種狀況，你在某個部落格看到一篇很喜歡的文章，想訂閱可是找不到 RSS 連結在哪。

Raydar 會在你瀏覽任何網頁的時候自動掃描頁面有沒有 RSS / Atom 的 <link rel="alternate"> 標籤，有的話會跳出一個訂閱提示，按一下就完成訂閱，不用自己翻原始碼或挖 URL。

如果某個網站沒有對外 expose 這種 link 標籤，你也可以從工具列的 popup 點「訂閱目前的 feed」手動加入。

三種閱讀模式

讀文章的時候每個人偏好不太一樣，所以 Raydar 提供三種模式

  • 閱讀器模式：把文章內文擷取出來，乾乾淨淨的排版，可以自己調字級、深淺色
  • Web 模式：直接把原網頁嵌進閱讀頁，不用離開分頁
  • 原始 RSS 模式：直接看 feed 的 XML，給開發者除錯用

我自己最常用的是閱讀器模式，因為很多新聞網站的版型實在太花，閱讀器模式可以讓你只看到該看到的東西。

完全本地優先，不需要註冊

Raydar 不需要註冊、不需要 email、不需要密碼，裝完就用。

訂閱清單、文章快取、設定全部存在你瀏覽器本地，跨裝置同步走的是 Chrome 內建的 chrome.storage.sync，由 Google 加密處理，作者完全碰不到任何使用者的訂閱資料。

也沒有接 Google Analytics、沒有崩潰回報、沒有 AI API、沒有 Newsletter bridge，全部都是純本地處理。

鍵盤快捷鍵

對重度閱讀的人來講鍵盤快捷鍵是必備，這個我也補上了

快捷鍵 功能
J / K 切換上下篇文章
S 收藏 / 取消收藏
M 切換已讀 / 未讀
O 在新分頁開啟原文
Shift + R 重新刷新所有訂閱
? 顯示快捷鍵說明

設計上是參考 Gmail、Google Reader 那一套 J/K 慣例，所以用過類似工具的人應該會很順手。

全文搜尋是純本地的

訂閱量大了之後，搜尋會變成必備功能。

Raydar 的全文搜尋用的是 MiniSearch，整個索引在瀏覽器記憶體裡建立，搜尋過程不會跟任何伺服器互動。中文跟日文還有特別處理斷詞，所以搜「Vue 元件」這種詞也找得到，不會卡在英文斷詞邏輯上。

資料可攜、隨時搬家

我自己很在意一件事，就是「使用者要離開的時候不要有摩擦」。所以 Raydar 從一開始就把資料匯入匯出做好

  • OPML 匯入匯出：跟其他 RSS 閱讀器無縫遷移
  • JSON 完整備份：訂閱清單、文章快取、設定、收藏一鍵打包

也就是說你今天不喜歡 Raydar 了，匯出 OPML 丟到 Feedly / Inoreader / NetNewsWire 都可以，不會被鎖住。

其他

除了上面這些主要功能，還有一些細節

  • 繁體中文、English、System 三段語言切換
  • Light / Dark / System 三段主題
  • 未讀數會顯示在 toolbar 圖示的 badge 上，一眼看到還有多少新文章
  • 標籤分類、來源置頂、關鍵字 mute 等等管理工具

整體就是想做成那種「裝起來就忘記它存在、可是每天打開來看資訊很順手」的工具。

怎麼裝

Raydar 已經上架 Chrome Web Store，直接點下面這個連結，再按「加入 Chrome」就可以了

Raydar — Chrome Web Store

裝完之後在工具列點一下 Raydar 圖示，會跳出 popup，第一次使用裡面會有提示，按照流程走就可以開始訂閱了。

如果你之前已經有在用其他 RSS 閱讀器，記得把舊閱讀器的 OPML 匯出，丟到 Raydar 的設定頁就可以一次匯入所有訂閱來源囉。

結語

RSS 真的不是死去的技術，它只是不再是主流話題而已。

對於想要遠離演算法、想要主動掌握資訊來源、不希望自己被推薦系統綁架的人來講，RSS 是一個用了之後就回不去的東西，希望這一篇可以讓本來不知道 RSS 的人有個比較具體的認識，也希望 Raydar 對於本來就在用 RSS 的人，是一個值得試一下的選項哩～
