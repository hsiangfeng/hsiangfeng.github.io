Mac 系統監控工具就推薦用 Stats！一款免費又開源，還能看 CPU、GPU、溫度、網速的軟體

前言

這一篇來分享一下我自己 Mac 上最愛用的 Stats 吧。剛好最近從 M1 跳到 M5 了，雖然轉移工具幫我無痛轉移，但不知道為什麼 Stats 每次都轉移失敗，所以這套軟體幾乎是我新 Mac 第一個會去裝的軟體，所以這邊就來分享一下它能做什麼、怎麼裝、還有我自己的一些設定習慣。

如果你之前有看過我寫的 Mac 上裝軟體到底該用哪種方式？Homebrew、官網下載、curl 完整比較 或 關於我推的 Raycast 擴充套件，強化你的 Raycast！，那這一篇基本上算是同一個系列 — 都是我自己長期掛在 Mac 上的東西。

所以 Stats 是什麼？

照慣例，先簡單介紹一下 Stats 是什麼東西。

簡單來講，Stats 是一款 macOS 的系統監控工具，相較於活動監視器，它更完善且直覺一次性把你需要看的東西列出來，像是我就會三不五時注意一下 Mac 溫度、網路流量、記憶體用量等等。

而這一款軟體本身是 MIT 授權的開源專案，所以完全免費使用，沒有任何功能限制，也沒有廣告或付費升級的煩惱，基本上你也不用太擔心它在背後偷做什麼奇怪的事情，也不會只有你遭殃，畢竟超多人用的。

這邊也快速列出一下它能夠看的範圍：

模組說明
CPU處理器使用率、各核心狀態、頻率
GPU顯示卡使用率（Apple Silicon 跟外接 GPU 都支援）
記憶體記憶體使用率、Swap 狀態
硬碟各個磁碟的讀寫速度、剩餘空間
感測器各種感測器數值（CPU 溫度、GPU 溫度、電壓、功耗等）
網路即時上下載流量、公開 IP、Wi-Fi 訊號強度
電池電池電量、健康度、充放電狀態
藍牙已連線的藍牙裝置電量
時鐘可以掛多個時區的時鐘
Remote遠端監控其他 Mac 的狀態

Stats 的模組列表

神奇吧？這 Stats 還有提供 Remote 呢！對於專門放一台 Mac mini 在家裡跑 OpenClaw、Hermes 使用者來講，這個功能就非常實用，可以直接在 Menu Bar 上看到遠端機器的狀態，不用再開一個遠端桌面軟體去看。

雖然我個人還沒用過 Remote 就是了，但看了一下 Stats 官網 介紹是覺得滿棒的。

安裝 Stats

講了那麼多，所以該怎麼安裝呢？這邊我依然推薦 Homebrew 的方式。

只需要在終端機輸入以下指令就可以安裝 Stats：

cmd
brew install --cask stats

Note Stats 是一個 GUI 應用程式，所以要用 --cask 來裝，不能用 brew install stats（不加 cask 的話 Homebrew 會去找命令列工具的 formula）。

如果你不想用 Homebrew 也沒關係，直接去 GitHub 的 Releases 頁 抓最新版的 Stats.dmg。

Stats 跑在 Menu Bar 上的示意圖

我最常開的幾個模組

接下來就來分享每一個模組啦～

CPU 模組

CPU 模組

CPU 模組是我長期保持開著的模組，畢竟可以讓我即時看到 CPU 的使用率，讓我可以快速判斷「電腦現在在幹嘛」，特別是當電腦突然變慢的時候，第一個會想看的就是 CPU 是不是被吃滿了。

GPU 模組

GPU 模組

我如果有跑 LLM 的話，我就會打開來看一下，畢竟 GPU 的使用率在這種情況下會非常明顯地飆高，這樣就可以知道是不是模型正在運算中，還是說其實它卡在某個步驟沒動。

記憶體模組

記憶體模組

記憶體模組也是我長期會關注的點，畢竟記憶體不夠用的話，系統就會開始把一些東西丟到硬碟上去來做記憶體交換，這時候整體效能就會大幅下降。

Note 稍微講一下，macOS 的記憶體管理跟 Windows 不太一樣，它會把沒用到的記憶體拿去當 cache，所以你會看到 Memory used 很高，可是其實系統還有很多可用的空間。

硬碟模組

硬碟模組

這我自己就很少看了，所以基本上都是關閉，除非我剛好要做一些需要大量讀寫硬碟的工作，才會打開來看一下。

感測器模組

感測器模組

感測器模組對於我來講非常的重要，畢竟我是拿 Air 居多，如果溫度沒有控制或關注的話，Mac 就會進入 Thermal Throttle（降頻）的狀態，這時候不管做什麼都會卡卡的...

而 Stats 的感測器模組可以讓我直接在 Menu Bar 上看到一些重要的溫度資訊，例如：

  • CPU 各核心的溫度
  • GPU 溫度
  • SSD 溫度
  • 電池溫度
  • 主機板各區的溫度
  • 整體功耗

感測器模組的詳細資訊

比較需要注意的是，這個模組溫度只會在 Menu Bar 上看到。

網路模組

網路模組

這也是我長期開啟的功能之一，畢竟網路有問題、跑的快不快，這個就是最重要的模組惹。

Note 公開 IP 的部分 Stats 是去打一個外部 API 才拿得到，所以如果你很在意隱私，可以在網路模組的設定裡面把這個關掉。

電池模組

電池模組

嗯...如果你是 Intel 時期的 Mac，我認為這功能滿重要的，因為 Intel 時期的 Mac 相較於 Apple Silicon 的 Mac 來講，電池的表現跟健康度都比較不穩定，所以如果你是 Intel 的話，這個模組就很建議開著看一下。

藍牙模組

藍牙模組

藍牙模組我就幾乎沒用過了，所以就...如果你發現藍牙不穩時，可以拿來看看。

時間模組

時間模組

時間模組我自己也沒什麼使用過，但方便在於可以設定多個時區，這個對於需要跟海外時差合作的人來講會比較實用（吧）。

最多，我自己是會掛一個美西時間，方便看一些國外服務的維護時間。

額外小技巧分享

有時候因為 menu bar 的空間有限，所以我會把一些我核心注重的細節放在桌面上，你只需要點桌面右鍵 => 編輯小工具

在桌面上編輯 Stats 小工具

接著找到你要拖曳放到桌面上的工具，然後直接拖曳到桌面上就可以了。

把 Stats 小工具拖曳到桌面上

這樣就可以在桌面上看到一些更詳細的資訊了，像是我自己就會把 CPU、GPU、記憶體的使用率放在桌面上。

常見問題

接下來我也列出一下一些問題。

桌面小工具總是出現「No data available」的問題？

這個問題是因為你需要打開「macOS widgets」這個功能才有辦法顯示，因為 Stats 的桌面小工具是用 macOS 原生的 Widget 功能來實現的，所以如果你沒有打開這個功能的話，就會一直看到「No data available」的提示。

Stats 桌面小工具出現 No data available

但這邊也要注意一下，開啟這個功能是會導致 Stats 一直針對硬碟做讀寫的，這個問題在官方 Issues 也有人提出過：Widgets do not update or high disk writes

桌面工具一直出現「Enable in Settings」的問題？

這方面其實我覺得一直都會發生，所以只能等官方修正了。

結語

最後就來總結一下吧。

所以我個人推薦使用的原因有以下：

  • 開源、免費、MIT 授權
  • 模組化設計，要看的才開，不要的關掉
  • 畫面乾淨、支援中文
  • 可以看到 macOS 原生不太願意給你看的東西，例如 CPU 溫度

所以希望這一篇對你有幫助哩～
