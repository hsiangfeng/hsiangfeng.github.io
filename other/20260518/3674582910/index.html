VSCode 連結都自動開在內建瀏覽器？教你關掉整合瀏覽器自動接管

前言

VSCode 1.109 推出新的整合瀏覽器（Integrated Browser）後，只要 workbench.browser.openLocalhostLinks 這個設定被打開，點終端機或聊天視窗裡面的 localhost 連結就會被自動拉進編輯器內顯示，畫面又小又難用。所以這一篇會記錄一下 workbench.browser.openLocalhostLinks 跟 simpleBrowser.useIntegratedBrowser 這兩個關鍵設定怎麼關，以及還是想偶爾用一下整合瀏覽器時可以怎麼從命令面板手動叫出。

為什麼會變成這樣？

這個行為其實不是 VSCode 一直都有的，而是從 1.109 版（差不多是 2026 年 1 月）開始新增了一個叫做「整合瀏覽器（Integrated Browser）」的功能，當這個功能開著時，只要 VSCode 偵測到你點的是 localhost 連結，就會優先把這個連結拉進編輯器內顯示，這樣你就不用切換到其他應用程式去看畫面。

立意上來講是不錯，但對許多開發者來講其實是滿困擾的，因為編輯器本來就已經被切成檔案總管、編輯區、輸出視窗等好幾塊，再多塞一個瀏覽器進來，能用的空間就會變得很小，你想在裡面測 RWD、開 DevTools 都不太方便 QQ...

整合瀏覽器跟 Simple Browser 有什麼關係？

如果你有用 VSCode 一段時間了，那你應該會聽過 Simple Browser，這是 VSCode 早期就有的一個內建瀏覽器套件，但它的限制比較多，例如：

  • 只能顯示靜態頁面
  • 不能登入需要驗證的網站
  • 不會保留 cookie

所以以前大家頂多拿來瞄一下 localhost 起來有沒有跑而已。

那麼新版的整合瀏覽器跟 Simple Browser 之間又是什麼關係呢？

簡單來講，整合瀏覽器是 Simple Browser 的進階版本，但兩者目前是並存的，並不是新的把舊的取代掉，你可以透過設定來決定命令面板叫出來的是哪一個。

整合瀏覽器強化了 Simple Browser 所缺乏的能力，所以它可以做到：

  • 登入需要驗證的網站（cookie 跟登入狀態都會被保留）
  • 瀏覽任何頁面，不限 localhost
  • 內建 DevTools，可以直接檢查元素
  • 把選到的元素丟給 Copilot 當作上下文

簡單來講，就是 VSCode 把一個比較接近 Chrome 的小瀏覽器塞進了編輯器裡，讓你不需要離開 VSCode 也能完整看一個網頁，而不是像以前 Simple Browser 那樣只能看個半套。

那為什麼點連結會被自動拉進去？

整合瀏覽器本身只是「多了一個可以用的瀏覽器」，會發生「點連結就被搶走」這件事，是因為 VSCode 提供了一個叫做 workbench.browser.openLocalhostLinks 的設定，只要這個設定是打開的，VSCode 任何地方（終端機、聊天視窗、Markdown 預覽、輸出面板等等）點到的 localhost 連結都會被導去整合瀏覽器，而不是丟給你系統預設的瀏覽器。

Note

1.109 釋出說明裡面寫的是「可以打開這個設定」，所以理論上預設是關閉的，但有些情境下會在你沒注意的時候被打開（例如點過某個提示、or 裝了某個套件偷偷幫你開），這時候你就會發現怎麼點 localhost 連結都跑去 VSCode 內。

解決方法

知道是哪個設定在搞鬼之後，把它關掉就解決了，這邊提供兩種方式，看你比較順手哪一種。

方法一：透過設定面板關掉

首先打開 VSCode 的設定面板：

  • macOS：Cmd + ,
  • Windows / Linux：Ctrl + ,

接著在最上方的搜尋框輸入：

Text
workbench.browser.openLocalhostLinks

在 VSCode 設定面板搜尋 workbench.browser.openLocalhostLinks

接著你應該會看到這個設定，把它的勾勾取消掉就好，這邊不需要重開 VSCode，會立刻生效。

方法二：直接改 settings.json

如果你比較喜歡改 JSON 檔案（or 有同步多台機器設定的需求），那也可以走這條路。

先按 Cmd + Shift + P（Windows 是 Ctrl + Shift + P）打開命令面板，然後輸入：

Text
Preferences: Open User Settings (JSON)

選擇之後它會幫你開啟 settings.json，接著我們只要在裡面加入這個設定：

JSON
··{
  "workbench.browser.openLocalhostLinks": false
}

在 VSCode settings.json 加入 workbench.browser.openLocalhostLinks 設定

存檔後就會立刻生效，再去終端機點一次 localhost 連結，畫面就會回到你系統的預設瀏覽器了哩～

順便處理：Simple Browser 套件自己的設定

關掉 workbench.browser.openLocalhostLinks 之後，理論上 localhost 連結就會回到正常，但這邊要特別提一下，VSCode 還有一個跟整合瀏覽器有關的設定叫做 simpleBrowser.useIntegratedBrowser，這個其實是「Simple Browser 套件自己的設定」，並不是 VSCode 核心提供的，從命名上其實就可以看出來。

設定屬於它在管什麼
workbench.browser.openLocalhostLinksVSCode 核心點 localhost 連結時要不要自動走整合瀏覽器
simpleBrowser.useIntegratedBrowserSimple Browser（內建套件）從命令面板叫 Simple Browser 時要不要改走新版整合瀏覽器

從這張表你應該可以發現，前綴名稱其實就反映了它的出處，workbench.* 是 VSCode 自己的，而 simpleBrowser.* 則是屬於 Simple Browser 這個內建套件的，所以如果你關了核心設定之後還是發現從命令面板叫出來的瀏覽器跑去新版整合瀏覽器，那就是這個套件設定還開著，搜尋一下名稱關掉就好。

還是想偶爾用一下整合瀏覽器？

把自動接管關掉之後，整合瀏覽器並不是就完全不能用了，如果哪天你想在編輯器內快速確認一下某個畫面，還是可以手動把它叫出來。

一樣按 Cmd + Shift + P（Windows 是 Ctrl + Shift + P）打開命令面板，接著輸入：

Text
Simple Browser

Simple Browser

按 enter 後輸入你想看的網址，整合瀏覽器就會在編輯器內開出一個分頁，這樣需要的時候自己叫出來，不需要的時候連結也不會被搶走。

結語

整合瀏覽器這個功能本身其實是不錯的，但預設行為對於習慣用外部瀏覽器當主力的人來講就比較困擾，那個小視窗實在難以取代日常開發在用的 Chrome 或 Edge，因此這時候把它關掉是比較好的選擇。

希望這一篇有讓你對 VSCode 整合瀏覽器的自動接管行為與關閉設定有更近一步的認識哩～如果你還想再順手把 Mac 端的開發環境整理一下，可以再看看 Mac 上裝軟體到底該用哪種方式？Homebrew、官網下載、curl 完整比較 這一篇，跟這一篇可以串成一條更完整的環境整理流程，我們下篇文章見囉～
