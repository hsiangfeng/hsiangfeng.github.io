前言

最近開發環境轉到 Mac OS X 上，所以這一篇就稍微紀錄一下 Mac 與 Windows 轉移之痛 Q_Q

鍵盤不同

首先最大不同就是以往的 Alt、Ctrl 會稍微有點不同，Mac 系統上面的這兩個按鈕所代表的稍稍不太一樣，更不用說切換輸入法的部分。

Windows Mac
Ctrl Control ⌃
Alt Option ⌥
Windown Command ⌘
Shift Shift ⇧
Caps Lock Caps Lock ⇪

切換輸入法

Windows 10 的切換中央文輸入法是按 Shift (預設狀況下)。

而 Mac 切換輸入法是在一個箭頭按鈕的上方的按鈕(Caps Lock ⇪)來切換中英文輸入法。

此外比較特別的是中文輸入法時若按著 Shift 出現的標點符號皆為全形，關於這一點我還真必須習慣一下。

輸入法

另外 Mac 的預設輸入法在選字是後選字的方式，意指選字的標點閃爍符號要在後面，關於這一點我也超不習慣，通常我在 Windows 系統選字是前選字(很難截圖到，所以就自己體驗囉~)。

基本上輸入法在中文輸入按住 Caps Lock 鍵 ⇪ 的時候會是小寫英文，例如：「ａｂｃｄ」

而 Shift ⇧ 則會是大寫英文，例如：「ＡＢＣＤ」，但是這邊你會發現一件事情都是全形的，所以如果你想要打半形英文的話，那就要切換成英文。

至於標點符到的部分基本上都是 Shift ⇧ + 標點符號 就可以了。

觸碰版手指操作

基本上如果有使用過 Windows 10 的筆電的話，對於 Mac 的觸碰版操作是不會覺得差異到哪裡去，因為會有 87 像，像是三指滑動虛擬桌面、兩指移動滾輪畫面以及縮放等等。

除了開啟網頁是上下頁可以使用雙指切換之外，基本上沒有太大的差異。

其他的部分則可以在「系統偏好設定」→ 「觸碰式軌跡板」這邊會有很多手勢教學。

只是在滾輪上會跟 Windows 稍微有點不太一樣，Windows 滾輪若向下滾則畫面會向下，但 Mac 是相反的。

所以基本上我傾向使用觸碰板來操作畢竟比較符合人體操作行為 XD

操作相關

其他操作相關按鈕也稍微有點不太一樣，舉例來講 複製、剪下以及貼上在 Windows 系統的快捷鍵是 Ctrl + X、C、V 等，但在 Mac 則是 Command + C、X、V 等。
