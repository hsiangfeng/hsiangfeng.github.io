Mac 上裝軟體到底該用哪種方式？Homebrew、官網下載、curl 完整比較

#Mac #macOS #nvm #Homebrew #軟體安裝 #開發環境 #mise

前言

如果你剛開始用 Mac 做開發，應該會發現一件事情：同一個工具，安裝方式居然有好幾種。像是裝 Git，有人說用 Xcode Command Line Tools 就有了，有人說用 Homebrew 裝比較好；裝 Node.js，有人說去官網下載，有人說用 nvm，有人說用 Homebrew。

這篇就來把 Mac 上面常見的安裝方式整理一下，搭配實際例子讓你知道什麼情境下該用哪種方式。

為什麼推薦用 Homebrew

在講各種安裝方式之前，先聊一下為什麼大部分教學都會推薦用 Homebrew。

Homebrew 是 Mac 上面最多人用的套件管理工具，如果你還沒安裝的話，可以參考我之前寫的 Mac OS 終端機美化看這一篇就夠了！。簡單來講它就是讓你用終端機指令來管理軟體的工具，那麼它到底比去官網下載好在哪裡？

統一管理所有軟體

用 Homebrew 裝的軟體，不管是命令列工具還是 GUI 應用程式，全部都可以用指令來管理：

如果是去官網下載的軟體，你要更新就得自己再去官網抓，要移除就得自己找殘留檔案清理，裝了二三十個工具之後就會變得很難管理。

自動處理相依性

有些工具在安裝的時候會需要其他工具先裝好，像是你裝某個套件可能需要先有 openssl 或 cmake，如果你是手動安裝的話，你得自己一個一個去找這些相依套件。Homebrew 會自動幫你處理這些，缺什麼它就幫你補什麼。

用 Brewfile 備份還原整台電腦的軟體

這是 Homebrew 一個很實用的功能，你可以把目前電腦上所有用 Homebrew 裝的軟體匯出成一個 Brewfile：

所以如果你換新電腦，或是想跟同事同步開發環境，只要把 Brewfile 分享出去，對方一個指令就能裝好所有東西。這個是去官網一個一個下載完全做不到的事情。

Formulae 跟 Cask

Homebrew 有兩種安裝模式，分別對應不同類型的軟體：

brew install 裝的是命令列工具，brew install --cask 裝的是有圖形介面的應用程式。用 Cask 裝的效果跟你去官網下載 .dmg 是一樣的，可是好處是之後可以用 brew upgrade --cask 統一更新，用 brew uninstall --cask 完整移除。

其他安裝方式

除了 Homebrew 之外，還有幾種安裝方式各有各的適用場景。

Xcode Command Line Tools

如果你不是做 iOS 開發，其實不需要裝完整的 Xcode，只要裝 Xcode Command Line Tools 就好，它包含了 Git、make、clang 這些基本的開發工具：

這通常是拿到新 Mac 之後第一個要跑的指令，因為很多工具（包含 Homebrew）都會需要它。

官網下載 .dmg / .pkg

就是去軟體的官方網站下載安裝檔。這種方式簡單直覺，可是缺點是更新要自己處理，移除的時候也不一定能清乾淨，因為有些軟體會在系統各處放設定檔。

如果同一個軟體 Homebrew 也有的話，建議優先用 Homebrew，管理起來方便很多。

curl 安裝腳本

有些工具的安裝方式是給你一行 curl 指令：

這種方式通常是因為這些工具需要修改你的 shell 設定檔（像是 .zshrc），用 Homebrew 的標準流程不太好處理，所以用腳本比較方便。Homebrew 本身也是用 curl 安裝的。

Note
用 curl 跑腳本之前，建議先確認來源是可信賴的官方儲存庫，因為這等於是直接在你的電腦上執行來自網路的程式碼。

版本管理器

如果你同時維護多個專案，而這些專案用的語言版本不一樣（像是 A 專案用 Node.js 18、B 專案用 Node.js 20），那你需要的是版本管理器，而不是直接用 Homebrew 裝。

常見的版本管理器：

工具 管理的語言 安裝方式
nvm Node.js curl 腳本
pyenv Python Homebrew 或 curl 腳本
rbenv Ruby Homebrew
mise 全部都能管（900+ 種工具） Homebrew 或 curl 腳本

其中 mise 可以同時管理 Node.js、Python、Ruby、Go、Java 等等，所以你不用為每個語言分別裝一個版本管理器。

如果你有興趣了解 nvm 的實際使用方式，可以參考我之前寫的 Mac 開啟專案時，自動使用 nvm use 切換到專案的 Node 版本。

實際案例

接下來用幾個常見工具來看看它們分別適合用哪種方式安裝。

Git

Git 裝完 Xcode Command Line Tools 就有了，不過 Xcode CLT 附的版本通常比較舊。如果你想用比較新的版本，用 Homebrew 裝就好：

Homebrew 裝的 Git 會自動取代 Xcode CLT 的版本，之後要更新只要跑 brew upgrade git。

建議：一般使用 Xcode CLT 的就夠了，在意版本的話用 Homebrew。

Node.js

Node.js 是最多人搞混安裝方式的工具：

去官網下載 .pkg 安裝檔不太推薦，因為它裝在系統層級的路徑，之後要切換版本或移除都比較麻煩。

建議：前端或 Node.js 開發者用 nvm 或 mise，偶爾才用到的話 Homebrew 就夠了。

Python

Mac 有內建 Python，可是版本舊而且不建議拿來開發：

建議：有在寫 Python 的話用 pyenv 或 mise，只是因為其他工具需要 Python 的話用 Homebrew。

VS Code

VS Code 是 GUI 應用程式，兩種裝法結果一樣：

或是去 VS Code 官網 下載 .dmg。

建議：已經在用 Homebrew 的話用 Cask 裝，管理起來比較統一。

總結

整理一下大原則：

你要裝的東西 建議的安裝方式
命令列工具（git、wget、jq） Homebrew
GUI 應用程式（VS Code、Chrome） Homebrew Cask 或官網下載
需要多版本的語言（Node.js、Python） 版本管理器（nvm、pyenv、mise）
Xcode、Mac 專屬 App Mac App Store
版本管理器本身（nvm、mise） curl 腳本或 Homebrew

以我自己的做法來講，拿到新 Mac 的安裝順序大概是這樣：

  1. xcode-select --install（基本開發工具）
  2. 裝 Homebrew（套件管理核心）
  3. 命令列工具全部用 Homebrew
  4. Node.js 用 nvm 管理
  5. GUI 應用程式看情況用 Homebrew Cask 或官網下載

其實不用把每種方式都記住，只要記得一個原則：能用 Homebrew 管理的就用 Homebrew，需要多版本的就用版本管理器，這樣就不太會選錯了哩～
