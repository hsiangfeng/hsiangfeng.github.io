Intro

Picking up the second half of the last post — let’s talk about the Dockerfile.

What is a Dockerfile?

A Dockerfile is essentially a script file. It tells Docker what to do in order to build a Docker Image. To know what Docker’s doing, you need to understand the commands you’ll see inside.

Dockerfile commands

Let’s walk through the common commands.

FROM

Tells Docker which Image to use as the base. Since an Image is a kind of disk image (we covered that last post), FROM is just saying “use this image as the starting point”.

CMD

CMD is similar to RUN, but with a key difference — CMD runs when the Container starts, while RUN runs when the Image is being built. Different lifecycle phases.

This one’s a bit subtle. The mental model: a Dockerfile runs line by line, and after the whole thing finishes, the result is packaged into an Image. RUN executes during the build (Image creation), while CMD executes only when you actually start a Container from that Image.

ARG

ARG is similar to ENV, but they run in different phases. ARG is available during the Image build, while ENV is available when the Container starts.
