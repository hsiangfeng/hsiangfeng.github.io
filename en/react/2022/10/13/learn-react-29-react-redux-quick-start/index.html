Intro

This post kicks off a brand new chapter. I'm only splitting it into two parts though, so there's a bit less to cover than usual. So what exactly is Redux? Let's get to know it in this post.

What is Redux?

So what is Redux? Redux is basically a state management tool. You can also think of it as a container — it helps us manage the state of an entire web app, and Redux itself can be used on its own.

Like I just said, Redux is a tool built specifically for state management and it can run standalone. That's why right on the official homepage you'll see this line:

A Predictable State Container for JS Apps

So we know Redux can be used independently. But some people see Redux and assume it's React's own state management tool, kind of like Vuex is for Vue. Because Redux can run on its own, though, you can actually use it with frameworks and libraries other than React too — Angular, Vue, jQuery, and so on. On top of that, Redux is tiny, only about 2kb.

At this point you might be thinking back to the useReducer example from an earlier chapter:

jsx
················································································const reducer = (state, action) => {
  switch(action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    default:
      return { count: 0 }
  }
}

const App = () => {
  const initialState = {
    count: 0,
  };

  const [state, dispatch] = React.useReducer(reducer, initialState);

  return (
    <div>
      Count: {state.count}
      <br />
      <button onClick={() => dispatch({ type: "decrement" })}>Decrement</button>
      <button onClick={() => dispatch({ type: "increment" })}>Increment</button>
    </div>
  );
}

const app = document.querySelector('#app');
const root = ReactDOM.createRoot(app);
root.render(<App />);

So what's the relationship between useReducer and Redux? At its core, useReducer is a Hook that the React team built after looking at Redux. But in practice the two still have some real differences. For example, Redux lets us share state across different pages or different components, whereas useReducer can only share state within the same component. Unless you bring in the Context API, under normal circumstances useReducer can't share state between different components.

The relationship between React Redux and Redux

So like I mentioned, Redux can be used on its own. But if it can run standalone, then what's the relationship between React Redux and Redux? It's basically because React uses Redux for state management so often that the React side put together a React Redux package. That package integrates Redux with React so we can use Redux more conveniently.

Installing React Redux and Redux Toolkit

Next up we're going to install React Redux and Redux Toolkit. Both of these are Redux-related packages, and Redux Toolkit is an official package from the Redux team that makes using Redux a lot easier.

So why use Redux Toolkit? If you head over to the Redux Toolkit docs, you'll see three pain points listed:

  • "Configuring a Redux store is too complicated"
    • Configuring Redux is too complicated
  • "I have to add a lot of packages to get Redux to do anything useful"
    • You have to install a bunch of packages just to get Redux doing anything useful
  • "Redux requires too much boilerplate code"
    • Redux requires too much boilerplate code

Because of those three things, we're going to use Redux Toolkit to set up Redux here. So first we install Redux Toolkit and React Redux:

Bash
npm install react-redux @reduxjs/toolkit

One thing to watch out for when installing: React Redux 8.x needs at least React 16.8.3 to use React Hooks.

Creating a Redux Store with Redux Toolkit

For this part we'll use this sample code as our starting point. I'd suggest downloading it first so it's easier to follow along.

If there are no issues, start by creating a store folder under src, then create an index.js file inside the store folder, and add the following code to index.js:

jsx
··import { configureStore } from "@reduxjs/toolkit";

const store = configureStore({
  reducer: {},
});

export default store;

Next, open src/main.jsx, import import store from './store' and import { Provider } from 'react-redux', and wrap <App /> with <Provider store={store}>:

jsx
··················import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App'
import './index.css'
import store from './store'
import { Provider } from 'react-redux'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <Provider store={ store }>
      <App />
    </Provider>
  </React.StrictMode>
)

So far we've finished setting up React Redux. Next we're going to tweak our original ToDoList code so it uses Redux for state management.

Creating a Redux Slice with Redux Toolkit

Now we'll create a React Slice, so create a todoSlice.js file inside the store folder, and import createSlice in todoSlice.js:

jsx
import { createSlice } from "@reduxjs/toolkit";

createSlice is a method provided by Redux Toolkit that makes creating a Redux Slice much easier. createSlice returns an object containing a reducer and actions, so we can use it like this:

jsx
··const todoSlice = createSlice({
  name: "todo"
});

There are two important things inside createSlice: initialState and reducers. initialState is the initial state, and reducers is an object containing all the actions. You can think of an action as a function, and each of these functions takes in two parameters, state and action.

Next let's convert the ToDoList code into Redux code. First we set up initialState, so the original const [ todoList, setTodoList ] = useState(JSON.parse(localStorage.getItem('todoList')) || []); that lived in App.jsx becomes the value of initialState:

jsx
··········import { createSlice } from "@reduxjs/toolkit";

const store = createSlice({
  initialState: {
    todoList: []
  },
  reducer: {},
});

And addTodo? That goes inside reducer:

jsx
······················import { createSlice } from "@reduxjs/toolkit";

const todoSlice = createSlice({
  name: "todo",
  initialState: {
    todoList: [],
  },
  reducers: {
    addTodo (state, action) {

    }
  },
});

Then we change the original setTodoList to state.todoList.push(action.payload);.

There's one important thing about Slices: when we export, we don't export todoSlice directly. Instead we export todoSlice.reducer, because todoSlice contains both reducer and actions, and we only need the reducer. So we export both todoSlice.reducer and todoSlice.actions:

jsx
export const { addTodo } = todoSlice.actions;
export default todoSlice.reducer;

So far we've converted the original code into Redux code. Next we export todoSlice and import it in src/store/index.js:

jsx
········import { configureStore } from "@reduxjs/toolkit";
import todoSlice from "./todoSlice";

const store = configureStore({
  reducer: {
    todoSlice,
  },
});

export default store;

Finally, back in App.jsx, we swap the original useState for useSelector and useDispatch, so we import useSelector and useDispatch:

jsx
import { useSelector, useDispatch } from "react-redux";

Then how do we grab the todoList that's sitting in Redux? We can get it like this:

jsx
const todoList = useSelector((state) => state.todoSlice.todoList);

As for addTodo, we additionally need to import addTodo:

jsx
import { addTodo } from "./store/todoSlice";

Now we hit a problem: since we named it addTodo in App.jsx and we also named it addTodo in todoSlice, there's a naming collision. So we rename the addTodo in App.jsx to handleAddTodo so it doesn't clash with todoSlice. The code now looks like this:

jsx
····························const handleAddTodo = (event) => {
  dispatch(addTodo({
      id: Date.now(),
      name: event.target.previousElementSibling.value,
      status: false
    }
  ));
  event.target.previousElementSibling.value = '';
};

Next, the original List was receiving setTodoList. On top of that, we also need to move the updateTodo that was originally written under List into todoSlice. The code now looks like this:

jsx
······················································import { createSlice } from "@reduxjs/toolkit";

const todoSlice = createSlice({
  name: "todo",
  initialState: {
    todoList: [],
  },
  reducers: {
    addTodo (state, action) {
      state.todoList.push(action.payload);
    },
    updateTodo(state, action) {
      const id = action.payload;
      const index = state.todoList.findIndex((todo) => todo.id === Number(id));
      state.todoList[index].status = !state.todoList[index].status;
    },
  },
});

export const { addTodo, updateTodo } = todoSlice.actions;

export default todoSlice.reducer;

Then the original code in List becomes this:

jsx
····const handleUpdateTodo = (event) => {
  const { id } = event.target.dataset;
  dispatch(updateTodo(id));
}

And with that, we've converted the original code into Redux code. This is just a simple quick start, but as always the sample code is available here.
