Now let’s move the code that was sitting on CodePen over into our project.

Moving the code over

For this part, I’d suggest renaming App.js to App.jsx. Whether the extension is .js or .jsx doesn’t actually make a difference and won’t cause any errors, but renaming it to App.jsx makes it easier to tell at a glance that the file uses JSX syntax. Down the road, when you see that extension, you’ll immediately know there’s JSX inside.

Next, just delete everything inside App.js and paste in the const App = () => {} part from CodePen. There’s one small detail to watch out for though. Originally we wrote our Hook like this: const [ todoList, setTodoList ] = React.useState(JSON.parse(localStorage.getItem('todoList')) || []);. But in CRA we’re importing Hooks a different way, so the syntax changes to const [ todoList, setTodoList ] = useState(JSON.parse(localStorage.getItem('todoList')) || []);. All we did was drop the React. prefix.

Deploying to GitHub Pages

So far we’ve moved the TodoList project into a real dev environment. Next we’re going to deploy the project to GitHub Pages so everyone can use our TodoList through a URL.

Step four: check the deploy result

So how do you know the deploy succeeded? The first place to look is the branch section. If the deploy worked, you’ll see your branches go from one to two: main and gh-pages.

The second place is the Settings tab. Scroll down and you’ll find the GitHub Pages section. If the deploy worked, you’ll see your GitHub Pages URL there.
