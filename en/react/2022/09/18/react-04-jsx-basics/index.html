Intro

You’ve probably heard people throw around the term “JSX” — but what actually is it? It kind of sounds like one of those things you absolutely have to learn if you want to write React.

What is JSX?

Before you write any React, you’ll need to understand JSX. And honestly, we’ve been writing JSX all along in the earlier posts — we just never stopped to look at it properly. So let’s get to know what JSX actually is.

Like I just said, we’ve already written JSX before — maybe you just didn’t notice. So… when exactly did we write JSX earlier? Let me pull up one of the previous examples.

That bit inside root.render(<h1>Hello React</h1>); — the <h1>Hello React</h1> part — that’s JSX.

JSX isn’t HTML, it isn’t a string, and it definitely isn’t CSS (well, obviously). It’s an extension to JavaScript that React introduced. The full name is JavaScript & XML, and it’s basically an “HTML in JavaScript” syntax extension (I’ll just call it JSX from here on).

And when you try to output the variable const example = <h1>Hello React</h1>;, you’ll notice JSX automatically converts into a React Element.

By comparison, using JSX directly is way nicer and easier to learn — it feels almost exactly like writing plain HTML.

So you might be wondering: why do we even need key? What’s it for? Basically key helps React identify whether an element has been changed, deleted, and so on, so it can decide whether to re-render. Let’s look at the example below.

That’s a simple example. Try typing 1, 2, 3 into the input fields first, then hit the “Reverse” button. You’ll notice the fields stay where they are, but the data has flipped — and that actually leads to some problems.

Now try adding key and type 1, 2, 3 again, then hit “Reverse”. This time you should see the 1 and 3 in the fields disappear, because React detected those two pieces of data changed and decided to re-render.

That’s exactly why React recommends adding key. And last thing to remember: if you can, use a unique value from your array data as the key, and only fall back to using the index as a last resort. Otherwise some weird stuff can happen.

JSX Expression

Let’s keep going with JSX. JSX can also embed an expression.

If you’re not sure what an expression is, you can read my earlier post: JavaScript Core Concepts (11) - Operators, Types and Grammar - Statements and Expressions (article in Mandarin).

Using it is really simple — you just use curly braces (some folks call them braces; I’ll just call them curly braces) inside JSX and put a variable in there. As long as it’s an expression, it’ll work fine.

Since it supports expressions, we can also write functions that return a result.

You can even write some weird code in there — as long as it’s an expression, it all goes.

(Side note: console.log itself is also an expression — it just doesn’t return anything by default.)

Now, because JSX only accepts expressions, you can’t write an if...else statement inside JSX.
