You'll Have to Learn React Eventually, So Why Not Now? - React Basics - What Even Is React? - (1)

#Vue #Composition API #React Hooks #React Class #Framework #Library #React Basics #You'll Have to Learn React Eventually, So Why Not Now?

Intro

No matter what, you’ve got to start by getting comfortable with React’s basic concepts and a little bit of its background. That groundwork makes it much easier to spot the differences when you go pick up other frameworks later. On top of that, I’m a Vue developer myself, so honestly I’m writing this with the mindset of someone who just wants to add one more framework to the toolbox. That means you’ll probably see a few side-by-side comparisons between Vue and React syntax along the way — mostly because I want fellow Vue devs to have an easier time getting started. Okay, enough preamble. Let’s get into it.

What even is React?

The React official site actually tells you straight up that it’s a JavaScript Library — a library of JavaScript functions. Notice that? What I wrote there is a library, not a framework.

You can head over to the React official site and check for yourself, because that’s genuinely what it says.

Vue’s official site, on the other hand, says it’s a JavaScript Framework.

At this point you might be wondering…

“What about Angular? What’s Angular? I don’t see it written on its site.”

Angular is also a JavaScript Framework (Vue is basically what you get when the author cherry-picked the good parts of Angular and built a framework out of them). It’s just that Angular’s official site doesn’t explicitly spell out what it is. But we can find this line over on the Wikipedia page:

Angular is a TypeScript-based free and open-source web application framework led by the Angular Team at Google and by a community of individuals and corporations

So we can quickly sum up the categories of the three big frameworks:

  • Vue → JavaScript Framework
  • React → JavaScript Library
  • Angular → JavaScript Framework

Every now and then you might hear people say VAR, VAR — and that VAR is just the first letters of these three big names mashed together (definitely not the var you use to declare variables).

  • V -> Vue
  • A -> Angular
  • R -> React

Framework? Library?

Quick summary up front:

A Framework hands you a complete architecture and a full solution. A Library is something you have to assemble and wire up the architecture and solutions yourself.

To put it in more human terms: think of a Framework as walking into a home-improvement store, picking out the furniture, and ordering a whole kitchen-and-bathroom set. The store’s crew shows up at your place and builds the exact kitchen-and-bathroom combo you wanted.

A Library is more of a do-it-yourself deal. Picture going to the same store and buying the ready-made parts — the bathtub, the tiles, the range hood, and so on — but the catch is you’ve got to put it all together yourself. That’s your classic DIY.

I won’t go deeper into this point here.
