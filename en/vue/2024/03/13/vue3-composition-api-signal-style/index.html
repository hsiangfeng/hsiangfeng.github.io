createSignal

Here’s the function:

It imports two APIs:

  • shallowRef — declares a ref that doesn’t deep-watch. If you update a property inside the object, the view won’t re-render
  • triggerRef — manually forces an update. Since shallowRef doesn’t deep-watch, when you update a nested property the view doesn’t re-render — so you call triggerRef to force it

Both are a bit abstract — let’s see them in action. Here’s a normal ref example:

Click “Change name” all you want — the view doesn’t update. shallowRef doesn’t deep-watch, so updating a property doesn’t trigger a re-render. The only way is to replace the whole object:

That’s where triggerRef comes in — manually force the update:

See the Pen Vue Signal-4 by Ray (@hsiangfeng) on CodePen.

You might wonder — why bother with shallowRef + triggerRef when ref just works? In larger projects with complex data structures, deep-watching has a real perf cost. That’s when shallowRef + triggerRef becomes a much more efficient way to track and update specific changes.
