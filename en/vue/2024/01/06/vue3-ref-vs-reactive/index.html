Conclusion

So, in practice — ref or reactive? This honestly depends on team conventions, but the official Vue docs recommend ref. ref handles a lot of edge cases for you, and personally I default to ref too.

Another simple rule of thumb: if you need to reassign the data, use ref; if you don’t, reactive is fine.

One final takeaway —

When the value you pass in is a primitive, ref is implemented with RefImpl. When it’s an object or array, ref uses reactive under the hood. Worth knowing — maybe an interviewer will ask one day (laughs).
