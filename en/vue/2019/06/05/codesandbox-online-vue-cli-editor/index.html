Writing Vue CLI Online With CodeSandbox

#Vue #CodeSandbox

Intro

We usually code in CodePen, JSBin, or JSFiddle, but none of those can handle a Vue CLI app. CodeSandbox does — so let’s take Vue CLI online! (feels like an ad)

  • Note: even though CodeSandbox lets you write Vue CLI online, I wouldn’t recommend it as your actual dev environment — it has some quirks (for example, Bootstrap imports break unless you use the CDN). CodeSandbox is best for quick experiments.

Sign up for CodeSandbox

First, open the CodeSandbox site.

  • Side note: CodeSandbox actually lets you spin up Vue CLI instantly, but I still recommend signing up — that way you won’t lose your work if the page crashes.

Click Create Sandbox.

Then click Sign in with GitHub.

Once signed in, you’ll land on this screen.

Create a Vue project

Quick heads-up: CodeSandbox creates Vue CLI version 2.5.22, with support up to 2.6.10.

How do you create one? First go to your dashboard — if you’re not sure how, click here.

Then click Create Sandbox.

You’ll see a template picker — choose Vue (Vue-Cli).

CodeSandbox spins up a Vue CLI app for you just like that.

Install Vue Router

The default sandbox doesn’t include Vue Router, so we need to install it. Click Add Dependency.

You’ll see this dialog:

Type Vue-Router in the search box — the first result is the one we want. Click it.

And now Vue-Router shows up under Dependencies.
