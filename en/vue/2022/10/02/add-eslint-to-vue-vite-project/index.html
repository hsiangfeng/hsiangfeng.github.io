Note

Heads up: this post is based on ESLint 9.0+. Also note that vite-plugin-eslint (mentioned later) hasn't been maintained for a while.

Intro

This post walks through adding ESLint to a Vue Vite project, using the Airbnb or Standard style guide.

Setting up the environment

First, create a Vue 3 Vite dev environment:

Bash
npm init vue@3

Follow the prompts to build out the Vue 3 + Vite project.

For Vue 2:

Bash
npm init vue@2

I'll demo with Vue 3, using Airbnb's style guide for this first environment.

Environment setup

For reference, here's the prompt sequence:

Bash
✔ Project name: … vue3-eslint-airbnb
✔ Add TypeScript? … No(✅) / Yes
✔ Add JSX Support? … No(✅) / Yes
✔ Add Vue Router for Single Page Application development? … No / Yes(✅)
✔ Add Pinia for state management? … No / Yes(✅)
✔ Add Vitest for Unit Testing? … No(✅) / Yes
✔ Add Cypress for both Unit and End-to-End testing? … No(✅) / Yes
✔ Add ESLint for code quality? … No / Yes(✅)
✔ Add Prettier for code formatting? … No(✅) / Yes

Quick note: I'm not installing Prettier because we'll use ESLint's rules for formatting. Installing both would mean two formatters with conflicting opinions, so we'll just stick with ESLint.

Installing ESLint Airbnb

Before installing ESLint, let's look at what the project already has. Open package.json — ESLint is already installed:

JSON
····························································{
  "name": "vue3-template-airbnb",
  "version": "0.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview --port 4173",
    "lint": "eslint . --ext .vue,.js,.jsx,.cjs,.mjs --fix --ignore-path .gitignore"
  },
  "dependencies": {
    "pinia": "^2.0.21",
    "vue": "^3.2.38",
    "vue-router": "^4.1.5"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^3.0.3",
    "eslint": "^8.22.0",
    "eslint-plugin-vue": "^9.3.0",
    "vite": "^3.0.9"
  }
}

There's also a .eslintrc.cjs in the project root:

JavaScript
······················/* eslint-env node */
module.exports = {
  root: true,
  'extends': [
    'plugin:vue/vue3-essential',
    'eslint:recommended'
  ],
  parserOptions: {
    ecmaVersion: 'latest'
  }
}

The default just enforces Vue 3's basic rules — Airbnb isn't included yet.

To install Airbnb:

Bash
npm install --save-dev eslint-config-airbnb-base

eslint-config-airbnb-base is Airbnb's core rule package.

If you wanted Standard, you'd install eslint-config-standard instead. For Google, eslint-config-google. This post focuses on Airbnb, so let's stick with eslint-config-airbnb-base.

Update .eslintrc.cjs

Now update the config. Default:

JavaScript
······················/* eslint-env node */
module.exports = {
  root: true,
  'extends': [
    'plugin:vue/vue3-essential',
    'eslint:recommended'
  ],
  parserOptions: {
    ecmaVersion: 'latest'
  }
}

Updated for Airbnb:

JavaScript
··························/* eslint-env node */

module.exports = {
  root: true,
  extends: [
    'plugin:vue/vue3-essential',
    'eslint:recommended',
    'airbnb-base',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
  },
};

For Standard, swap airbnb-base for standard.

Pay attention to the order of extends — ESLint applies them in order. airbnb-base should come last so Vue 3 rules are applied first, then Airbnb on top. If you put airbnb-base first, Airbnb rules would apply first and then conflict with Vue 3 rules, throwing errors.

You can verify this against how Vue CLI projects do it:

JavaScript
······················module.exports = {
  root: true,
  extends: [
    'plugin:vue/vue3-essential',
    '@vue/airbnb',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
  },
};

@vue/airbnb comes last for the same reason.

Side note: @vue/airbnb is Vue's tweaked version of the Airbnb rules. If you don't want to use @vue/airbnb, plain airbnb-base works, but you'll likely spend more time tuning rules.

ESLint is now set up. Test it by opening main.js — you should see lots of red "Missing semicolon.eslint" errors:

eslint

For auto-fixing on save, see my earlier post "Auto-fix ESLint errors in VSCode (article in Mandarin)".

Open App.vue and try deleting things from template or declaring unused variables to confirm ESLint flags them:

ESLint

Install eslint-import-resolver-alias

Since Airbnb is strict, you'll also want eslint-import-resolver-alias so ESLint can resolve the @ alias defined in vite.config.js:

JavaScript
····················import { fileURLToPath, URL } from 'node:url';

import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
});

Without this plugin, code like this errors:

HTML
····················································<script setup>
import { RouterLink, RouterView } from 'vue-router';
import HelloWorld from '@/components/HelloWorld.vue'; // Unable to resolve path to module '@/components/HelloWorld.vue'.eslint(import/no-unresolved)
</script>

<template>
  <header>
    <img alt="Vue logo" class="logo" src="@/assets/logo.svg" width="125" height="125" />

    <div class="wrapper">
      <HelloWorld msg="You did it!" />

      <nav>
        <RouterLink to="/">Home</RouterLink>
        <RouterLink to="/about">About</RouterLink>
      </nav>
    </div>
  </header>

  <RouterView />
</template>

Install:

Bash
npm install --save-dev eslint-import-resolver-alias

Open .eslintrc.cjs and add settings underneath extends. Inside settings, configure import/resolver with the alias map and extensions:

JavaScript
····················································································/* eslint-env node */

module.exports = {
  root: true,
  extends: [
    'plugin:vue/vue3-essential',
    'eslint:recommended',
    'airbnb-base',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
  },
  settings: {
    'import/resolver': {
      alias: {
        map: [
          ['@', './src'],
        ],
        extensions: ['.js', '.vue'],
      },
    },
  },
};

Reopen the file that was erroring — ESLint resolves the alias correctly now.

ESLint

Install eslint-plugin-import

eslint-plugin-import is required for the Airbnb config — it covers import-related rules like import order, file existence, and so on.

Bash
npm install --save-dev eslint-plugin-import

Without it you might see:

Bash
· The plugin "eslint-plugin-import" was referenced from the config file in ".eslintrc.cjs » eslint-config-airbnb-base » /home/runner/work/project-name/project-name/node_modules/eslint-config-airbnb-base/rules/imports.js".

If you hit this kind of error, install eslint-plugin-import.

Fixing vite.config.js errors

After all the setup, opening vite.config.js shows a weird error:

'vite' should be listed in the project's dependencies, not devDependencies. eslint(import/no-extraneous-dependencies) '@vitejs/plugin-vue' should be listed in the project's dependencies, not devDependencies. eslint(import/no-extraneous-dependencies)

vite.config.js

This one's odd because Vite's default setup puts these in devDependencies — that's intentional. We need to tell ESLint that these packages are allowed in devDependencies.

Add import/core-modules to settings:

JavaScript
········································································································/* eslint-env node */

module.exports = {
  root: true,
  extends: [
    'plugin:vue/vue3-essential',
    'eslint:recommended',
    'airbnb-base',
  ],
  parserOptions: {
    ecmaVersion: 'latest',
  },
  settings: {
    'import/resolver': {
      alias: {
        map: [
          ['@', './src'],
        ],
        extensions: ['.js', '.vue'],
      },
    },
    'import/core-modules': [
      'vite',
      '@vitejs/plugin-vue',
    ],
  },
};

Reopen vite.config.js and the error's gone.

vite.config.js

import/core-modules tells ESLint these packages are valid even when they live in devDependencies instead of dependencies.

Show ESLint warnings in the browser

If you want ESLint warnings visible in the dev browser, install vite-plugin-eslint:

Bash
npm install --save vite-plugin-eslint

Update vite.config.js:

JavaScript
········································import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import eslintPlugin from 'vite-plugin-eslint';
import { fileURLToPath, URL } from 'node:url';

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    vue(),
    eslintPlugin({
      include: ['src/**/*.js', 'src/**/*.vue', 'src/*.js', 'src/*.vue'],
    }),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url)),
    },
  },
});

Now ESLint warnings show up in the browser when there are errors:

ESLint

Extra

On some Node versions you might hit:

Bash
[vite] Internal server error: Failed to load plugin 'n' declared in '.eslintrc.cjs': Cannot find module 'eslint-plugin-n'

Fix:

Bash
npm install --save-dev eslint-plugin-n eslint-plugin-promise
