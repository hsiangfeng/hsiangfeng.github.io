【Day 15】Slow Computer, But I Don't Want to Reinstall Windows Again!

Slow Computer, But I Don’t Want to Reinstall Again

After learning how to reinstall Windows, I got into a habit — any time the computer felt “slow”, virus or not, I’d just nuke it and start over.

Back then I was very proud of having burned a so-called “All-You-Can-Eat” disc packed with cracked software. Pop it in, and it was like a buffet — pick whatever you want to install. Looking back, that’s so cringy, and I treated that disc like a treasure.

Note
“All-You-Can-Eat” disc is old-school slang from the CD-burning era. Those discs were full of pirated commercial software like Adobe Photoshop or Microsoft Office. Another piece of period slang for the same idea was “instant noodles”.

Honestly, every time the computer had an issue I’d reinstall, which got old fast — software had to be reinstalled one by one, all my settings and records were gone. So I eventually gave that up and looked for other ways to make the computer faster.

“Is there a way to make the computer faster? Let me Google it” — that’s what I thought at the time.

I opened the browser, typed “ways to make your computer faster”, and got a flood of articles, all teaching me various speed-up tricks like:

  • 5 minutes to a faster computer
  • A few steps to a faster computer
  • What to do when your computer slows down
  • Secrets to a faster computer

Skimming through them, I noticed they basically all said the same things:

  1. Disable startup programs
  2. Clean Temp files
  3. Defragment the disk

The fourth option was always “upgrade your hardware” — which was wishful thinking for me at the time. Where would a student get the money? I had to think hard before ordering instant noodles at an internet café, let alone replacing hardware.

One specific time I trusted an article that said:

“Try right-clicking to see folder sizes, then delete the big folders to speed up your computer!”

So I went ahead and deleted C:\Windows\System32. You read that right. And my deletion habit at the time was Shift + Delete, which means it skipped the Recycle Bin entirely.

Note
In Windows, pressing Delete is like throwing something in the “trash can” — you can still fish it out later. Shift + Delete is more like the “incinerator” — it’s gone, and there’s no undo.

I definitely yelled something at the screen, then quietly proceeded to reinstall Windows yet again…

Disabling startup programs

Most software adds itself to startup when it gets installed. For example:

  • HiCOS (smart card management tool)
  • Line (chat app)
  • Microsoft Office
  • …and so on

The more software you install, the more startup items pile up, and the slower your boot time becomes.

On Windows, you used to disable startup items with msconfig, but starting from Windows 10, that’s been moved to Task Manager. Open Task Manager, switch to the “Startup” tab, and you’ll see every program that runs at startup.

Look at the “Status” and “Startup impact” columns:

A word of caution though — don’t just disable startup programs randomly. Confirm whether each one actually needs to run at boot. If it doesn’t, then disable. Otherwise you’ll hit unexpected issues.

If you’re not sure, Google what the program is first, then decide.

Cleaning Temp files

Temp files are temporary files generated by Windows as it runs. Over time they accumulate and slow the system down by taking up resources.

When are these temp files generated? Some examples:

When you use a browser, it generates all kinds of cached files — images, page contents, music. You don’t need to re-download stuff to play or browse it again because the browser cached it locally.

Another example: Office (Word, Excel, etc.) also generates temp files. You’ve probably had Office crash before but still recover what you were editing — that’s because Office auto-saves temp files while you work, into the Temp folder. They get deleted when you close the app normally.

Even though many of these files end in .tmp, a lot of them are actually complete files (.docx, .xlsx) with the .tmp extension — you can open them directly.

Windows has two kinds of Temp folders:

  • User Temp: C:\Users\<username>\AppData\Local\Temp
  • System Temp: C:\Windows\Temp

Files inside temp folders are usually safe to delete, but some may be actively in use by the system or running programs. So to be safe, reboot first to make sure nothing’s holding them, then clean up.

If you’re lazy like me and know a bit of scripting, you can just write a simple .bat file to clean these up.

Note
.bat is the Windows batch file extension. It lets you run a sequence of commands at once, kind of like a Linux shell script.

I actually wrote a .bat years ago to clean both User and System Temp folders. Here it is for reference:

Quick line-by-line breakdown:

  • @echo off: Turns off command echoing.
  • echo Cleaning temp folders...: Prints status to the console.
  • echo User Temp folder:: Prints a header.
  • echo %temp%: Prints the current user’s Temp folder path. If your username is “Ray”, it’d print C:\Users\Ray\AppData\Local\Temp.
  • del /f /a /q /s "%temp%\*" >nul 2>&1:
    • del: Delete files (won’t delete folders).
    • /f: Force delete read-only files.
    • /a: Include hidden and system files.
    • /q: Quiet mode, no confirmation prompts.
    • /s: Recursively delete subdirectories.
    • "%temp%\*": All files in the Temp folder.
    • >nul 2>&1: Suppress all output and errors.
  • echo System Temp folder:: Header.
  • del /f /a /q /s "C:\Windows\Temp\*" >nul 2>&1: Same as above, for system Temp.
  • echo Cleanup complete!: Prints confirmation.
  • pause: Waits for a keypress before closing.

A quick reference table for del flags:

Flag Description
del Delete files (won’t delete folders by default)
/f Force delete read-only files
/a Include hidden and system files
/q Quiet mode, no prompts
/s Recursively delete subdirectories
"%temp%\*" All files in the Temp folder
>nul 2>&1 Suppress all output and errors

Take this .bat as reference only. If you don’t have a programming background, just use Windows’ built-in Disk Cleanup — it’s safer. The .bat does file deletion, which is inherently risky — a wrong path can break your system!

How do you actually use the .bat? Pretty straightforward — paste the code into a new text file, change its extension to .bat (e.g., clear_temp.bat), and double-click it.

Note
When using a .bat, right-click and select “Run as administrator” to make sure you have enough permission to delete the files.

Defragmenting the disk

Disk defrag isn’t a direct “make computer faster” tool — it optimizes how files are physically stored on disk so the system reads them more efficiently, which improves overall performance.

Why do we need this?

Earlier in this series I covered the basics of HDDs: when the system reads a file, the read head moves to where the file lives. If the file is large enough, it gets split across multiple regions of the disk, forcing the head to jump around to read all the pieces — which slows things down. That’s “file fragmentation”.

Note
Each disk sector has a fixed size. When a file exceeds a sector’s size, it gets split into multiple chunks stored across the disk in different locations. That’s the source of “file fragmentation”.

Example: a 4GB file on a disk where each sector is 1GB gets split into four chunks, spread across the disk. When the head reads the file, it has to keep moving around, slowing things down.

Defragmentation reorganizes the file fragments so they sit contiguously on the disk. Now the head doesn’t have to keep moving, and read speed improves.

Heads up: defrag is only useful for HDDs. SSDs basically don’t need it, because:

  • Fast access: SSDs are way faster than HDDs to begin with, fragmentation barely matters.
  • Random access: SSDs don’t use a head — they don’t move to read.
  • Limited write cycles: SSDs have a finite number of write operations, so repeatedly defragging actually shortens their lifespan.

So if you have an SSD, skip defrag. It’s an HDD-specific tool.

Wrap-up

The best way to make your computer faster is just to buy a new one.
