My first C language experience

Growing up, I’d picked up plenty of computer-related knowledge, but never formally learned a programming language.

Because I was super into Ragnarok Online, I ended up researching how to set up private servers, and along the way touched some simple scripting languages.

Note
Scripting language — a programming language designed for “quick write and run”. In short, you write it and it runs immediately, no compilation steps like C or Java.

This counted as my real entry into the programming world. Surprisingly, this experience became an important foundation for my later programming.

Note
Private server — a player-hosted game server where the host sets the rules. Not officially licensed, often has copyright issues or legal risks.

The scripting language I touched wasn’t a formal programming language, but it included basic programming concepts — variables, conditionals, loops, etc.

  1. Save the code as a .txt file.
  2. Drop it into the server’s npc folder.
  3. Restart the server — it’s live!

eAthena script isn’t this post’s focus. Just know I first touched “programming language” through it.

Later, during school, the teacher announced next semester would teach us a programming language — C.

Looking back, this was the textbook “From Intro to Quit”. That’s when I deeply realized: programming is harder than raid dungeons.

“Today we’re learning C!” the teacher wrote a big C on the board.

“C is basically the foundation of all programming languages. Many languages derive from C.” she added.

“Oh! So if I learn C, almost everything else becomes accessible?” I was super hyped — thinking that with a solid formal foundation, scripting later would be easier.

But reality slapped me hard.

My memory of C stops at this single line: printf("Hello, World!\n");

C was too hard for me. Once we hit pointers, I was done. After struggling a bit, I gave up and went back to gaming.

I didn’t learn pointers, but I leveled up my game character.

The grandfather of all languages: C

Why do many courses pick C as the intro language?

  1. Foundation of computing: most OSes (Unix, Linux, Windows), embedded systems, even many compilers are written in C. Learning C helps you understand how computers work.
  2. Language features: C forces you to learn memory management (pointers, dynamic allocation) and precise data types — building intuition for low-level concepts like Stack, Heap, memory leaks.
  3. Grandfather of all languages: many languages — JavaScript, Java, C#, Go, etc. — are influenced by C, basically evolved from it.

That’s why people say:

“If you know C, every other language is trivial!”

Because you’ve internalized the most foundational syntax patterns and logic, you ramp up on other languages faster.

E.g., JavaScript’s for loop and C’s for loop are nearly identical:

This structure makes JavaScript’s loops familiar — you’ve already done loops in C, so ramp-up is fast.

Also, as mentioned earlier, C forces you to understand memory management. In JavaScript, you don’t have to worry about this — JS has built-in Garbage Collection to auto-release unused memory, letting you focus on logic.

But this doesn’t mean you can completely ignore memory issues — bad logic can still cause memory leaks. With a C foundation, you’d appreciate memory management more.

Note

  1. Memory leak: program doesn’t release unused memory during execution. Over time it exhausts system resources, hurting performance or crashing the program.
  2. Garbage Collection: the language auto-manages memory allocation and release, lowering memory leak risk.

Below I’ll briefly cover some basic C syntax for initial concepts.

Just emphasizing: reading this post won’t teach you C — there’s a lot more to it. This is barely the surface.

Hello, World!

The classic intro example for all programming languages:

Note
“Hello, World” is the classic programming intro — demonstrates the language’s basic syntax and output. Lets beginners quickly experience writing and running the simplest program.

Don’t worry, line-by-line:

  • #include: tells the computer “add this tool”. In C, used to include libraries — letting us use pre-written functionality.
  • <stdio.h>: C’s Standard Input Output Library — like a toolbox of common functions, e.g., printf.

So this line means: “Hey computer! I want to use tools from the standard I/O library!”

Continuing:

  • int: C data type for “integer”. Here, the return type of main. Data type details come later.
  • main: the “Main Function” in C. Every C program must have one — it’s the entry point.
  • void: main doesn’t accept parameters.
  • {}: C block delimiters wrapping function content.

So this line means: “Hey computer! Here’s my main function. When you run the program, start here! It’s called main, accepts no parameters, returns an integer.”

Now the code body:

  • printf("Hello, world!\n");: displays “Hello, world!” on screen. \n is the newline character.
  • return 0;: main returns 0 — program ended normally. C convention: non-zero (like return 1;) indicates abnormal exit.

So this line means: “Hey computer! Output ‘Hello, world!’ on screen, then a newline! Then tell me the program succeeded and return 0!”

That’s the most basic C intro. Pretty simple, right?! (No.)

Data types

We mentioned int earlier — also called “data type”. This concept exists in nearly all programming languages.

Why do we need data types? A few reasons:

  1. Memory management: different types occupy different space, helping efficient memory use. E.g., int usually takes 4 bytes, char takes 1 byte.
  2. Type safety: types catch errors and reduce odd bugs or data loss. Note: different types can have implicit conversion, like auto-converting float to int — can lose data or produce odd results. Common practice: compile with warning options (-Wall -Wextra) or treat warnings as errors (-Werror).
  3. Code readability: explicit types make code clearer and easier to maintain.

Note
Implicit Conversion: when different types do arithmetic or assignment, the compiler auto-converts one to the other. No explicit conversion needed.

Common C data types fall into three categories:

  1. Integer: for integer values — int, short, long, etc.
  2. Floating point: for decimal values — float, double, etc.
  3. Character: for single characters — char.
