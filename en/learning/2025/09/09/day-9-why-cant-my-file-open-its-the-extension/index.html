Why can't my file open? Turns out it's the extension

"Have you used 'Instant Messenger' before? Not Line / WhatsApp / Messenger — I mean the older instant messengers like Yahoo! Messenger and MSN — the ones that ruled the world back in the day."

Instant Messenger (source: web)

I forget which year Yahoo! Messenger and MSN started getting popular, but basically if you had a Yahoo or MSN account, you could chat with friends worldwide in real time.

One day, I got a private message from a classmate on Messenger...

"Ding-dong~~~" — that classic IM sound.

"Hey! Ray, are you free? I want to ask you to fix my homework." I was at an internet café gaming hard, and his Messenger window suddenly popped up, instantly switching my game to the desktop.

"Argh!" I shouted — the game was peak action (shooter mid-match).

Imagine you're about to land the kill shot, or playing a LoL team fight, and suddenly your screen flips to the desktop — wouldn't you rage?

"I'll look in a sec! I'm playing CS!" I responded, irritated.

"Stop CSing! Help me with my homework first!" classmate's response was instant.

(Ding-dong! Ding-dong! Ding-dong!)

Honestly, IM back then was super annoying. Every message played a ding-dong sound, and the chat window popped up over your screen. The only way to avoid being interrupted was to log off entirely.

"Fine! What needs fixing?!" If I hadn't just finished the match, I'd have cursed at him.

"Yesterday in computer class, the teacher said we need to change line spacing in one Word paragraph, but my computer can't do it. Can you fix it?" he said.

"OK~ change to xx lines, right?" Line spacing takes no time, so I agreed.

"Yes!" he replied speed-of-light and sent me the file.

After receiving the file, I realized it wouldn't open. Every click brought up this:

Choose what to open with (source: web)

I don't remember the file format anymore — only that the café's computers had restrictions and didn't even have Microsoft Word installed.

Where extensions come from

Now let's explain why we hit the won't-open problem — starting with "file extensions".

A file extension is the string after the file name that indicates the file type. It usually starts with a period (.) followed by one or more letters.

Examples:

  • example.txt: a text file, extension is .txt.
  • example.jpg: an image, extension is .jpg.
  • example.bat: a batch file, extension is .bat.
  • example.docx: a Word file, extension is .docx (older versions used .doc).
  • example.xlsx: an Excel file, extension is .xlsx.

In short, the extension tells the OS "what kind of file this is", so the OS knows which program to use to open it.

Typical extension rules:

  • Extensions are usually 1–4 letters, like .txt, .jpg, .exe.
  • Usually lowercase, but some OSes are case-sensitive, so sometimes you'll see uppercase like .JPG.
  • Starts with a period followed by letters, e.g., example.txt.
  • Without an extension, the OS treats the file as "unknown type" — usually can't open directly unless you know the format.

"But my computer doesn't show extensions!" you might say.

Most OSes hide extensions by default. To prevent users from accidentally deleting or changing the extension and breaking the file. To show extensions manually:

  • Windows: in File Explorer, click "View" menu and check "File name extensions".
  • Mac: in Finder, click the "Finder" menu, choose "Settings", then check "Show all filename extensions" under "Advanced".

Quick reference table:

TypeExamplesNotes
Documents.doc, .docx, .pdf, .txtWord, PDF, plain text
Images.jpg, .png, .gif, .svgVarious image formats
Audio.mp3, .wav, .flacMusic files — pop songs, recordings

But here's something important...

"The extension does NOT equal the file's content."

The extension just tells the OS what kind of file this is supposed to be — it doesn't represent the actual content.

For example, you can rename example.txt to example.jpg, then click it. The OS will try to open it with an image viewer. There'll be an error, but the file's content hasn't actually become an image.

So if someone tells you:

"Showing extensions improves security"

They're really saying...

"Showing extensions tells you what kind of file this is, so you can decide whether to open it"

Usually when we see .exe, .bat, etc., we should be extra careful, because they might contain "malicious code".

Note "Malicious code"? In short, code that harms your computer — viruses, trojans, ransomware, etc.

Earlier on, a common malicious trick was "double extension attacks", like example.jpg.exe. The real extension is .exe, but .jpg is dropped in the name to trick people into thinking it's an image. Clicking it actually runs an executable.

Quick side story: in early DOS times, file naming had strict limits — name max 8 letters, extension max 3 letters. This was called the 8.3 naming convention. Over time this restriction became history. Now file names can be up to 255 characters, and extensions can be up to 255 characters too — no more 8.3 lock-in.

Common misconception: that changing the extension changes the file format. Not true. Renaming only changes the name. The result is just that the OS uses the wrong program to open it — the content stays the same.

How to find the actual file format

Quick tips on finding the actual format.

Say you had example.jpg and a friend pranked you by renaming it to example. Some time later, you can't tell what format it was — only guessing.

On Mac, use the file command:

bash
file example

It'll show the format, like:

bash
example: JPEG image data, Exif standard: [TIFF image data, big-endian, direntries=11, manufacturer=Apple, model=iPhone 12 Pro Max, orientation=upper-left, xresolution=170, yresolution=178, resolutionunit=2, software=17.0.3, datetime=2024:01:15 17:09:51, hostcomputer=iPhone 12 Pro Max], baseline, precision 8, 4032x3024, components 3

The key part: JPEG image data — confirming it's a JPEG.

On Windows, you can open it with Notepad and check the content. Usually the file starts with specific characters, e.g.:

txt
<xmpGImg:width>256</xmpGImg:width>
<xmpGImg:height>248</xmpGImg:height>
<xmpGImg:format>JPEG</xmpGImg:format>

You can see it's a JPEG image. But this trick only works for very small files — large files can't be opened in Notepad easily.

So I recommend using Git Bash with:

bash
file example

What's Git Bash? If you have Git installed on your computer, Git Bash comes with it. It's a Linux-like CLI on Windows, letting you use many Linux commands directly.

If you don't know Git yet, no worries — we'll cover it in a later chapter.

Closing thoughts

Honestly, I used to do this trick a lot — rename example.rar to example, so the OS couldn't identify the file type.

When my dad clicked it, he'd get a "can't open this file" error and stop trying~
