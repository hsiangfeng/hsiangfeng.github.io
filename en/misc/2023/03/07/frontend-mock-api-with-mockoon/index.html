A Mock API Skill Every Frontend Dev Should Have: Try Mockoon

#Mock API #Mockoon #Frontend Development #Backend Development #API Development

Intro

For frontend devs — junior or senior — knowing how to use a Mock API is genuinely worth it. This post walks through Mockoon, my go-to Mock API tool.

Why learn Mock APIs

Mock API basically means simulated API. Why simulate? Let me set a scene with a realistic dev situation.

You’ve got a feature with 7 days to ship — frontend and backend both needed. The backend says they’ll have endpoints ready in 5–6 days. You estimate your work (UI + API integration) at 5–6 days too. Question — as the frontend dev, can you really wait until backend ships endpoints before you start integrating?

Of course not. Integrating always has its own friction — endpoints that don’t respond, missing fields, mismatched shapes — so you’re going to need to keep talking to the backend as you go. If you wait, the deadline’s blown. And we haven’t even reserved 1–2 days for testing yet.

This is where Mock API pays off. You ask the backend to ship the API spec first — URL, method, request params, response shape — and you turn that spec into a Mock API. Now once your UI is done, you start integrating immediately, while the backend keeps building the real endpoints. Both sides work in parallel, frontend isn’t blocked. It’s a pretty solid workflow.

What is Mockoon

Mock API is a concept; there are lots of tools to implement it — Postman, Mocky, MockServer, Mockoon, etc. This post focuses on Mockoon, which is what I use.

Head to the Mockoon website, grab the installer for your OS, and install it.

(Download page: https://mockoon.com/download/)

Click “Start using Mockoon” and you’re in.

Using Mockoon

Mockoon ships with a demo API, but let’s actually walk through building one. Say I’m the backend and I hand you this TodoList API spec:

Now into Mockoon.

Create a Mockoon environment

Open Mockoon and click “Create a new environment”.

Give it a name (e.g., Todolist-example), pick a path, and hit “Create”.

The new environment shows up in Mockoon’s sidebar.

Quick note — see that localhost:3001 at the bottom? That’s the URL the Mock API will be served on. You can change it later if you want, but leave it for now.

Export environment

If you want to share your environment with someone else, right-click and pick “Show in folder” — Mockoon stores environments as JSON files, just hand the JSON over.

Import environment

Whoever receives the JSON clicks the folder icon at the top (“Open an environment”) and drops in the file.

Create a Mock API

Click “TodoList-example” in the sidebar — you should see this:

Click the arrow at the top — three things you can create:

  • CRUD Route
  • HTTP Route
  • Folder

CRUD Route generates a full CRUD API for you. HTTP Route is a custom route you define yourself. Folder is for organizing — e.g., put all the TodoList routes under one folder.

We’re going with HTTP Route, so pick that. The right pane is where you fill in the API details.

Start the Mock API

Time to fire it up — the start button is here:

But before launching, two settings to tweak — otherwise your frontend will hit issues integrating.

Change the Port

Click “TodoList-example”, then the “Setting” button on the right.

If you don’t like port 3001, switch it to 3000. The new port takes effect as soon as you start.

Set up CORS

Now click the Headers tab at the top, then the “Add CORS header” button.

Mockoon auto-fills the right CORS headers.

Now you can start Mockoon and integrate against it from your frontend without browser blocks.

Faker data generation in Mockoon

Beyond simulating APIs, Mockoon can generate fake data too — built on [Faker.js], so check the Faker.js docs for the full set of helpers.

Quick rundown on the syntax:

  • {{#repeat 5}} repeat 5 times
  • {{guid}} random GUID
  • {{lorem 5}} 5 lorem ipsum words
  • {{boolean}} random true/false
  • {{/repeat}} end of repeat

These are Mockoon’s wrapped helpers. For the full list, check the Mockoon templating helpers docs.

Closing thoughts

Mockoon is a tool I reach for a lot. When the backend can’t ship endpoints in time, I just ask them to send the spec first, build the Mock API in Mockoon, then swap to real endpoints when backend’s done — clean switch. Mockoon starts and stops quickly, and the GUI is straightforward. Solid tool.
