Google

Intro

Google Forms and Google Calendar are two tools most of us use a lot, both for work and for life. This post is about using GAS (Google Apps Script) to integrate them — let's get more done with less manual copying.

The scenario

Quick context on what we're building.

Say you're an office admin. You hand out a Google Form for your manager to fill in, and then for every submission, you have to manually go to Google Calendar and create an event based on the form data.

This post solves that. The use case as a user story:

  • User submits data through a Google Form
  • Submitted data auto-writes into a Google Sheet
  • Every 10 minutes, the script reads new rows and creates corresponding Google Calendar events

OK, on to the build.

You'll need:

  • A Google account
  • A Google Form
  • A Google Calendar

GAS (Google Apps Script)

Google Apps Script

GAS stands for Google Apps Script. Basically a Google-hosted JavaScript runtime with first-party access to Google's APIs — so anything you'd want to automate across Google products, GAS is the obvious tool.

If you're already a frontend dev you'll feel right at home. If you're not from a coding background, I'll do my best with detailed screenshots and step-by-step instructions.

Create a GAS project

To create a GAS project, open the sheet you just linked to your form, click "Extensions" → "Apps Script":

You should land on the GAS editor's initial state:

Note If you don't open Apps Script via the sheet's "Extensions" menu, you'll run into a few issues later. I'll mention them as we go — but the recommendation is always to open GAS from the sheet.

Connect to the sheet

Now to read sheet data. Replace myFunction with this (comments included):

js
function getSheetData() {
  // Get the active sheet (the one your form is linked to)
  const spreadsheet = SpreadsheetApp.getActiveSheet();

  // Define the data range — B2 down through column F
  const getRange = spreadsheet.getRange('B2:F');

  // Convert to a 2D array
  const displayDataValues =  getRange.getDisplayValues();

  // Log it so it shows in the Execution log
  Logger.log(displayDataValues);

  // Return the data (will use this later)
  return displayDataValues
}

Tweak the function to format the data nicely — paste this in:

js
function getSheetData() {
  // Get the active sheet (linked to your form)
  const spreadsheet = SpreadsheetApp.getActiveSheet();
  // Range — B2 down through column F
  const getRange = spreadsheet.getRange('B2:F');
  // Convert to 2D array
  const displayDataValues =  getRange.getDisplayValues();

  // Buffer for processed data
  const data = [];

  // Loop through each row
  for (let i = 0; i < displayDataValues.length; i++) {
    // Check if the first column has a value (i.e., there's data)
    if (displayDataValues[i][0]) {
      // Shape each row into an object for easier downstream use
      data.push({
        email: displayDataValues[i][0],     // Submitter email
        name: displayDataValues[i][1],      // Submitter name
        date: displayDataValues[i][2],      // Booking date
        startTime: displayDataValues[i][3], // Start time
        endTime: displayDataValues[i][4],   // End time
      });
    }
  }
  // Log it
  Logger.log(data);
  // Return
  return data
}

What do 0, 1, 2, 3, 4 map to? They're the indices of columns B, C, D, E, F in the sheet — the range we set earlier. Adjust as needed to match your own sheet:

Build a pseudo-env-var file

I'm calling it "pseudo" environment variables because GAS doesn't actually have env vars — but the pattern of putting config in one place is the same idea.

Quick recap on env vars — they're a mechanism for using different config in different environments (e.g., different DB passwords for dev vs. prod) without touching code.

Earlier we copied a Google Calendar ID. To make it easier to maintain (change the ID in one place, not scattered everywhere), we'll put it in a config-style file.

Click the + next to "Files" and create a new script called Env:

Env file

Contents:

js
const env = {
  calendarId: 'da39a3ee5e6b4b0d3255bfef95601890afd80709@group.calendar.google.com', // paste your calendar ID
}

(Reminder — that calendar ID is just an example, not a real one.)

Done. Easy.

Connect to Google Calendar

Create another file called main — main logic lives here. Paste this:

js
function setCalendar() {
  // Read sheet data — array of objects
  const sheetData = getSheetData();

  // Get the Google Calendar by ID
  const calendar = CalendarApp.getCalendarById(env.calendarId);
}

The "needs authorization" prompt shows up again — click through it the same way (Review permissions → Advanced → Go to project (unsafe) → Allow). Only happens once.

Nothing actually does anything yet — we just got past the permission gate.

Advanced time parsing → calendar

This part's slightly tricky. Create another file called parseTime, paste this:

js
function parseTime(date, time) {
  const timeRegex = /([上下]午) (\d+):(\d+):(\d+)/;
  const match = time.match(timeRegex);

  if (!match) {
    throw new Error(`Invalid time format: ${time}`);
  }

  let hours = parseInt(match[2], 10);
  const minutes = parseInt(match[3], 10);
  const seconds = parseInt(match[4], 10);

  if (match[1] === '下午' && hours < 12) {
    hours += 12;
  }

  const parsedTime = new Date(date);
  parsedTime.setHours(hours, minutes, seconds);

  return parsedTime;
}

parseTime

This is a bit dense. All it's doing is parsing time strings stored in the sheet into a format Google Calendar can use.

Note The regex here matches Traditional Chinese AM/PM markers (上午/下午) because that's what Google Sheets renders when your locale is zh-TW. If your locale renders AM/PM in English, you'd want a regex like /(AM|PM)\s+(\d+):(\d+):(\d+)/i instead. The simpler approach in the next section sidesteps the locale issue entirely.

Back to main.gs — fill in the rest:

js
function setCalendar() {
  // Read sheet data
  const sheetData = getSheetData();

  // Get Google Calendar
  const calendar = CalendarApp.getCalendarById(env.calendarId);

  // Loop over each row
  sheetData.forEach((data) => {
    // Convert start/end times to a format Calendar accepts
    const startTime = parseTime(data.date, data.startTime);
    const endTime = parseTime(data.date, data.endTime);

    // Create the calendar event from the sheet row
    calendar.createEvent(data.name, startTime, endTime, {
      description: data.email, // put email in event description
    });
  });
}

Click "Run". Once it finishes you'll see your form data show up as calendar events:

Calendar events

Congrats — sheet data is now landing in your calendar.

Simpler time parsing → calendar

Honestly that's a lot. Here's an easier route that sidesteps parseTime by using a Sheets-side trick.

Open your sheet and add two new columns: Start Time and End Time:

Sheet tweak

In Start Time enter =D2+E2, in End Time enter =D2+F2, drag the formula down:

Time formula

This combines the date column with each time column into a proper datetime that Google Calendar accepts natively. Now you can ditch parseTime and update main.gs:

js
function setCalendar() {
  // Read sheet data
  const sheetData = getSheetData();

  // Get Google Calendar
  const calendar = CalendarApp.getCalendarById(env.calendarId);

  // Loop over each row
  sheetData.forEach((data) => {
    // Create the calendar event from the sheet row
    calendar.createEvent(data.name, data.startTime, data.endTime, {
      description: data.email,
    });
  });
}

Update getSheetData to grab the new columns and use getValues (raw values) instead of getDisplayValues (formatted strings):

js
function getSheetData() {
  // Get the active sheet (linked to your form)
  const spreadsheet = SpreadsheetApp.getActiveSheet();
  // Range — B2 down through H now that we added columns
  const getRange = spreadsheet.getRange('B2:H');
  // Raw values — gives us Date objects directly
  const displayDataValues =  getRange.getValues(); // <- swapped getDisplayValues for getValues

  // Buffer for processed data
  const data = [];

  // Loop through each row
  for (let i = 0; i < displayDataValues.length; i++) {
    // Check if the first column has a value
    if (displayDataValues[i][0]) {
      // Shape each row into an object
      data.push({
        email: displayDataValues[i][0],     // Submitter email
        name: displayDataValues[i][1],      // Submitter name
        date: displayDataValues[i][2],      // Booking date
        startTime: displayDataValues[i][5], // Start time (computed column)
        endTime: displayDataValues[i][6],   // End time (computed column)
      });
    }
  }
  // Log
  Logger.log(data);
  // Return
  return data
}

Note The difference between getValues and getDisplayValues — the former returns underlying raw values (e.g., Date objects), the latter returns the formatted strings as you see them in the sheet.

The duplicate event problem

You'll quickly notice — every time setCalendar runs, it creates events for every row in the sheet, even ones already added before. So we need a duplicate check.

Create a new file isEventExist:

js
function isEventExist(calendar, name, startTime, endTime) {
  const events = calendar.getEvents(startTime, endTime, {
    search: name,
  });

  return events.length > 0;
}

Update main.gs:

js
function setCalendar() {
  // Read sheet data
  const sheetData = getSheetData();

  // Get Google Calendar
  const calendar = CalendarApp.getCalendarById(env.calendarId);

  // Loop over each row
  sheetData.forEach((data) => {
    // Check if the event already exists
    const isExist = isEventExist(calendar, data.name, data.startTime, data.endTime);

    // Only create if it doesn't already exist
    if (!isExist) {
      calendar.createEvent(data.name, data.startTime, data.endTime, {
        description: data.email,
      });
    }
  });
}

No more duplicate events on re-runs.
