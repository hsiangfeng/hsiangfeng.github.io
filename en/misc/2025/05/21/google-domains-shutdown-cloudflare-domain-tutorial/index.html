Google Domains Shut Down — Buying a Domain on Cloudflare and Deploying a Site, Step by Step

#Domain #Google Domains #Cloudflare Domain

Intro

A while back I wrote a tutorial on buying a domain (using Google Domain) and pointing it at your blog. But on September 7, 2023, Google Domain announced it was shutting down domain registration, so I figured I’d write a Cloudflare Domain version of the same tutorial.

Cloudflare Domain

Most people have heard of Cloudflare — it’s one of the world’s largest CDN providers, and they offer a bunch of services, domain registration being one.

What’s the benefit of using Cloudflare for your domain? Mainly that you get a pile of their other services for free without needing to wire anything up — CDN, SSL, DNS, etc., all built in. You don’t have to go shopping for separate providers.

This post covers three things:

  • Buying a Cloudflare Domain
  • Deploying a static site to Pages
  • Migrating a Squarespace Domain (formerly Google Domain) over to Cloudflare Domain

Buying a Cloudflare Domain

Open the Cloudflare site and click “Sign Up” in the top right to create an account — I’ll skip the registration walkthrough. Once you’re in, find “Domain Registration → Register Domains” in the left sidebar.

In the search box, type the domain name you want and hit “Search”. I’ll use “israynotarray” as the example.

After the search, available domain names show up below. Pick the one you want and click “Confirm” to start the purchase.

For the demo I’ll pick “israynotarray.win“.

Note
Prices vary by TLD, and each TLD has a meaning behind it. For example: .com was originally for commercial/business use, but became a general-purpose TLD; .org for non-profits; .net for network providers; .info for information sites; .win for “victory/success” connotations. There are many more — feel free to look them up.

One thing to watch: domains are billed yearly, and many registrars run first-year discounts that bump back to regular pricing on renewal. In this example, year one is US$4.16 but starting year two it’s US$5.16. Make sure you know what you’ll be paying long term.

After confirming, you fill in some registration info.

Why all this info? Because it goes into the WHOIS database — the public registry for domain ownership, which shows the registrant’s contact info, registrar, etc.

Pay special attention to the auto-renewal setting. If you don’t want it auto-renewing, turn it off — otherwise it’ll renew automatically when the term expires.

When payment goes through, you see this:

If you’ve used other domain registrars before, you’ll notice Cloudflare doesn’t seem to offer a separate WHOIS privacy service — that’s because Cloudflare’s domain registration enables WHOIS privacy by default. If you want to verify, look at the “WHOIS information” section and you’ll see a bunch of “DATA REDACTED”.

If you really want to double-check, go to Domain Registration → find your domain → Manage → Settings — you’ll see WHOIS privacy status (on / off).

At this point you own your own domain. Now let’s actually use it.

Deploy a Pages static site

Now we’ll deploy a static site to Cloudflare Pages. In the sidebar, find Compute (Workers) → Workers & Pages.

On the page that opens, click “Pages”. You’ll see two deployment options:

  • Import an existing Git repository
  • Direct upload

Most engineers go with “Import an existing Git repository” — I’ll use that here.

I’ve prepared a GitHub project you can fork.

Click “Import existing Git repository”. You’ll be asked GitHub or GitLab — pick GitHub.

Click “Connect GitHub” and authorize Cloudflare to access your GitHub account. I chose “Only select repositories” so Cloudflare doesn’t get access to all my repos — pick what works for you.

After authorization, you’re back at the project selection screen. Pick the project you want to deploy.

Now you can configure things like the branch to deploy and environment variables. My example is a plain static HTML file so there’s nothing to configure — I’ll just hit “Save and Deploy”.

Deployment takes a moment.

When it’s done, you see this:

Click the URL at the top — that’s the URL Cloudflare Pages auto-generated. Click and you can see your deployed site.

Point your domain at Pages

You’ve deployed the project to Cloudflare Pages, but the URL is one Cloudflare auto-generated (https://cloudflare-doamin-example.pages.dev/). Now let’s point your own domain at it.

The screen will look a bit different now since we’ve already deployed a project.

Click on the project you just deployed. This is the project dashboard, but what we’re looking for is “Custom Domains”.

Click “Set up a Custom Domain”.

You’ll be prompted for the custom domain. We bought “israynotarray.win“ earlier, so enter that and submit.

It’ll confirm the DNS record — don’t change anything, just click “Activate Domain”.

The activation needs a moment for DNS to verify.

Note
Why does verification take time? Because DNS needs to validate identity, which takes a while. Documentation suggests 1–2 hours; in practice mine took 10–15 minutes.

When you see this screen, the domain is live:

Open israynotarray.win and you’ll see your deployed static site.

Bonus: migrating a Squarespace Domain to Cloudflare Domain

Since Squarespace acquired Google Domain, anyone who originally bought a domain through Google Domain got migrated to Squarespace. Here’s how to move it over to Cloudflare Domain.

Quick heads up: this section is a continuation of Buying a domain from scratch with Google Domain (article in Mandarin), i.e. for people who originally bought through Google Domain and are now on Squarespace — so some steps might look different from a fresh setup.

Open your Squarespace Domain account, log in, find “Domains” in the left sidebar — your domains will be listed. Pick the one you want to move.

Click in, find “Domain Lock” in the dashboard, and turn it off (you can’t transfer with the lock on).

Note
After turning off Domain Lock, you’ll need to wait a bit before transfer is possible. In my testing it took about 10–15 minutes for the lock to actually clear.

Now head back to Cloudflare Domain, log in, and find Domain Registration → Transfer Domains in the sidebar.

You should see the eligible domains.

Transferring from one registrar to another costs a small fee. Click “Confirm Domain” if you’re good with it. You’ll then be asked for the “Auth Code” to confirm you actually want to pay for the transfer.

Where do you get the Squarespace auth code? It’s in the dashboard.

Click “Request transfer code” — Squarespace emails you the code. Paste it into Cloudflare and the transfer is underway.
