No Postman? No Problem — Testing APIs with curl

#API #Postman #curl #Testing

Intro

Sometimes you can’t use Postman to test an API, and you can’t install other tools either — maybe you’re SSH’d into a server (servers usually have no GUI). That’s when curl is your friend.

What is curl?

curl is short for Client URL. It’s a network protocol tool that supports a huge range of protocols — FTP, FTPS, HTTP, HTTPS, and many more. Almost any network protocol you can think of, curl can test.

Note
Some people write curl, others write cURL. I’ll stick with curl throughout this post.

Wrapping up

Quick reference table for everything covered:

Flag What it does Example
-X Set HTTP method (POST, PUT, DELETE, etc.) curl -XPOST https://api.example.com/items
-H Add a header (one -H per header) curl -H "Authorization: Bearer YOUR_TOKEN"
-d Send body data (POST/PUT) curl -d '{"content":"Buy dinner"}' -H "Content-Type: application/json" https://api.example.com/todos
-v Show verbose connection details (request/response headers, etc.) curl -v https://example.com
-o Write response to a file curl -o result.html https://example.com
-A Set User-Agent (mimic a browser/device) curl -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64)" https://example.com
\ (backslash) Line continuation for readable multi-line curl curl -XPOST https://api.example.com \ -H "Authorization: Bearer ..." \ -d '{"key":"value"}'
