How to Get GitHub Pro for Free (via GitHub Student Developer Pack)

#GitHub #GitHub Pro

Intro

GitHub already gives away a lot for free, but Pro still has some real perks. This post is on getting Pro for free.

GitHub free vs. GitHub Pro

Quick recap on the differences.

Back before Microsoft acquired GitHub, the gap between free and Pro was bigger — only Pro could create private repos. Microsoft made private repos free for everyone after the acquisition. Reference (article in Mandarin).

Pro used to cost $7/month. After the acquisition, the price dropped to $4 — and the perk gap got smaller.

That said, free still has limits. Private repos on free are capped at 3 collaborators. If you have a fourth or fifth dev on a private repo, you need Pro. Branch protection rules are also Pro-only.

Getting GitHub Pro

Two main paths:

  • Pay (the joke option)
  • Get it free via student account

Sure, you could just pay (or have your company pay). But if you’re a broke developer who happens to have a school email, you can use GitHub’s Student Developer Pack.

The application process has changed since I first wrote this, so here’s the updated walkthrough.

Applying for the Student Developer Pack

Easy version — sign up for GitHub with a school email, then head to GitHub Student Developer Pack and click “Get your pack”.

But most devs are already using a personal GitHub account they’ve had for years — maybe with their blog tied to it — so switching accounts isn’t ideal. Also, adding the school email to your existing account boosts your credibility on GitHub (GitHub sees you as a student, which factors in elsewhere).

Go to GitHub Settings → Emails:

Add your school email under “Add email address”. (For Taiwan schools that’s an .edu.tw domain — for other countries it’s typically .edu or country-specific.) Verify via the link sent to your school inbox.

Once verified, head to GitHub Student Developer Pack and click “Get your pack”:

What’s the Student Developer Pack? Special offers GitHub bundles for students — many tools and services are free, some for a full year:

GitHub Pro is one of them:

On the application page, I’d recommend writing meaningful descriptions for “What is the name of your school?” and “How do you plan to use GitHub?”

Click “Submit your information”. One thing to note — when picking the verification email, pick the school email (e.g., .edu.tw), not your personal one, or it’ll fail.

You’ll also need to upload student proof — student ID, enrollment letter, etc. (Screenshot omitted.)

One important thing — if your student ID isn’t in English, upload a translated version along with the original. Otherwise it gets rejected.

After submission, GitHub takes about 1–2 business days to approve.

Once approved, your account picks up a “Pro” badge:

After upgrading to Pro, the private-repo collaborator cap is gone, branch protection rules are available, and Actions minutes increase to 3,000.

You can verify via Personal billing:

Per GitHub’s terms — as long as you’re still a student (still have access to your school email), you keep Pro plus all other Student Developer Pack benefits.
