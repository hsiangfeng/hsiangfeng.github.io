Discord Role Bot Tutorial: Auto-Assign Roles via Invite Links (Invite-Role Bot)

#Discord

Intro

The company has been gradually moving onto Discord, and there are a few features Discord doesn’t have natively. So here’s a quick post about one of them.

Creating an invite link

Note
The author of this bot doesn’t seem very responsive these days. If you’d rather use something more actively maintained, betterinvites works the same way.

Normally, when you want to invite a friend to your Discord server, you go through “Create Invite” (next to the gear icon on a channel):

You’ll get an invite link:

If you don’t tweak the settings, the link will expire after 7 days. You can change that by clicking “Edit invite link” below:

Heads up: invite links are tied to the channel you generated them from. If you generate one in a private channel, anyone joining via that link enters the private channel — which usually causes weird issues (like the user seeing a blank screen until they switch channels or restart the Discord client). The safer bet is to always generate invites from a public channel.

Inviting Invite-Role Bot to your server

The typical scenario: you want members joining via a specific link to automatically get role X. Discord doesn’t ship with this, so you need a third-party bot. The one we’ll use here is Invite-Role Bot.

Wiring the invite link to a role

This is the main event. Quick rundown of Invite-Role Bot’s commands:

  • ~help: Show all commands.
  • ~prefix: Change the command prefix (default is ~, you can change to /). Needs admin permission.
  • ~invrole list: Show configured invite-link → role mappings. Often contains sensitive info, so use in a private channel. Needs admin permission.
  • ~invrole connect [URL] [@ROLE]: Map an invite link to a role. Anyone joining via that link gets the role automatically.
  • ~invrole disconnect [URL]: Remove a mapping.
  • ~invite: Get the bot invite link to share with others.

The commands you’ll actually use are ~invrole list, ~invrole connect [URL] [@ROLE], and ~invrole disconnect [URL].

Now share that invite link. Anyone joining via it will get your configured role automatically.

Troubleshooting

If ~invrole connect [URL] [@ROLE] keeps failing, the issue is probably that the bot’s own role is too low in the hierarchy. Just drag it up.

A bot can only assign roles that are below its own role in the hierarchy. Pull the bot’s role up and it’ll work.
