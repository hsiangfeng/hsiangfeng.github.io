Intro

One day I went to install something and got "zsh: command not found: brew install". So I'm writing this down.

What I saw

One day I typed brew install flyctl and got:

bash
zsh: command not found: brew install flyctl

Then I tried brew list, same thing:

bash
zsh: command not found: brew list

But I hadn't changed anything recently. And weirder still — when I typed just brew, I got the help output:

brew help works

The fix

First thought — maybe my PATH got broken. The line in question:

bash
export PATH="/usr/local/bin:$PATH"

Gut call was reinstalling Homebrew:

bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

(Homebrew's install is painfully slow, by the way.)

After install, brew install flyctl still failed. So I actually read the installer's output this time:

Installer hint

bash
- Run these three commands in your terminal to add Homebrew to your PATH:
    echo '# Set PATH, MANPATH, etc., for Homebrew.' >> /Users/rayxu/.zprofile
    echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/rayxu/.zprofile
    eval "$(/opt/homebrew/bin/brew shellenv)"

These three lines just set up Homebrew's PATH — they boil down to:

bash
# Set PATH, MANPATH, etc., for Homebrew.
eval "$(/opt/homebrew/bin/brew shellenv)"

The installer defaults to .zprofile, but I use zsh, so I swapped it to .zshrc:

bash
echo '# Set PATH, MANPATH, etc., for Homebrew.' >> /Users/rayxu/.zshrc
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/rayxu/.zshrc
eval "$(/opt/homebrew/bin/brew shellenv)"

Reopened the terminal, ran brew install flyctl — still no luck. Poking around the docs, I noticed an example brew install wget, so I tried it. That one worked. And after that, brew install flyctl worked too.

So why? It turns out brew install pulls in a few basic dependencies behind the scenes:

  • gettext
  • libunistring
  • libidn2
  • ca-certificates
  • openssl@1.1

For some reason my wget had gone missing, which was blocking other installs that depended on it. Installing wget pulled those basics back in, and everything else started working.
