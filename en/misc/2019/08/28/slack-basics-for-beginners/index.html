Slack Basics for Beginners

#Slack #Software

Intro

A walkthrough of basic Slack — slightly long, but Slack is one of the most common team communication tools, so worth knowing.

Slack

Slack is a team communication tool — similar to LINE or WhatsApp in concept, but built for team workflows. You can delete sent messages, edit sent messages, organize conversations into topic channels and threads.

It supports team-wide workspaces with sub-channels and private channels. Slack is free for most personal/small-team use, and many companies adopt it as their internal communication tool.

Joining Slack

You can create your own workspace or join an existing one. Here I’ll use an existing workspace as an example.

Typically someone shares an invite link. Click it and you get a screen asking for your email — to check if you have a Slack account:

If you already have a Slack account, you’ll join directly. First-time users get a verification email:

Click through to verify:

Then Slack asks for your name and password — these will be your default identity for any future Slack workspace (your email becomes your login):

After that you’re in:

Installing the desktop app

I’d recommend installing the Slack desktop client. Without it, web-only Slack means you’ll miss messages frequently.

Slack’s own banner at the top often says: We strongly recommend enabling desktop notifications if you'll be using Slack on this computer. Enable them.

Better — install the actual desktop app. Download page: Download Slack.

Direct links:

  • Windows
  • Mac
  • Linux

After installing, opening the app gives you a blank login screen:

Click “Sign In” and a browser window opens asking for the workspace URL:

That URL slug is the workspace identifier (from when you joined — e.g., playSlackYa):

Type it in:

Then you’ll be redirected to log in:

After login you might end up in web Slack (not sure why Slack does this). Back to the desktop app, click “Sign In” again — now you’ll see the workspace in the top-left. Click into it and you’re done:

A bit clunky the first time, but you only do it once per workspace.

To add another workspace later, click the menu in the top-left → Window → “Sign in to Another Workspace” and repeat:

UI tour

Quick tour of the basic UI.

Channels

Left sidebar shows channels. Two types:

  • Public channels — anyone can join
  • Private channels — invite-only

Click any channel to switch and start chatting (or just lurking):

To join an existing public channel, click “Add a channel”. You can also create your own:

Direct Messages

1-on-1 DMs. If the person isn’t already in your DM list, click the + to add them:

Apps

Slack supports many integrations to speed up workflows. I haven’t used much, so skipping.

Threads

Threads are one of my favorite features — focus a follow-up discussion on a specific message instead of derailing the channel:

Editing messages

Code

Slack supports both inline code and code blocks:

  • Inline code — wrap text in single backticks
  • Code block — wrap text in triple backticks

Markdown

Slack supports basic Markdown — once you know Markdown, you’ve got Slack:

Other

Slack has lots more — drag and drop files, Ctrl + V to paste screenshots, etc. Too many to cover.

To change Slack’s UI language, see How to Change Slack’s UI Language.

Wrapping up

That’s most of the day-to-day Slack basics. Not hard, and honestly Slack makes code-related chat way better than LINE — in LINE you’ll watch your code message get buried by chit-chat within seconds. Slack’s threads avoid that.
