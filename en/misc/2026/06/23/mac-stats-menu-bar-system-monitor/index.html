Stats — The Free Open-Source macOS Menu Bar System Monitor I Use Every Day (CPU, GPU, Temperature, Network)

#Mac #macOS #Stats #Menu Bar #Productivity #System Monitor #Open Source

Intro

This post is about my favourite tool on macOS — Stats. I recently jumped from an M1 to an M5, and even though Apple’s migration tool moved everything over cleanly, Stats somehow refused to migrate. So it ended up being basically the first app I reinstalled on the new machine, which is a pretty good summary of how much I rely on it. This post covers what it does, how to install it, and a few of my own setup habits.

If you’ve read my earlier posts — How Should You Install Software on a Mac? Homebrew vs Official Download vs curl or Raycast Extensions I Actually Use — this one belongs in the same series. All of them are tools I’ve kept on every Mac for years.

So what is Stats?

Quick intro first.

Stats is a system monitor for macOS. Compared to Activity Monitor, it puts everything you’d want to glance at right where you can see it — I check Mac temperatures, network throughput, and memory use throughout the day without ever opening anything.

It’s an MIT-licensed open-source project, so it’s completely free, no feature gates, no ads, no upgrade nags. And you don’t really need to worry about it doing anything sketchy in the background — if it did, you wouldn’t be the only one to find out, since a lot of people use it.

Here’s the quick rundown of what it can monitor:

Module What it shows
CPU Processor usage, per-core state, frequency
GPU GPU usage (Apple Silicon and external GPU both supported)
Memory RAM usage, swap state
Disk Per-disk read/write speed, remaining space
Sensors Sensor readings (CPU temp, GPU temp, voltage, power, etc.)
Network Live up/down throughput, public IP, Wi-Fi signal strength
Battery Battery level, health, charge/discharge state
Bluetooth Battery level of connected Bluetooth devices
Clock Multiple timezone clocks
Remote Remote monitoring of other Macs

Pretty cool, right? Stats also has a Remote module. If you’ve got a Mac mini at home running OpenClaw or Hermes, this is genuinely useful — you can keep an eye on the remote machine straight from the menu bar instead of opening a remote desktop session.

I haven’t actually used Remote myself yet, but reading through the Stats site writeup it looks pretty nice.

The modules I actually use

Going through each module one by one.

CPU module

CPU is one I leave on permanently. Being able to see live CPU usage means I can answer “what is this thing doing right now?” almost instantly — especially when the Mac suddenly feels sluggish. The first thing I want to know is whether the CPU is pinned.

GPU module

I turn this one on when I’m running an LLM locally. GPU usage spikes hard in that scenario, so a quick glance tells me whether the model is actually doing work or stuck somewhere mid-step.

Memory module

I watch memory closely too — once you run out, the system starts swapping to disk and everything slows to a crawl.

Note
One thing to mention — macOS handles memory differently from Windows. It’ll happily use unused RAM as cache, so you might see Memory used looking high while the system actually has plenty of free room.

Disk module

I rarely look at this one, so I usually keep it off — unless I’m doing something disk-heavy and want to see throughput.

Sensors module

The Sensors module is genuinely important to me — I’m mostly on an Air, and if I don’t watch temperatures the Mac will hit thermal throttling and everything stutters.

Stats reads temperatures straight into the menu bar:

  • Per-core CPU temperatures
  • GPU temperature
  • SSD temperature
  • Battery temperature
  • Various motherboard zones
  • Overall power draw

One thing worth noting — for this module the temperatures basically only show up in the menu bar.

Network module

Another one I leave on permanently. Whether the network’s working — and how fast it’s running — is one of the things I most want at a glance.

Note
Stats fetches your public IP by calling an external API. If that bothers you privacy-wise, you can disable it in the Network module settings.

Battery module

Hmm, this one’s pretty useful if you’re on an Intel-era Mac — Intel Macs have noticeably less stable battery behaviour and health compared to Apple Silicon, so I’d recommend keeping this one on if that’s you.

Bluetooth module

I barely touch this one. If your Bluetooth feels flaky, it’s worth opening up.

Clock module

I don’t really use this one either, but the convenience is that you can pin multiple timezones — handy if you’re collaborating across time zones (probably).

At most I pin a US West Coast time, mainly so I can check maintenance windows on overseas services.

Extra tip — desktop widgets

The menu bar gets crowded fast, so I move a few of the things I care about onto the desktop instead. Right-click the desktop → Edit Widgets:

Then find the widget you want and drag it onto the desktop:

This lets you keep more detail on the desktop itself — for me, it’s CPU, GPU, and memory usage.

Common questions

A couple of issues that come up.

Desktop widgets show “No data available”?

You need to enable the “macOS widgets” option for them to work. Stats’s desktop widgets are built on top of macOS’s native Widget framework, so if that option’s off, you’ll just see “No data available” forever.

Heads-up though — enabling this also makes Stats hammer disk writes pretty constantly. The maintainer’s tracking the issue: Widgets do not update or high disk writes.

Desktop widgets stuck on “Enable in Settings”?

This one seems to happen regularly with no obvious workaround — I’m just waiting on an upstream fix.

Wrap-up

Quick wrap-up.

The reasons I personally recommend Stats:

  • Open source, free, MIT-licensed
  • Modular — turn on what you want, leave the rest off
  • Clean interface, with localisation
  • Surfaces things macOS itself isn’t very forthcoming about, like CPU temperature

Hope this is useful.
