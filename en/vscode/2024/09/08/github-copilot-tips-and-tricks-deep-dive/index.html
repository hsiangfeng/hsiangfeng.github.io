GitHub Copilot

Intro

I've been using GitHub Copilot for a while. This post collects tips and tricks I've picked up so you can ship faster. Let's get into it.

What is GitHub Copilot?

Quick clarification first — GitHub Copilot and Copilot are two different things. "Copilot" refers to Microsoft Copilot, not GitHub Copilot. This post is about GitHub Copilot.

Why call this out? Because Microsoft Copilot used to be called "Bing AI", and Microsoft later renamed it to Copilot, which collides with GitHub Copilot and confuses people. So I'm being explicit upfront.

Anywhere this post says "Copilot", I mean GitHub Copilot, not Microsoft Copilot.

What is Copilot, exactly? It's an AI tool built by GitHub and OpenAI, trained on open-source code hosted on GitHub. It speeds up code writing.

Software moves fast, so to be specific: this post is written against GitHub Copilot 1.229.0, September 2024. If features change later, check the official docs.

Note Copilot's positioning is "pair programming". Pair programming means two people coding together — one primary, one helper. Copilot is the helper, just AI-powered instead of human. With Copilot's help, you finish code faster.

Installing GitHub Copilot

Installation is straightforward — search "GitHub Copilot" in the VSCode extension marketplace and install.

Note Copilot supports more than VSCode — it also works in Microsoft Visual Studio, Vim, Cursor, JetBrains, etc. But I'll focus on VSCode here.

You'll find two extensions: GitHub Copilot and GitHub Copilot Chat.

Either way, both get installed together. One is the core Copilot extension, the other is the chat panel.

GitHub Copilot extensions

After installing, you'll see the Copilot icon in the VSCode bottom-right and the Chat panel on the left.

Copilot installed

Note GitHub Copilot is paid — roughly USD 100/year. But if you're a student, you can get it free through GitHub Education (article in Mandarin).

Using GitHub Copilot

The basic flow: as you write code, Copilot suggests completions. Honestly, even while writing this article, Copilot was predicting what I was about to type. Hit Tab to accept.

Copilot predicting text

That's the basic functionality. Let's look at the other tricks.

Note GitHub Copilot uses your open workspace as training context, so completion quality scales with the quality of code in your workspace.

Generate code from comments

I already showed Copilot auto-completing text, but it can also generate full code from comments.

Just write a comment and let Copilot do the rest:

Comment to code

In the example above I used JSDoc (article in Mandarin) for the comment, and Copilot auto-generated the implementation. Plain comments work too.

Comment to code

If you don't like the result, hover over the suggestion and cycle through alternatives.

Cycling suggestions

If even the alternatives aren't great, click "..." and find Open Completions Panel for more candidates.

Open Completions Panel

Completions Panel

When one looks good, click "Accept Suggestion".

Trigger Copilot to write code via command

If you don't want comments hanging around in your file, you can invoke Copilot via command instead. Press Cmd + I (Mac) or Ctrl + I (Windows) to bring up the Copilot prompt, and type what you want.

Copilot inline command

For example, ask Copilot to write an AJAX call:

md
Please write a fetch call to the TodoList API at https://jsonplaceholder.typicode.com/todos/1, log the result via console.log, and handle errors — if the AJAX fails, alert "AJAX failed".

Inline command result

If you're not happy, keep iterating until you are.

Iterating

You can also click the dropdown to switch to Chat mode — Copilot then becomes a conversation, and you can keep going until you hit the checkmark.

Chat 1

Chat 2

Attach files as Copilot context

You can also use the inline command with file attachments — point Copilot at a Markdown file and let it generate everything based on what's in there.

Say you have a Prompt.md containing:

md
# Code prompt

Please use fetch to call the TodoList API at `https://jsonplaceholder.typicode.com/todos/1`, log the result via console.log, and handle errors — if the AJAX fails, alert "AJAX failed".

Open the file you want code in, press Cmd + I (Mac) or Ctrl + I (Windows), click the attachment icon, attach Prompt.md, and tell Copilot what to do. It'll read Prompt.md and generate code based on it.

Attaching file

Got buggy code? Select it, invoke Copilot, and ask for a fix.

Right-click context menu

Once you've selected code, right-click to find the "Copilot" option and pick what you want.

Right-click menu

For example, "Generate Tests" — Copilot writes the tests in a new file in one shot.

Generate tests

You can also have it generate a README.md for the project, using the file-attachment feature.

Copilot Chat @ commands

Copilot Chat has built-in commands that start with @. These are Chat-mode-only.

Note These give Copilot more precise context than free-form chat.

@terminal — ask how to do terminal stuff

Ask Copilot how to do something in the terminal. E.g. "How do I start my project?"

@terminal

Or "How do I create a file from the terminal?"

@terminal

@terminal gives much more accurate, terminal-specific answers than general chat.

@vscode — ask about VSCode

For VSCode-specific questions. E.g. "How do I make CRLF default to LF?"

@vscode

@vscode lets you quickly solve VSCode-related questions without leaving the editor.

@workspace — ask about your workspace

One of Chat's most powerful commands. Ask about your workspace specifically. E.g. "Can you generate tests for expiresInToMilliseconds in utils/time.js?"

@workspace is usually slower because it scans your workspace before generating, and it'll tell you which files it referenced.

@workspace

Note Sometimes @workspace can't reference the entire workspace because of Token limits (the Token budget for what's sent to Copilot). If your workspace is huge, split files into smaller ones and ask again.

Wrapping up

GitHub Copilot keeps getting better and more powerful. Once you pick up the tricks, you can ship features way faster. But one important reminder — AI is here to accelerate you, not replace you. Always review what it produces yourself, both for code quality and security.
