A (Slightly Long) Casual Look at JavaScript Functions

#Vue #JavaScript #ES6 #Function #Named Function #Anonymous Function #Arrow Function #IIFE

Intro

I did a live intro session over at Hexschool, and honestly the whole thing had me nervous out of my mind. I know I’m pretty forgetful by nature, so skipping the write-up afterward felt kind of wrong. That’s how this slightly-long casual look at JavaScript functions got born.

As a dev, you absolutely use functions all the time. But because we use them so constantly, we kind of stop thinking about them. So this post is a (slightly long) casual look at functions — partly to jog my own memory and clear up some concepts along the way.

Note
This post doesn’t get into function statements vs. function expressions — it’s just about function in general.

  2. Anonymous functions

Function

I’ve talked through a lot of the mechanics, but I still haven’t said what a function actually is. Let me start by quoting how MDN puts it:

Functions are one of the fundamental building blocks in JavaScript. A function is a JavaScript procedure—a set of statements that performs a task or calculates a value. To use a function, you must define it somewhere in the scope from which you wish to call it.

That reads pretty stiff, so here’s my own version:

“A function is basically a small block of code — a wrapper that packages up code or tasks you need to run over and over.”

Or to make it even more everyday: think of a function as a cardboard box. The box (Function) is packed with the tools (Code Block) we’re going to use. When we need them, we open the box (Calling functions) and assemble and run the tools in order.

Calling Functions

In the earlier examples we defined a function, but you’ll notice that just pasting the code into your browser’s DevTools Console does nothing. That’s because we never actually called the function. Once a function is defined, it just sits there quietly waiting for you to wake it up. Here’s another short bit from MDN:

Defining a function does not execute it. Defining it names the function and specifies what to do when the function is called. Calling the function actually performs the specified actions with the indicated parameters.

Here’s a short quote from MDN as well:

The Function constructor creates a new Function object. In JavaScript, every function is actually a Function object.

Notice the phrase Function object. MDN says outright that a function in JavaScript is an object. You can also see Kuro point this out in his IT Ironman series - Day 10: Functions basics (article in Mandarin):

As I mentioned earlier when covering variable types, everything except the primitive types is an object.
When you check a function with typeof, you get “function” back, which makes it seem like function is one of JavaScript’s defined types — but it’s actually still a kind of Object.
You can think of it as a special object (value) that can be invoked.
