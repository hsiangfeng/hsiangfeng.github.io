20 Ways to Work With JavaScript Arrays (And Which Ones Mutate the Original)

#JavaScript

Intro

This post came out of watching a video called 20 ways to work with JS arrays (in Mandarin). There’s one chart in it that’s honestly pure gold.

So I figured I’d just write a post and note down all 20 ways to work with arrays.

GitHub: js-array-operations

YouTube: 20 ways to work with JS arrays (in Mandarin)

The 20 array methods

In that video, the methods get split into two big groups: ones that might mutate the original array depending on what you do, and ones that don’t touch the original array. That sounds abstract written out, so the easiest thing is to just play with them and see for yourself.

Might mutate the original array

push

splice

splice is a bit special — it can do two different things.

Get (or delete) items at a specific position

If you want to delete items at a specific index or position, use splice().

splice(start, deleteCount) takes two arguments:

  • The first is the starting index
  • The second is how many items to delete (or pull out). If you leave it off, it just takes (deletes) everything from the start index to the end.

Now if you check any, only A and B are left.

But what if you only want to pull out (or delete) one item? Just pass the second argument.

Now if you check the original any again, you’ll see C is gone.

(The key with splice is that second argument — leave it off and it takes/deletes everything from the start index onward.)

Put items back at a specific position

Same idea as before: if you can grab items at a position, you can also put them back at a position — and it’s also splice. To insert at a specific position, you use three arguments.

MDN splice

By default, turning an array into a string joins everything with commas.

But if you want to use something other than a comma — a slash, for example — use join.

reduce

reduce basically feels like accumulating. The most common use is adding up all the numbers in an array, and it kind of works like a loop.

To use reduce you have to pass a function, and that function usually takes two arguments, often called prev (the previous one) and next (the next one).

So when reduce runs, it goes something like this:

prev starts as 1 and next is 2 → 1 + 2 = 3. Then that result, 3, becomes the new prev, and next is 4 → 3 + 4, and so on, 7 + 5… until you land on the answer 55.

One thing to watch: reduce must return something, otherwise you’ll get undefined.

Heads up: if you pass strings, you just get string concatenation instead.

MDN reduce

reduceRight

Since reduce accumulates from the first item to the last, there’s also a method that goes the other way — from the last item back to the first.

With numbers you usually can’t tell the difference.

But with strings, the difference is obvious.

MDN reduceRight

indexOf

indexOf is mainly for getting the index position of an item in the array.

If it can’t find the item, it returns -1.

(Why -1, though? Because indexOf indexes start at 0, so 0 already means A. If “not found” returned 0, it’d clash with A — so anything not found returns -1 instead.)

MDN indexOf

lastIndexOf

lastIndexOf is a little different from indexOf. indexOf returns as soon as it finds a match, while lastIndexOf scans the whole array and returns the index of the last match.

some

some checks whether items in the array match a condition, and it returns true if any single item matches. In short, if even one item in the array satisfies the condition, the whole thing returns true.

(Heads up: some needs a function.)

MDN some

every

If you want to check the array and only return true when everything matches, use every.

So once every single item matches, it returns true.

You can also use every to check the length of each string.

MDN every
