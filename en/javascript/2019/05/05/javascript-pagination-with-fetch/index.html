Building Pagination From Scratch in JavaScript (with the Fetch API)

#JavaScript

Intro

When a site has a lot of data, you end up needing pagination. Sometimes the backend handles it and just sends you one page at a time — that’s the easy case, you mostly just configure things on the database side and don’t have to worry. But if the backend dumps all the data to the frontend in one go, then you have to build the pagination logic yourself. So I wrote this post to drill the pagination logic into my own head.

Figuring out what we need

First, it helps to nail down the big-picture requirements.

Generally there are two key numbers you start from, and everything else extends out from there:

  • How many items to show per page
  • The total number of items

Wrapping up

I honestly didn’t expect pagination logic to be this fiddly — I got stuck at one point and was hunting around for answers to reference.

I worked through it using an approach taught in some online courses, but writing it out myself once and documenting it really helped it stick.

Of course there’s room to make this pagination logic more complete, but I didn’t bother — it’s just a small exercise, I purely wanted to wrap my head around the pagination logic.
