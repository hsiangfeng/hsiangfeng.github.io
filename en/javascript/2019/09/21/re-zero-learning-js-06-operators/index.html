RE:Zero — Learning JS From Scratch, Day 6: Operators (and Why 1 + '1' Is '11')

#JavaScript #11th iThome Ironman Contest #RE:Zero — Learning JS From Scratch

Intro

Day 6 of my reincarnated JS life. This time we’re going to talk about operators. We’ve already run into a few of them in earlier posts, but here I want to go through the common ones a bit more properly.

Operators

In everyday dev work, the two operators you see the most are the assignment operator (=) and the dot operator (.). But there are plenty of others — the plus operator (+), minus operator (-), multiply operator (*), divide operator (/), increment operator (++), decrement operator (--), modulo operator (%), and so on. That bunch is collectively called arithmetic operators.

There’s another group too: greater-than (>), less-than (<), greater-than-or-equal (>=), less-than-or-equal (<=), and friends. These are called comparison operators.

And there’s one more group called logical operators, but on Day 6 we’re going to skip those for now.

Comparison Operators

Comparison operators compare two (or more) things and return a boolean — true or false. So again, let’s just look at an example:

There are also a couple of common comparison operators worth calling out:

  • == & === — equality operators
  • != & !== — inequality operators

You might be wondering what the difference is between == & != and === & !==. In short: == & != do loose comparison. When you type 1 == '1';, you get true. Meanwhile === & !== do strict comparison — they check the type too — so 1 === '1'; gives you false.

For real-world dev work, I’d recommend using strict comparison over loose comparison. So we can sum it up like this:

  • regular equal or not-equal (loose comparison)
  • strict equal or not-equal (strict comparison)
