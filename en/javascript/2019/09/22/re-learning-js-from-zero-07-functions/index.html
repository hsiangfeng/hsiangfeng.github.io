Re:Learning JS From Zero — Day 7: Functions, Your Reusable Code Wrappers

#JavaScript #RE：從零開始的學習 JS 生活 #第 11 屆 iT邦幫忙鐵人賽

Intro

Day 7 of getting reincarnated into JS land. This time we’re talking about functions. Functions are one of those things you really have to understand to write code, so let’s get into it.

Functions

Every programming language has functions (or methods? whatever you want to call them). So what are they actually for?

Say you’ve got some action that prints out 1 to 100, and you need to use it in a bunch of places — the homepage, a product list, a checkout screen, and so on. If you don’t use a function, then for every page you’d be copy-pasting that same chunk of code over and over. That’s a maintenance nightmare. Imagine your boss walks over and says “actually, make it 1 to 200.” Now if you’ve got 20 pages, you’re editing 20 places — and good luck not missing one.

So you can think of a function as a way to wrap up some functionality you’ve already written. You just call it and run it, and it does the thing you prepared in advance. Usually the stuff inside a function is the more repetitive code, and you just import it into whatever page needs it.
