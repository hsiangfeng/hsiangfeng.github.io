A Deep Dive Into JavaScript's this: Every Way It Bites You

#JavaScript #this

Intro

I’ve written a bunch of posts about this already, but this time I wanted to go a bit deeper into the this keyword. For a JavaScript engineer or frontend dev, this is one of those keywords you run into all the time — so it’s worth really understanding it.

this

Let’s start with what this even is. On MDN, this is filed under “Expressions and operators” as a primary expression.

What does that mean? Things like {}, [], and function are all expressions. Notice the English word — an expression. So from here on, whether I say “expression” or “statement expression,” I’m talking about expressions.

Quick note: when I say function here, I mean an anonymous function expression — declaring a function without giving it a name. If you can’t quite tell the difference yet, don’t worry, I’ll give plenty of examples.

So… where does this come from? In short, JavaScript creates this keyword every time it sets up an execution context — and that includes creating a function. Just keep that fact in mind for now, and let’s go through how this behaves and what it points to in different situations.

arrow function expression

Before explaining arrow functions, we need to understand the difference between traditional functions and arrow functions. MDN explains that an arrow function has no this, arguments, super, or new.target of its own. The last three aren’t the point — the key one is “an arrow function has no this of its own.” What does that mean? When you use an arrow function, since it has no this of its own, where does its this come from? It refers to the outer scope — the parent (one level up).

That’s why so many of the earlier examples behaved this way: this wandered off at first, but switching to an arrow function fixed it.

For example:

In this example, if we’d written array.forEach(function () { ... });, this would point to window or undefined. As I mentioned, an arrow function has no this of its own, so it refers to the parent’s this (kind of like the prototype chain). A traditional function, having its own this, ends up forming the simple call I described earlier. That’s why using arrow functions where appropriate can simplify your code and make this easier to work with.
