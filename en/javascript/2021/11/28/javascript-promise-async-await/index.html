What Is a Promise in JavaScript? And Why You Really Need to Get It

#JavaScript #Promise #Async #Await #ES7 #ES6 #axios #fetch

Intro

This post is going to talk about what a Promise actually is in JavaScript, plus the Async/Await syntax that sits on top of it.

The Idea Behind Promise

A Promise is, well, a promise. Not in the sense that the word “Promise” translates to “a promise” in your language — if it were that simple I wouldn’t bother writing a whole post about it.

So let me start with why you should care about Promise in the first place.

When you’re building things, you run into so-called “async” events all the time. For example, this code looks like it should run in order:

But what you actually get is…

So what does that have to do with Promise? In real development, we usually want things to run in the order we intend. If the code just jumps around however it likes, the result changes too.

So what’s the concept behind a Promise? The word translates to “a promise,” sure — but what does that mean in practice? Let me use a real-life example.

Say you go to a packed department store to grab a snack — fried chicken, let’s say.

After you order, the staff hands you a pickup ticket or one of those buzzer pagers.

Once you’ve got that ticket or pager, you just wait until it tells you to come collect your food.

In the meantime you’re free to go do other stuff — you’ll come back and deal with it later. But the order is fixed: you order the chicken first, then pick it up.

So you can think of it like this. A Promise basically means: I promise you I’ll do something later.

So how do you actually write one? Let’s keep going.

Promise Static Methods

On top of that, Promise also gives you some static methods (methods you can call without instantiating anything):

  • Promise.all(iterable)
  • Promise.race(iterable)
  • Promise.reject(reason)
  • Promise.resolve(value)
