JavaScript Number Formatting: The Easiest Way (Thousands Separators and Currency)

Intro

I recently had a task where I needed to format some numbers into a thousands separator or currency format. So I figured I'd write up how I went about it.

Thousands Separator Formatting

When you're working on anything involving money or reports, you'll usually need to format numbers with thousands separators. And there are quite a few ways to do it:

js
const number = 1234567.89;
const formattedNumber = number.toLocaleString('en-US'); // "1,234,567.89"
console.log(formattedNumber); // "1,234,567.89"

This method formats the number based on the locale you pass in. 'en-US' means US formatting, which uses a comma as the thousands separator.

But if you want a custom separator, you can use a regular expression:

js
function formatNumberWithCommas(number) {
  return number.toString().replace(/\B(?=(\d{3})+(?!\d))/g, ".");
}

const number = 1234567.89;
const formattedNumber = formatNumberWithCommas(number); // "1.234.567.89"
console.log(formattedNumber); // "1.234.567.89"

This function turns the number into a string, then uses a regex to insert your separator (here it's .) before every group of three digits.

And if you want to slap a currency symbol on the front, you can do this:

js
function formatCurrency(number, currencySymbol = '$') {
  return currencySymbol + number.toString().replace(/\B(?=(\d{3})+(?!\d))/g, ",");
}

const number = 1234567.89;
const formattedCurrency = formatCurrency(number, 'NT$'); // "NT$1,234,567.89"
console.log(formattedCurrency); // "NT$1,234,567.89"

You can probably already tell that whether it's thousands separators or currency formatting, both are basically pulled off with a regular expression.

If you'd rather not roll your own like this, you might reach for a package like numeral. But... that package isn't maintained anymore and it's pretty old, so a lot of people just end up writing their own.

Intl.NumberFormat

The old-school methods work, but they're not exactly intuitive or easy to maintain. So... is there a more modern, JavaScript-native way to do this? Yep, there is — it's Intl.NumberFormat.

For modern development I'd recommend the built-in Intl.NumberFormat. It performs well and there's nothing to install. Here's the basic shape of it:

js
const formatter = new Intl.NumberFormat(locales, options);
formatter.format(number);
  • locales: the locale to use, e.g. 'en-US', 'zh-TW', and so on.
  • options: the formatting options, e.g. style, currency, minimumFractionDigits, etc.

Intl.NumberFormat is a built-in JavaScript object made specifically for formatting numbers, and it handles all kinds of formatting needs really conveniently. Here's an example:

js
const number = 1234567.89;
const formatter = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
});
const formattedCurrency = formatter.format(number); // "$1,234,567.89"
console.log(formattedCurrency); // "$1,234,567.89"

As you can see, this not only handles the thousands separators but also adds the currency symbol directly — no need to hand-write a regex anymore. And it automatically adjusts the format based on the locale, which is super handy.

If you want to format into a different currency, you just change the currency value. For example:

js
const number = 1234567.89;
const formatter = new Intl.NumberFormat('ja-JP', {
  style: 'currency',
  currency: 'JPY',
});
const formattedCurrency = formatter.format(number); // "￥1,234,568"
console.log(formattedCurrency); // "￥1,234,568"

And just like that, you've got the number formatted as Japanese yen.

options

options itself has a bunch of other settings you can use. Here are some of the common ones:

  • style: the formatting style
    • 'decimal': decimal format (the default)
    • 'currency': currency format
    • 'percent': percentage format
  • currency: the currency type (e.g. 'USD', 'EUR', 'JPY', etc.). You only need this when style is set to 'currency'.
  • currencyDisplay: how to show the currency symbol
    • 'symbol': show the currency symbol (e.g. $, €, ¥, etc.)
    • 'code': show the currency code (e.g. USD, EUR, JPY, etc.)
    • 'name': show the currency name (e.g. US Dollar, Euro, Japanese Yen, etc.)
  • minimumIntegerDigits: the minimum number of digits in the integer part — if it's short, it pads with leading zeros.
  • minimumFractionDigits: the minimum number of digits in the fractional part — if it's short, it pads with trailing zeros.
  • maximumFractionDigits: the maximum number of digits in the fractional part — anything beyond this gets rounded.
  • minimumSignificantDigits: the minimum number of significant digits.
  • maximumSignificantDigits: the maximum number of significant digits.

Examples

Now let me run through an example for each option.

style

What you want to format the number as:

js
// decimal (default)
new Intl.NumberFormat('zh-TW', { style: 'decimal' }).format(1234567.89);
// "1,234,567.89"

// currency
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'TWD' }).format(1234567.89);
// "$1,234,567.89"

// percent
new Intl.NumberFormat('zh-TW', { style: 'percent' }).format(0.123);
// "12%"

currency

When style: 'currency', this says which currency to use:

js
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'USD' }).format(1234.5);
// "US$1,234.50"

new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'JPY' }).format(1234.5);
// "¥1,235"  (no decimals, since the yen doesn't have fractional units)

new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'EUR' }).format(1234.5);
// "€1,234.50"

currencyDisplay

Decides how the currency is displayed:

js
// symbol (default)
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'USD', currencyDisplay: 'symbol' }).format(1234);
// "US$1,234.00"

// code
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'USD', currencyDisplay: 'code' }).format(1234);
// "USD 1,234.00"

// name
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'USD', currencyDisplay: 'name' }).format(1234);
// "1,234.00 美元"

// narrowSymbol (some currencies drop the country code)
new Intl.NumberFormat('zh-TW', { style: 'currency', currency: 'USD', currencyDisplay: 'narrowSymbol' }).format(1234);
// "$1,234.00"

Note The browser, Node.js version, and ICU data all affect the output. If you need to guarantee you get NT$, you can additionally specify currencyDisplay: 'narrowSymbol' or handle it manually.

minimumIntegerDigits

Sets the minimum number of digits in the integer part, padding with leading zeros:

js
new Intl.NumberFormat('zh-TW', { minimumIntegerDigits: 3 }).format(7);
// "007"

new Intl.NumberFormat('zh-TW', { minimumIntegerDigits: 5 }).format(123);
// "00123"

minimumFractionDigits

The minimum number of decimal places to show:

js
new Intl.NumberFormat('zh-TW', { minimumFractionDigits: 2 }).format(5);
// "5.00"

new Intl.NumberFormat('zh-TW', { minimumFractionDigits: 4 }).format(5.1);
// "5.1000"

maximumFractionDigits

The maximum number of decimal places to show (extras get rounded):

js
new Intl.NumberFormat('zh-TW', { maximumFractionDigits: 2 }).format(5.6789);
// "5.68"

new Intl.NumberFormat('zh-TW', { maximumFractionDigits: 0 }).format(5.9);
// "6"

minimumSignificantDigits

The minimum number of significant digits, padded with zeros:

js
new Intl.NumberFormat('zh-TW', { minimumSignificantDigits: 3 }).format(7);
// "7.00"

new Intl.NumberFormat('zh-TW', { minimumSignificantDigits: 5 }).format(123);
// "123.00"

maximumSignificantDigits

The maximum number of significant digits — extras get rounded.

js
new Intl.NumberFormat('zh-TW', { maximumSignificantDigits: 3 }).format(12345);
// "12300"

new Intl.NumberFormat('zh-TW', { maximumSignificantDigits: 4 }).format(0.0012345);
// "0.001235"
