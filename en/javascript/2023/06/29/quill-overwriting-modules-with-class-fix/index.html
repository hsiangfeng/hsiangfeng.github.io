Fixing the 'quill Overwriting modules/xxx with class...' Warning

#Vue #JavaScript #Quill #VueQuill

Intro

Just a quick note here on a warning I hit while using Quill — the “quill Overwriting modules/xxx with class…” one — and how I got rid of it.

The problem

I was using the VueQuill package to work with Quill, and while using it I kept getting this error in the console:

So why go with options? I did try the way VueQuill wraps things officially, but I found that extending it gets pretty hard (like pulling in highlight, for example), so I switched to the options approach instead.

Now let me circle back and explain why “quill Overwriting modules/xxx with class...“ keeps showing up in the first place. We know that with v-if and route switching, the component itself gets destroyed. But Quill’s modules are registered on Quill as a whole — they don’t get cleared automatically just because the component was destroyed. So when you create the component again, it tries to register the same module again, and that’s where the warning comes from.

So all you need to do is check whether it’s already been registered before registering, and skip it if it has. That clears the problem right up.
