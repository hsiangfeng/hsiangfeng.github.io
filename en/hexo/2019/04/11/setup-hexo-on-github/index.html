Setting Up Hexo + GitHub

#Hexo

Intro

There are plenty of tutorials online for Hexo + GitHub, but since I’m building a pure tech blog, here’s my own writeup of how to set up Hexo and deploy it to GitHub.

Note: the theme I’m using is NexT.

Versions & environment

Post environment

NexT version

You’ll want a basic grasp of Git, GitHub, and npm. If you don’t know Git or npm yet, I’d recommend reading my earlier posts on the basics so the commands below make sense:

1. Create a repo on GitHub

Repository name format:

Don’t set your blog up inside a subfolder. The most common bad pattern I see is your-username.github.io/blog. That setup will probably hit weird issues — common ones I’ve heard:

  • CSS & JS break
  • Articles don’t render correctly
  • Third-party plugins can’t get the correct URL path
  • … and so on

So I strongly recommend hosting your blog at the root, like my blog — fewer landmines.

Parameter notes:

  • path:
    • Default is search.xml
    • If you change it to search.json, the output is a JSON file, smaller in size
  • field:
    • post (Default) — only searches posts
    • page — only pages
    • all — includes everything
  • format — the search page format (XML only)
    • html (Default) — compressed raw HTML
    • raw — original text (Markdown)
    • excerpt — only excerpt (in my tests this seemed to be only the title??)

A note on NexT versions

I’m on NexT 7.1.0. If you’re on NexT 5, some settings will look different — I’d recommend upgrading to 7.

If you’re setting up a fresh Hexo site, just go straight to the latest 7.2.0 — it includes some GA / FB fixes I helped contribute, so unless you have specific reasons to stick with an older version, just use the latest.

If you have feature requests for the theme, try opening an issue on the official GitHub — your wish might come true. You can also leave a comment on this post, and if time permits and the feature seems broadly useful, I’ll look into it and bring it up with the NexT team.
