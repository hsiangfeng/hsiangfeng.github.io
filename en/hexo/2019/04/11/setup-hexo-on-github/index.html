Intro

There are plenty of tutorials online for Hexo + GitHub, but since I'm building a pure tech blog, here's my own writeup of how to set up Hexo and deploy it to GitHub.

Hexo site

Note: the theme I'm using is NexT.

Versions & environment

text
OS: Windows 10 X64
Node.js: V11.7.0
IDE: VS Code

Post environment

Hexo version

  • hexo: 3.8.0
  • hexo-cli: 2.0.0

NexT version

  • NexT: 7.1.0

Prerequisites

You'll want a basic grasp of Git, GitHub, and npm. If you don't know Git or npm yet, I'd recommend reading my earlier posts on the basics so the commands below make sense:

  • Basic Git commands (article in Mandarin)
  • Differences between npm install commands (article in Mandarin)

1. Create a repo on GitHub

Create a new repository

Repository name format:

text
your-username.github.io

Don't set your blog up inside a subfolder. The most common bad pattern I see is your-username.github.io/blog. That setup will probably hit weird issues — common ones I've heard:

  • CSS & JS break
  • Articles don't render correctly
  • Third-party plugins can't get the correct URL path
  • ... and so on

So I strongly recommend hosting your blog at the root, like my blog — fewer landmines.

2. Install Hexo

Hexo is installed via command line, so open your terminal and run:

js
npm install hexo-cli -g

hexo-cli install

  • This installs Hexo-Cli globally (-g) via npm. Should be done in a few seconds.

3. Create a Hexo project

cd into the directory where you want to create the project, then run (replace projectname with your own):

cmd
hexo init projectname
  • This initialises a new Hexo project.

Init Hexo

Enter the Hexo directory

Once created, cd into the new folder:

cmd
cd projectname

cd into folder

Install dependencies

The fresh Hexo project still needs its npm packages installed:

js
npm install
  • This installs the dependencies from package.json.

Start Hexo

Last step — start it:

cmd
hexo server

Start Hexo

Theme source

The theme I use is NexT 7.1.0.

NexT theme

Official theme repo: theme-next/hexo-theme-next.

Official theme

Switching themes

Unzip the theme into the themes folder under your blog:

themes

Open _config.yml: Edit _config.yml

Find around line 73-76:

js
theme: hexo-theme-next-master

Change theme to the folder name you just downloaded, then restart:

cmd
hexo server

Hexo basic configuration

The main bits to edit in _config.yml:

yml
# Site
title: 是 Ray 不是 Array # Site title
subtitle: 歡迎來到網路世界。 # Subtitle
description: 是 Ray 不是 Array # Site description
keywords: # Keywords
author: Ray # Site author
language: zh-TW # Site language
timezone:

# URL
url: http://yoursite.com # Your site URL — mine is hsiangfeng.github.io
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

Deploy to GitHub

Install the deploy plugin first — Hexo doesn't ship with it:

js
npm install hexo-deployer-git --save

Update the Deployment section in _config.yml:

yml
deploy:
  type: git # mode
  repo: https://github.com/hsiangfeng/ray.github.io.git # your GitHub repo URL
  branch: master # branch

Then deploy: PS. Watch out for spacing — type, repo, branch each need a leading space.

js
hexo d -g
  • d = deploy, g = generate static pages.

Other commands

js
hexo new [title]
  • Create a new post.
js
hexo s -g
  • s = server, -g = generate static pages.
js
hexo d -g
  • d = deploy, -g = generate static pages.
js
hexo clean
  • Delete generated static pages and cache.

Add Google Analytics

Using NexT 7.1.0 as the example.

In the theme's _config.yml, around lines 754-757:

yml
google_analytics:
  tracking_id:
  localhost_ignored:

Update to:

yml
google_analytics:
  tracking_id: UA-XXXXXX-1 # your UA tracking ID
  localhost_ignored: true

You may also need a small fix to get GA actually firing. (This issue is fixed in NexT 7.2.0.)

text
File path: themes\hexo-theme-next-master\layout\_third-party\analytics\google-analytics.swig

Replace the contents with:

swig
{% if theme.google_analytics.tracking_id %}
  <script async src="https://www.googletagmanager.com/gtag/js?id={{ theme.google_analytics.tracking_id }}"></script>
  <script>
    var host = window.location.hostname;
    if (host !== "localhost" || !{{theme.google_analytics.localhost_ignored}}) {
      window.dataLayer = window.dataLayer || [];
      function gtag(){dataLayer.push(arguments);}
      gtag('js', new Date());
      gtag('config', '{{ theme.google_analytics.tracking_id }}');
    }
  </script>
{% endif %}

The issue is that the default src path in the NexT template is missing the https:// prefix.

Add Sitemap

Google uses sitemap.xml to discover your site, but Hexo doesn't ship with sitemap generation:

js
npm install hexo-generator-sitemap

Once installed, regenerate the site and submit sitemap.xml in Google Search Console.

If Search Console asks for verification, update around line 131 of the theme's _config.yml (using NexT here):

yml
# Google Webmaster tools verification.
# See: https://www.google.com/webmasters
google_site_verification: # verification code

Add search to Hexo

There's no built-in site search, so you'll need a plugin.

  • Use the NexT official fork of the plugin:

NexT

cmd
npm install hexo-generator-searchdb --save

Then update the site's _config.yml — paste the following anywhere:

yml
# Search
search:
  path: search.xml
  field: post
  format: html
  limit: 100

Parameter notes:

  • path:
    • Default is search.xml
    • If you change it to search.json, the output is a JSON file, smaller in size
  • field:
    • post (Default) — only searches posts
    • page — only pages
    • all — includes everything
  • format — the search page format (XML only)
    • html (Default) — compressed raw HTML
    • raw — original text (Markdown)
    • excerpt — only excerpt (in my tests this seemed to be only the title??)
    • more — more (????)
  • limit — how many posts to index

Now enable Search in the theme (using NexT here). Open the theme's _config.yml, search for Local search, and set enable to true.

My search config:

yml
search:
  path: search.json
  field: post
  format: html
  limit: 10000

Format differences

html (Default):

html

raw:

raw

excerpt:

excerpt

more:

more

A note on NexT versions

I'm on NexT 7.1.0. If you're on NexT 5, some settings will look different — I'd recommend upgrading to 7.

If you're setting up a fresh Hexo site, just go straight to the latest 7.2.0 — it includes some GA / FB fixes I helped contribute, so unless you have specific reasons to stick with an older version, just use the latest.

If you have feature requests for the theme, try opening an issue on the official GitHub — your wish might come true. You can also leave a comment on this post, and if time permits and the feature seems broadly useful, I'll look into it and bring it up with the NexT team.
