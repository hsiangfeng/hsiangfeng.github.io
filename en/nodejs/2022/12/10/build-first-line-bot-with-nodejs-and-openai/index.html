Build Your First LINE Bot with Node.js — Using OpenAI GPT-3 and GPT-3.5 as Examples

#Node.js #OpenAI #ChatGPT #GPT-3 #GPT-3.5 #LINE Bot #API #Render #LINE

Intro

Danger
The main focus is GPT-3, but I’ve added GPT-3.5 conversion and sample code at the end. If you want to use GPT-3.5, check the later section.

OpenAI’s ChatGPT has been blowing up recently. OpenAI also exposes an API we can integrate with. This post focuses on GPT-3, but also covers how to switch to ChatGPT API. We’ll use Node.js to build a LINE chatbot backed by OpenAI.

Setup

A few prep steps before we get into the code.

Note
This post was written on December 10, 2022. If the UI has changed since, follow whatever you see currently.

Apply for OpenAI API

Go to OpenAI and sign up. You can use a third-party login (Google, Microsoft) or email.

After signup, you can apply for the API.

If you already have an account, just log in.

After login, you’ll see:

Click your avatar → “View API keys”.

On the API keys page, click “Create New Secret key”. You’ll get your OpenAI key — save it.

Danger
The API key is sensitive. Don’t share it with anyone.

What’s this key for? We’ll use it to call OpenAI’s API to use GPT. If you’ve never tried GPT, play with ChatGPT — it’s smarter than GPT-3 and can do things like:

  • Plan a Japan travel itinerary
  • Write marketing copy
  • Find bugs in your code
  • Solve coding problems

…and a lot more.

ChatGPT API isn’t open yet, but we can use GPT-3, which is what this API key is for.

Note
OpenAI gives you a free trial credit ($18 USD for 3 months). Pricing varies by model — Davinci is $0.0200 USD per 1K tokens. See pricing.

Extra

What are tokens?
Tokens are characters in your input. For example, “I want to travel to Japan” has 6 words/tokens. (Token counting actually breaks down by tokenization rules rather than word count, but characters are a rough analog.)

Apply for LINE Developer

Now apply for a LINE Developer account. Go to LINE Developer. Log in with a LINE account.

Once logged in, go to the LINE Developer Console.

The dashboard is empty, so click “Create a new provider”. Think of a provider as a company — it can host multiple services like LINE Bot, LINE MUSIC, etc.

We’ll call ours “Example Bot”.

Creating is fast. Click “Create” and you’ll land on the Example Bot page.

Click “Create a Messaging API channel” since we’re building a chatbot.

Fill in the fields:

  • Channel type
    • Keep the default Messaging API
  • Provider
    • Keep “Example Bot”
  • Company or owner’s country or region
    • Pick your country
  • Channel icon
    • Upload later, skip
  • Channel name
    • Let’s call it “Run-Away School” (sorry boss)
  • Channel description
    • Brief description like “Hi, I’m Run-Away School’s little customer service rep”
  • Category
    • Pick “Other media”
  • Subcategory
    • “Media (Other)”
  • Email address
    • For LINE to contact you about your bot
  • Privacy policy URL and Terms of use URL
    • Skip, not needed

Check both consent boxes and click “Create”.

(Page is too long to screenshot.)

After clicking “Create”, a confirmation popup appears — click ok.

Now we have a LINE Bot. Get the following from “Basic settings”:

  • Channel ID
  • Channel secret

Then switch to “Messaging API” and find “Channel access token”:

Click “Issue” — you’ll get a long Token. Save it for later.

LINE Bot and OpenAI signups done. Time to code.

Wrap-up

The LINE Bot creation flow is incredibly long with a lot of steps and details. But you also learn how to integrate third-party services and deploy code to Render — a great little practice project. If you want to try a chatbot, this method works.

One thing to watch out for: to make OpenAI’s GPT smarter, you need to pass the entire context (including bot replies) along with new messages. Otherwise it’ll feel a bit dumb. But LINE chatbots aren’t great for this — over time the message volume gets huge and you can’t delete messages, so I left this out. We reply per-question without including history.

Info
If you find Render slow for debugging, consider ngrok — it exposes your local service publicly. See this post (article in Mandarin).

Note

Render’s free plan goes to sleep after about 15 minutes of inactivity, like Heroku’s old free tier. Waking up takes 30 seconds to a minute. To avoid this, consider UptimeRobot to keep it awake (exploiting the worker).
