Intro

Danger The main focus is GPT-3, but I've added GPT-3.5 conversion and sample code at the end. If you want to use GPT-3.5, check the later section.

OpenAI's ChatGPT has been blowing up recently. OpenAI also exposes an API we can integrate with. This post focuses on GPT-3, but also covers how to switch to ChatGPT API. We'll use Node.js to build a LINE chatbot backed by OpenAI.

Note This post doesn't walk through every line of code — the signup flow alone is already pretty long. If you don't want to wait until the end for the sample code, here it is: Sample code for this post

Setup

A few prep steps before we get into the code.

Note This post was written on December 10, 2022. If the UI has changed since, follow whatever you see currently.

Apply for OpenAI API

Go to OpenAI and sign up. You can use a third-party login (Google, Microsoft) or email.

Sign up

After signup, you can apply for the API.

If you already have an account, just log in.

After login, you'll see:

Welcome to OpenAI

Click your avatar → "View API keys".

View API keys

On the API keys page, click "Create New Secret key". You'll get your OpenAI key — save it.

OpenAI Key

Danger The API key is sensitive. Don't share it with anyone.

What's this key for? We'll use it to call OpenAI's API to use GPT. If you've never tried GPT, play with ChatGPT — it's smarter than GPT-3 and can do things like:

  • Plan a Japan travel itinerary
  • Write marketing copy
  • Find bugs in your code
  • Solve coding problems

...and a lot more.

ChatGPT API isn't open yet, but we can use GPT-3, which is what this API key is for.

Note OpenAI gives you a free trial credit ($18 USD for 3 months). Pricing varies by model — Davinci is $0.0200 USD per 1K tokens. See pricing.

Extra

What are tokens? Tokens are characters in your input. For example, "I want to travel to Japan" has 6 words/tokens. (Token counting actually breaks down by tokenization rules rather than word count, but characters are a rough analog.)

Build the project

I'll skip the basics — you probably know how to init a project and install Node.js. I'm using pnpm here, but you can use npm or yarn. Quick commands:

Note The commands below are for Mac. Windows users, look up the equivalents.

bash
# Create folder
mkdir example-line-bot
# Enter folder
cd example-line-bot
# Init project
pnpm init

Project initialized. Time to install the LINE Bot package.

Install packages

Install:

bash
pnpm i @line/bot-sdk express dotenv

Then create app.js in the project root.

Write the LINE Bot

Import @line/bot-sdk, express, and dotenv:

js
require('dotenv').config();

const express = require('express');
const line = require('@line/bot-sdk');

LINE provides a sample we can use as-is to test:

js
require('dotenv').config();

const line = require('@line/bot-sdk');
const express = require('express');

// create LINE SDK config from env variables
const config = {
  channelAccessToken: process.env.CHANNEL_ACCESS_TOKEN,
  channelSecret: process.env.CHANNEL_SECRET,
};

// create LINE SDK client
const client = new line.Client(config);

// create Express app
// about Express itself: https://expressjs.com/
const app = express();

// register a webhook handler with middleware
// about the middleware, please refer to doc
app.post('/callback', line.middleware(config), (req, res) => {
  Promise
    .all(req.body.events.map(handleEvent))
    .then((result) => res.json(result))
    .catch((err) => {
      console.error(err);
      res.status(500).end();
    });
});

// event handler
function handleEvent(event) {
  if (event.type !== 'message' || event.message.type !== 'text') {
    // ignore non-text-message event
    return Promise.resolve(null);
  }

  // create a echoing text message
  const echo = { type: 'text', text: event.message.text };

  // use reply API
  return client.replyMessage(event.replyToken, echo);
}

// listen on port
const port = process.env.PORT || 3000;
app.listen(port, () => {
  console.log(`listening on ${port}`);
});

Create .env with:

env
CHANNEL_ACCESS_TOKEN=
CHANNEL_SECRET=

Try node app.js — if it doesn't start, check the previous steps.

Integrating OpenAI API

The main event. The LINE Bot setup is long and complex, so I covered it carefully — to make sure you don't hit issues here.

Install the OpenAI package

Install:

bash
pnpm i openai

OpenAI provides a sample:

js
const { Configuration, OpenAIApi } = require("openai");

const configuration = new Configuration({
  apiKey: process.env.OPENAI_API_KEY,
});
const openai = new OpenAIApi(configuration);

const completion = await openai.createCompletion({
  model: "text-davinci-002",
  prompt: "Hello world",
});
console.log(completion.data.choices[0].text);

We'll modify this to work with LINE Bot.

OpenAI offers multiple models:

  • text-davinci-003
  • text-curie-001
  • text-babbage-001
  • text-ada-001

text-davinci-003 is the strongest, so we'll use that. See model intro.

Update app.js

Import OpenAI in app.js:

js
const { Configuration, OpenAIApi } = require("openai");

Instantiate:

js
const configuration = new Configuration({
  apiKey: process.env.OPENAI_API_KEY,
});
const openai = new OpenAIApi(configuration);

apiKey is the OpenAI API Key we got earlier, from .env.

If you haven't applied for it, you can't continue. Apply for an OpenAI API Key.

.env should now look like:

env
CHANNEL_ACCESS_TOKEN=
CHANNEL_SECRET=

OPENAI_API_KEY=

Modify the handleEvent function from LINE's sample:

js
async function handleEvent(event) {
  if (event.type !== 'message' || event.message.type !== 'text') {
    // ignore non-text-message event
    return Promise.resolve(null);
  }

  const completion = await openai.createCompletion({
    model: "text-davinci-003",
    prompt: event.message.text ,
  });

  // create a echoing text message
  const echo = { type: 'text', text: completion.data.choices[0].text };

  // use reply API
  return client.replyMessage(event.replyToken, echo);
}

We changed prompt to event.message.text (user input). completion.data.choices[0].text is OpenAI's response.

Save, commit, push to GitHub. Render will auto-redeploy.

Note Remember to add OPENAI_API_KEY to Render's environment variables, otherwise you'll get errors.

After deploy, message your LINE Bot — it'll talk back.

Chat started

You'll notice replies are short. We haven't told it how many Tokens to use:

js
const completion = await openai.createCompletion({
  model: "text-davinci-003",
  prompt: event.message.text ,
  max_tokens: 200,
});

max_tokens controls response length. 200 should be plenty.

Trim OpenAI's response

OpenAI responses come with extra whitespace because it breaks responses into sections separated by spaces. Trim:

js
const completion = await openai.createCompletion({
  model: "text-davinci-003",
  prompt: event.message.text ,
  max_tokens: 200,
});

const echo = { type: 'text', text: completion.data.choices[0].text.trim() };

Push again, and the bot will reply cleanly.

Clean reply

You now have a working chatbot integrated with OpenAI.

Using GPT-3.5

OpenAI later released the ChatGPT API. I'll show what changes — the code conversion is pretty minimal.

Before:

js
const completion = await openai.createCompletion({
  model: "text-davinci-003",
  prompt: event.message.text,
  max_tokens: 200,
});

Change createCompletion to createChatCompletion (weirdly the official docs don't mention this). Change prompt to messages and adjust model:

js
const completion = await openai.createChatCompletion({
  model: "gpt-3.5-turbo",
  messages: [{
    role: 'user',
    content: event.message.text,
  }],
  max_tokens: 200,
});

Also update const echo = { type: 'text', text: completion.data.choices[0].text.trim() }; to:

js
const echo = { type: 'text', text: choices.message.content.trim() || 'Sorry, I have nothing to say.' };

That's the switch from GPT-3 to GPT-3.5.

Full code:

js
require('dotenv').config();

const express = require('express');
const line = require('@line/bot-sdk');
const { Configuration, OpenAIApi } = require("openai");

const configuration = new Configuration({
  apiKey: process.env.OPENAI_API_KEY,
});
const openai = new OpenAIApi(configuration);

// create LINE SDK config from env variables
const config = {
  channelAccessToken: process.env.CHANNEL_ACCESS_TOKEN,
  channelSecret: process.env.CHANNEL_SECRET,
};

// create LINE SDK client
const client = new line.Client(config);

// create Express app
// about Express itself: https://expressjs.com/
const app = express();

// register a webhook handler with middleware
// about the middleware, please refer to doc
app.post('/callback', line.middleware(config), (req, res) => {
  Promise
    .all(req.body.events.map(handleEvent))
    .then((result) => res.json(result))
    .catch((err) => {
      console.error(err);
      res.status(500).end();
    });
});

// event handler
async function handleEvent(event) {
  if (event.type !== 'message' || event.message.type !== 'text') {
    // ignore non-text-message event
    return Promise.resolve(null);
  }

  const { data } = await openai.createChatCompletion({
    model: "gpt-3.5-turbo",
    messages: [
      {
        role: 'user',
        content: event.message.text,
      }
    ],
    max_tokens: 500,
  });

  // create a echoing text message
  const [choices] = data.choices;
  const echo = { type: 'text', text: choices.message.content.trim() || 'Sorry, I have nothing to say.' };

  // use reply API
  return client.replyMessage(event.replyToken, echo);
}

// listen on port
const port = process.env.PORT || 3000;
app.listen(port, () => {
  console.log(`listening on ${port}`);
});

One more thing about messages. We currently send an array with one object:

js
[{
  role: 'user',
  content: event.message.text,
}]

This means the message is from the "user". If you want to maintain conversation context like ChatGPT, there's another role — system. Add the bot's previous response too:

js
[
  {
    role: 'user',
    content: event.message.text,
  },
  {
    role: 'system',
    content: "Hi, I'm a bot",
  }
],

Now ChatGPT knows which message is from the bot and which is from the user.

Wrap-up

The LINE Bot creation flow is incredibly long with a lot of steps and details. But you also learn how to integrate third-party services and deploy code to Render — a great little practice project. If you want to try a chatbot, this method works.

One thing to watch out for: to make OpenAI's GPT smarter, you need to pass the entire context (including bot replies) along with new messages. Otherwise it'll feel a bit dumb. But LINE chatbots aren't great for this — over time the message volume gets huge and you can't delete messages, so I left this out. We reply per-question without including history.

Sample code if you want to grab it directly. Most of the code follows the official samples with minimal modification, so you can use it as-is.

Hope you have fun building a LINE Bot.

Info If you find Render slow for debugging, consider ngrok — it exposes your local service publicly. See this post (article in Mandarin).

Note

Render's free plan goes to sleep after about 15 minutes of inactivity, like Heroku's old free tier. Waking up takes 30 seconds to a minute. To avoid this, consider UptimeRobot to keep it awake (exploiting the worker).
