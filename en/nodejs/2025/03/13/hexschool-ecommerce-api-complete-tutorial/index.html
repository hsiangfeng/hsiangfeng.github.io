2025-03-13 Nodejs

Hexschool Capstone Class Study Group: Complete E-commerce API Tutorial

#Nodejs

Intro

A walkthrough of running the backend code provided by Hexschool to spin up an API server and testing it with Postman.

Prerequisites

Before continuing, make sure you have:

  • Node.js v20 or above installed
    • Node.js official site
    • Bonus: Installing nvm on Mac OS X, Installing and Using nvm on Windows 10
  • Postman installed
    • Postman site
    • Bonus: Quick start guide to Postman (article in Mandarin)
  • Git installed
    • Git site
  • A GitHub account
    • GitHub site
  • Visual Studio Code installed
    • VS Code site
  • Chrome browser
    • Chrome site

Once these are ready, we can move on.

Note
This post doesn’t cover how to install each of these — if you’re not familiar, look up the basics first.

Firebase project setup

The Hexschool project uses Firebase Storage for image storage, so we need to set up Firebase first.

Note
Firebase’s new policy requires a credit card for some features — but don’t worry, Google provides a free tier.
Firebase’s UI may differ slightly per account, but I’ll describe it from scratch.

Create a Firebase project

Open Firebase and click “Go to console” in the top right.

Click “Add project” or “Create a project”.

A full-screen modal pops up — enter a project name. I’m using “Firebase-Express”, then click “Continue”.

Second step — Firebase asks if you want Google Analytics. We don’t need it, so disable and click “Create project”.

Wait a moment — go use the bathroom.

Once ready, click “Continue”.

You’ll land on the project console. Firebase project ready.

Set up Firebase Storage

Note
This step matters — without it, you can’t get the Storage Bucket URL, which the code needs later.

Click “Storage” in the left sidebar, then “Upgrade project”.

Firebase will ask you to upgrade your account. To prevent abuse, anyone using Firebase Storage now needs to link a credit card. Don’t worry, click “Create Cloud Billing Account”.

Google Cloud modal pops up — click “Confirm”.

Fill in tax info, link a credit card at the bottom. I used a card I’m not actively using. Click “Start my free trial”.

Note
New users get $300 USD of free credit from Google.

It’ll set up billing — wait a moment.

Done, back to Firebase.

Note
If your account has issues — cross-region, invalid card — Google might ask you to verify again.

You might see a budget setup prompt. Skip it for now — set it up later if needed.

Now link the Cloud Billing account to the Firebase project. Click “Link Cloud Billing Account”.

When you see this, you’re done.

To check current usage, click the gear → “Usage and billing”.

Back to Firebase Storage. Click “Get started”.

Choose a Storage region — US-CENTRAL1 is the default, no need to change. Click “Continue”.

Set security rules — keep the defaults.

When you see this, Storage is set up.

Note
Why use Firebase instead of Google Cloud directly? Firebase is built on top of Google Cloud, but Firebase adds convenient features — Authentication, Firestore, Realtime Database, Storage, Hosting, etc. — with friendlier UI.

Get the Firebase Admin SDK

For image uploads we need the Firebase Admin SDK. Click the gear → “Project settings”.

Click “Service accounts” — you’ll see the Firebase Admin SDK page.

Click “Generate new private key” (you can pick your programming language for sample code). You’ll get a JSON file — save it carefully, we’ll use it.

Danger
Don’t share the key with anyone.
Don’t push the key to GitHub (if you did, delete that repo and regenerate the key).

Firebase prep done. Moving on.

The e-commerce project

Download Hexschool’s project. Click this link: “Hexschool Capstone E-commerce API“ and click “Download ZIP” to get the project archive.

Extract and open it in VS Code. Project structure:

The project includes commands:

What each does:

  • start: build and start Docker environment
  • restart: restart Docker environment
  • stop: stop Docker environment
  • clean: clean up Docker environment
  • dev: start dev mode
  • test:unit: run Jest unit tests
  • test:integration: run Jest integration tests
  • init:schema: initialize database

The commonly used ones are start, restart, stop, clean, test:integration. Don’t worry about the others.

Note
Select the entire JSON before Join Lines — otherwise you’ll break the JSON format.

Now get the Firebase Storage Bucket. Go back to Firebase Console, click “Storage” in the sidebar. Find gs://... — that’s your Storage Bucket name. Copy into FIREBASE_STORAGE_BUCKET.

Environment variables are set. Time to restore the project.

How to develop

So far we’ve just set up the project. Let me show how to actually develop with it.

Prep for local dev

The earlier .env is for Docker, not local dev. To develop, update:

What we’re changing:

  • DB_HOST: comment out postgres, uncomment localhost
  • PORT: change 8080 to 5500

Because we’re switching from Docker to local dev, change DB_HOST to localhost. Change PORT to 5500 to avoid conflict with Docker’s port.

Important — confirm Docker PostgreSQL is running properly. You should see this:

Both dots green. Otherwise the project won’t start.

Self-check todo before continuing:

  • Ran npm ci to install packages
  • Copied .env.example to .env
  • Pasted Firebase Admin SDK key into FIREBASE_SERVICE_ACCOUNT
  • Copied Firebase Storage Bucket name into FIREBASE_STORAGE_BUCKET
  • Confirmed Docker Desktop shows two green dots
  • Confirmed Postman tests work
  • Changed DB_HOST to localhost, PORT to 5500
  • Confirmed Docker PostgreSQL is running

All checked? Start the project:

Dev workflow

The project includes test files to verify your code matches expected behavior. How to use them?

Open “Hexschool Capstone E-commerce API“ and switch to the answer branch. This branch has the completed answers — you can reference them while developing.

Workflow:

  1. Implement features based on the DB schema and API docs from the course
  2. After implementing, copy the test files from answer to your main branch
  3. Run npm run test:integration for integration tests

Note
Test files are based on the course’s API docs, so reference those when writing your code.

If tests pass, your code is correct. If not, check your code.

Let me demo the workflow.

Automating tests with GitHub Actions

Besides running tests locally, you can use GitHub Actions to auto-run on every push. That way every commit gets tested.

Apply for a GitHub Token

First, get a GitHub Token. Go to GitHub Settings (click your avatar).

Click “Developer settings” in the sidebar.

Click “Personal access tokens”, then “Generate new token (classic)”.

Fill in:

  • Note: Token name
  • Expiration: Token expiry
  • Select scopes: Check repo and workflow

Ignore other fields, click “Generate token”:

Copy this token — it only shows once. Keep it safe.

Set up GitHub Repository Secrets

Open GitHub and create a new repo.

Set up Secrets and variables for this repo. Settings → “Secrets and variables” → “Actions”.

Click “New repository secret”. Add:

  • FIREBASE_SERVICE_ACCOUNT: Firebase Admin SDK key
  • FIREBASE_STORAGE_BUCKET: Firebase Storage Bucket name
  • DEPLOY_GITHUB_TOKEN: GitHub Token

These are the env vars we set up earlier.

Note
Tests use Firebase Admin SDK, hence why we set this up. Others aren’t needed — .env.example defaults are fine. When GitHub Actions runs, it auto-copies .env.example to .env.

Once that’s done, push to your repo. Every push will trigger GitHub Actions to auto-test.

That’s the full workflow.
