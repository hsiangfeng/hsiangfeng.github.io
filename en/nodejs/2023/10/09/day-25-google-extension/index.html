Day 25 - Google Extension

#Node.js #Node.js 30 Days Series

Intro

We’re entering a new chapter — this one introduces Google Extension. What is it? Let’s quickly get acquainted in this post.

Chrome Extension

As the most-used browser, Chrome — you’ve definitely browsed its extension store, right? These extensions are what we call Chrome Extensions. Through them you can make your browser more powerful — ad blockers, translators, video downloaders, etc. — all written in JavaScript.

(Inner voice: unless you’re an outlier like me using Firefox, most developers use Chrome QQ)

Why do we want to learn Chrome Extension? Remember what I mentioned in the very first post?

V8 is an open-source JavaScript engine developed by Google. It compiles JavaScript code into machine code so the computer can execute it.

So now the question is:

“What’s the relationship between Node.js and Chrome Extension?”

In reality, the former is server-side JavaScript and the latter is browser-side JavaScript, but both use V8 to execute JavaScript. Still confused? Chrome Extensions run in the browser as JavaScript and use V8 to execute it. So you can think of Chrome Extension as a JavaScript program in the browser.

You should notice — Chrome Extensions are basically just HTML, CSS, JavaScript. So really easy for web devs to pick up.

Note
Firefox’s extension implementation differs slightly from Chrome Extension since their core engines differ. To develop Firefox extensions, see Firefox Extension.

Basically any browser based on the V8 engine can use these — Edge (Chromium-based), Arc, etc. So you can develop a Chrome Extension and use it in other browsers too.

Note
Chromium is an open-source browser project also from Google. Chromium’s core also uses V8, so they’re complementary. Chrome is based on Chromium, and the newer Edge and Arc are also based on Chromium.

Importing external packages

If you want to use npm packages in Chrome Extension, you’ll need bundling tools — Webpack, Parcel, etc. Chrome Extension itself is just plain CSS, HTML, JavaScript files. So to use npm packages, you bundle them into a single file via a bundler, then import into the Chrome Extension.

Note
Looking at it differently — it’s like running JavaScript on web pages. We use bundlers to bundle into a single file, then import into the page.

In the next post we’ll cover how to use a bundler to bundle Chrome Extensions.
