Mac: Auto-Switch Node.js Version via nvm When Opening a Project

#Node.js #nvm #zsh #oh-my-zsh #zsh-nvm #Powerlevel10k

Intro

As a Node.js dev, switching Node.js versions comes up all the time. Each project uses a different version, and it’s super easy to forget to switch. So here’s a note on auto-switching with nvm whenever you open a project on Mac.

Prerequisites

A heads up before you continue — you need to have already followed my earlier posts to set up zsh and nvm. If you haven’t, finish those first or you’ll likely hit weird issues:

  • Mac OS terminal beautification, all in one post (article in Mandarin)
  • Installing nvm on Mac OS X

If you’ve got nvm installed and zsh (oh-my-zsh) configured, you’re good to continue.

Installing zsh-nvm

As I mentioned, each project uses a different Node.js version, and your nvm default might also differ. For example, your project needs 18.17.0 but nvm defaults to 16.15.0 — easy to forget nvm use 18.17.0. zsh-nvm handles the switching for us.

The zsh-nvm docs list multiple install methods:

  • antigen
  • zplug
  • zgen
  • manual
  • as an oh-my-zsh custom plugin

I’ll go with “As an Oh My ZSH! custom plugin” since I’m using oh-my-zsh.
