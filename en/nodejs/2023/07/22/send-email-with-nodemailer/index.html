Node.js: Sending Email with Nodemailer

#Nodejs #Nodemailer #Gmail #OAuth2 #Email

Intro

A note on sending email with Nodemailer. Covers two approaches with Gmail — OAuth2, and username/password (App Password).

Prep

I’ll provide a base environment for you:

  • node-nodemailer-init

This is a simple Express scaffold with nodemailer, googleapis, and dotenv installed — everything else is Express generator boilerplate.

If you’d rather not use the template, build your own Express project and install:

Nodemailer

I wrote an earlier post on Full-stack Warrior’s Path Node.js - OAuth 2.0 & nodemailer & Gmail (article in Mandarin), but that’s pretty outdated now, so I’m rewriting.

What’s Nodemailer? It’s a Node.js package for sending email. Sites really need this — verification codes, notification emails, subscription emails — Email functionality comes up all the time.

The official docs include a fairly complete example:

Our goal is to send Email through Gmail with OAuth2, so let’s see how to do that.

First, Gmail has some limits worth knowing:

  • Daily send limit: 500
  • Recipients per email above 500
  • Max email size: 25 MB

Other things — spam, unsolicited ads, virus emails — are also not allowed. Google might lock your account, so be careful.

Sending via OAuth 2.0

Older Nodemailer tutorials used username/password. Google has since restricted that since it’s less secure, pushing OAuth2 instead. It’s more complex because you need to handle authorization, so I’ll start with the harder approach.

Create a project

Go to Google Cloud Platform and create a new project. You could reuse an old one but a new one keeps things cleaner.

Pick whatever name. I called mine “nodemailer-example”.

Wait a moment for creation…

Once created, find it in the notification or in the project picker.

Enable APIs and Services

Click the hamburger menu on the left, then “APIs & Services Enabled”.

Click “ENABLE APIS AND SERVICES” at the top.

Search “Gmail API” and select it.

Click “Enable”.

Set up OAuth consent screen

Back in “APIs & Services”, click “Credentials”.

Before creating credentials, if it’s your first time, you’ll be prompted to configure the OAuth consent screen. Click “Configure consent screen”.

Choose “External” and click “Create”.

In “Edit app registration”, fill in the required fields (others are optional):

  • App name: I went with “nodemailer-example”
  • User support email: your email
  • Developer contact info: your email

If unsure, just follow the image:

The “Scopes” page — just continue.

“Test users” page — if your app is in “Testing” state, you need to add test users who can use the app. Click “Add Users” and add your own Google account. For production, you don’t need this. I’ll skip and click “Save and continue”.

Once you reach “Summary”, OAuth consent screen setup is done.

Create credentials

Back to “Credentials”, click “Create Credentials”, then “OAuth client ID”.

Choose “Web application” for application type, then:

  • Name: I used “nodemailer-example-web”
  • Authorized redirect URIs: http://localhost:3000/auth/google/callback

Note
In an earlier post, I mentioned using https://developers.google.com/oauthplayground/ for this. We’re using a different approach now.

Click “Create” — you’ll get a Client ID and Client Secret. Don’t leak them. If they’re misused, Google may lock your account. (You can also download a JSON file with both values.)

That wraps up the Gmail OAuth setup. Now to the code.

Sending via Username/Password (App Password)

Now another approach. Why?

The OAuth approach has a limitation — if today’s logged-in user isn’t A but B, you can’t send because the Token and Refresh Token belong to A. So I want to show another way — hardcode the username and password to bypass this.

Set up Google

Go to Google Account settings, click “Security”.

If you don’t have 2-Step Verification enabled, do that first or you can’t continue.

Note
Tested with a new account — Google now requires 2-Step Verification.

With 2-Step Verification on, click into it, enter your password, scroll down to find “App passwords”.

Click in and choose:

  • App: “Mail”
  • Device: “Other (Custom name)”

It’ll prompt for a name — name it “nodemailer-example” and click “Generate”.
