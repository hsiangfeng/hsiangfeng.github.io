Intro

A note on sending email with Nodemailer. Covers two approaches with Gmail — OAuth2, and username/password (App Password).

Prep

I'll provide a base environment for you:

  • node-nodemailer-init

This is a simple Express scaffold with nodemailer, googleapis, and dotenv installed — everything else is Express generator boilerplate.

If you'd rather not use the template, build your own Express project and install:

bash
npm install nodemailer googleapis dotenv

Nodemailer

I wrote an earlier post on Full-stack Warrior's Path Node.js - OAuth 2.0 & nodemailer & Gmail (article in Mandarin), but that's pretty outdated now, so I'm rewriting.

What's Nodemailer? It's a Node.js package for sending email. Sites really need this — verification codes, notification emails, subscription emails — Email functionality comes up all the time.

The official docs include a fairly complete example:

js
"use strict";
const nodemailer = require("nodemailer");

const transporter = nodemailer.createTransport({
  host: "smtp.forwardemail.net",
  port: 465,
  secure: true,
  auth: {
    // TODO: replace `user` and `pass` values from <https://forwardemail.net>
    user: 'REPLACE-WITH-YOUR-ALIAS@YOURDOMAIN.COM',
    pass: 'REPLACE-WITH-YOUR-GENERATED-PASSWORD'
  }
});

// async..await is not allowed in global scope, must use a wrapper
async function main() {
  // send mail with defined transport object
  const info = await transporter.sendMail({
    from: '"Fred Foo 👻" <foo@example.com>', // sender address
    to: "bar@example.com, baz@example.com", // list of receivers
    subject: "Hello ✔", // Subject line
    text: "Hello world?", // plain text body
    html: "<b>Hello world?</b>", // html body
  });

  console.log("Message sent: %s", info.messageId);
  // Message sent: <b658f8ca-6296-ccf4-8306-87d57a0b4321@example.com>
}

main().catch(console.error);

Our goal is to send Email through Gmail with OAuth2, so let's see how to do that.

First, Gmail has some limits worth knowing:

  • Daily send limit: 500
  • Recipients per email above 500
  • Max email size: 25 MB

Other things — spam, unsolicited ads, virus emails — are also not allowed. Google might lock your account, so be careful.

Sending via OAuth 2.0

Older Nodemailer tutorials used username/password. Google has since restricted that since it's less secure, pushing OAuth2 instead. It's more complex because you need to handle authorization, so I'll start with the harder approach.

Writing Gmail OAuth code

I provided the base environment earlier, so we can dive right in.

Note Make sure nodemailer, googleapis, and dotenv are installed, or you'll hit errors.

Create .env

In the project root, create a .env:

env
CLIENT_ID=your Client ID
CLIENT_SECRET=your Client Secret
REDIRECT_URI=http://localhost:3000/auth/google/callback

These come from the credential we just created. Or download the JSON file — it has everything:

JSON

Create the Google OAuth2 Client

Create a config folder in the root, with a file googleOAuth2Client.js:

js
require('dotenv').config();

const { google } = require('googleapis');

const googleOAuth2Client = new google.auth.OAuth2(
  process.env.CLIENT_ID,
  process.env.CLIENT_SECRET,
  process.env.REDIRECT_URI
);

module.exports = googleOAuth2Client;

We use googleapis to create the OAuth2 Client. google.auth.OAuth2 takes:

  • CLIENT_ID
  • CLIENT_SECRET
  • REDIRECT_URI

Build the Google OAuth2 URL

Create routes/auth.js:

js
var express = require('express');
var router = express.Router();

const googleOAuth2Client = require('../config/googleOAuth2Client');

const SCOPES = [
  'https://mail.google.com/',
];

router.get('/login', (req, res) => {
  const authUrl = googleOAuth2Client.generateAuthUrl({
    access_type: 'offline',
    scope: SCOPES,
  });
  res.redirect(authUrl);
});

router.get('/google/callback', async (req, res) => {
  const code = req.query.code;
  try {
    const { tokens } = await googleOAuth2Client.getToken(code)

    googleOAuth2Client.setCredentials(tokens);
    req.session.tokens = tokens;

    res.redirect('/email/user');
  } catch (err) {
    console.error('Error authenticating with Google:', err);
    res.status(500).send('Error authenticating with Google');
  }
});

module.exports = router;

In app.js, add:

js
var authRouter = require('./routes/auth');

// ... lots of other code omitted

app.use('/auth', authRouter);

Now you can hit http://localhost:3000/auth/login to get the OAuth2 URL — you'll be redirected to Google's login page.

Google login

Pick an account. You'll get a warning screen — don't worry, it's your own app. Click "Advanced" then "Go to nodemailer-example (unsafe)".

Unsafe app

Click "Allow".

Allow

You'll be redirected back to http://localhost:3000/auth/google/callback with a code. The code is used to get tokens, which authenticate you.

We store tokens in session so we can use them later.

You'll likely hit this error:

bash
Error authenticating with Google: TypeError: Cannot set properties of undefined (setting 'tokens')

This is because Express generator doesn't enable session by default. Add express-session:

bash
npm install express-session

In app.js:

js
var session = require('express-session');

// ... lots of other code omitted

app.use(session({
  secret: 'keyboard cat',
  resave: false,
  saveUninitialized: true,
  cookie: { secure: false }
}))

This fixes the "TypeError: Cannot set properties of undefined" error.

Note One subtle gotcha — the docs example uses secure: true. In local dev, your session won't persist with that — it gets cleared on every page navigation. I'm using secure: false so session works in local dev.

Build the Nodemailer OAuth2 transport

In routes/index.js, set up Nodemailer:

js
router.post('/user/send', (req, res) => {
  const {
    refresh_token,
    access_token,
  } = req.session.tokens;
 

  const transporter = nodemailer.createTransport({
    service: 'gmail',
    auth: {
      type: 'OAuth2',
      user: 'sender Gmail',
      clientId: process.env.CLIENT_ID,
      clientSecret: process.env.CLIENT_SECRET,
      refreshToken: refresh_token,
      accessToken: access_token,
    },
  });

  const mailOptions = {
    from: 'sender Gmail',
    to: 'recipient',
    subject: 'Email subject',
    text: 'Email body',
  };

  transporter.sendMail(mailOptions, (err, info) => {
    if (err) {
      console.error(err);
      res.status(500).send('Error sending email');
    } else {
      console.log(info);
      res.send('Email sent');
    }
  });
});

module.exports = router;

Nodemailer setup is done. You can send emails from the page now.

Sending via Username/Password (App Password)

Now another approach. Why?

The OAuth approach has a limitation — if today's logged-in user isn't A but B, you can't send because the Token and Refresh Token belong to A. So I want to show another way — hardcode the username and password to bypass this.

Many people who've tried username/password with Gmail have hit:

bash
Error: Invalid login: 535-5.7.8 Username and Password not accepted. Learn more at 535 5.7.8  https://support.google.com/mail/?p=BadCredentials k17-20020aa78211000000b00682562bf477sm3695625pfi.82 - gsmtp

Because Google restricted username/password sending. Used to be you'd just turn on "Less secure app access" in Google settings — but that's been removed.

So I'll teach another way.

Update .env

Add to your .env:

env
GMAIL_USER=sender Gmail
GMAIL_PASS=sender Gmail password

Next, we configure Google — but it's not as complex as before.

Go to Google Account settings, click "Security".

Write the code

Time to write code. Fill in your sender email and the App Password into .env:

env
GMAIL_USER=example@gmail.com # don't worry, this is fake
GMAIL_PASS=d81ydvsldqiwdhnc # don't worry, this is fake

Note The generated app password might have spaces — strip them. GMAIL_USER is your Gmail, GMAIL_PASS is the App Password.

In routes/email.js, paste:

js
router.post('/server/send', async (req, res, next)=> {
    const transporter = nodemailer.createTransport({
      service: 'gmail',
      auth: {
        user: process.env.GMAIL_USER,
        pass: process.env.GMAIL_PASS,
      },
    });

    await transporter.verify();

    const mailOptions = {
      from: process.env.GMAIL_USER,
      to: 'recipient@example.com',
      subject: 'Email subject',
      text: 'Email body',
    };

    transporter.sendMail(mailOptions, (err, info) => {
      if (err) {
        console.error(err);
        res.status(500).send('Error sending email');
      } else {
        console.log(info);
        res.send('Email sent');
      }
    });
});

You're sending emails now — not through OAuth2 but through username/password — without the limitation I mentioned.
