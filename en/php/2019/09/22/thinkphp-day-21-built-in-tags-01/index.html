Intro

We've actually used a few of ThinkPHP's built-in template tags in the earlier chapters — volist being the obvious one — but I never really stopped to explain what they do. So I'm starting a small series on the built-in tags. Let's kick it off with the looping tag everyone reaches for first: volist.

The volist tag

We've used this loop tag a couple of times already. The basic usage looks like this:

html
{volist name="user" id="list"}
<ul class="list-group">
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

volist grabs the data your backend passed over via name, then uses id to store and output each row. There are a few other parameters too. Say you want the output to start from the second record — you'd use offset to set the starting point. And if you want to limit it to just one record (my table only has four rows right now), that's what length is for:

html
{volist name="user" id="list" offset="2" length="1"}
<ul class="list-group">
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

volist has another nice feature: empty. When there's no data, it skips the loop entirely and outputs whatever you set in empty instead:

html
{volist name="user" id="list" offset="2" length="1" empty="No data"}
<ul class="list-group">
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

The catch is that empty won't take HTML directly. If you want to pass HTML in, you have to send empty from the controller via assign(), like this:

php
$this->assign('empty','<span>No data</span>')

Then on the frontend you just receive it with $empty:

html
{volist name="user" id="list" offset="2" length="1" empty="$empty"}
<ul class="list-group">
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

What about the KEY? By default, if you want to output the key in ThinkPHP, you use $i. You only need a different syntax if you explicitly set key:

html
{volist name="user" id="list" key="index"}
<ul class="list-group">
  <li class="list-group-item">KEY:{$index}</li>
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

If you don't set key, just use the default $i:

html
{volist name="user" id="list"}
<ul class="list-group">
  <li class="list-group-item">KEY:{$i}</li>
  <li class="list-group-item active">Name:{$list.name}</li>
  <li class="list-group-item">Weight:{$list.weight}</li>
  <li class="list-group-item">Height:{$list.height}</li>
</ul>
{/volist}

The for tag

The for tag is more straightforward, but watch out — this loop can't receive backend data. It's just a plain loop. Say you want to output 1 through 100, you'd write it like this:

html
{for start="1" end="100" name="item"}
  {$item}
{for/}

I won't bother demoing this one. If you only want the even numbers, add step — step defaults to 1:

html
{for start="1" end="100" step="2" name="item"}
  {$item}
{for/}

step controls how the counter increments. Set it to 2 and the loop adds +2 each time, and so on.

The foreach tag

Last up is another tag that's a lot like volist: the foreach tag. It's very similar to volist, just with slightly different syntax. I'll rewrite the earlier volist example as a demo:

html
<div class="container">
  <h1>{$title} - foreach tag</h1>
  {foreach $user as $vo}
  <ul class="list-group">
    <li class="list-group-item">KEY:{$vo.id}</li>
    <li class="list-group-item active">Name:{$vo.name}</li>
    <li class="list-group-item">Weight:{$vo.weight}</li>
    <li class="list-group-item">Height:{$vo.height}</li>
  </ul>
  {/foreach}
</div>

One thing foreach can do is talk to the Model directly for data, skipping the controller entirely:

html
<div class="container">
  <h1>{$title} - foreach tag</h1>
  {foreach :model(user)->all() as $vo}
  <ul class="list-group">
    <li class="list-group-item">KEY:{$vo.id}</li>
    <li class="list-group-item active">Name:{$vo.name}</li>
    <li class="list-group-item">Weight:{$vo.weight}</li>
    <li class="list-group-item">Height:{$vo.height}</li>
  </ul>
  {/foreach}
</div>

Personally though, I don't recommend doing this. It tends to cause maintenance headaches down the line, so I'd still go with volist.

Wrap-up

This post covered the three looping tags ThinkPHP gives you: volist, for, and foreach. In practice though, you'll mostly just reach for volist — as you can see from the examples above, it's more capable and more convenient. For a more detailed rundown, check out the loop tags chapter in the official manual (page in Mandarin).
