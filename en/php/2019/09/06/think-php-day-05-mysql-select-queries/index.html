Day5-Learning ThinkPHP from Scratch-MySQL Basics-Data Part 2 (SELECT Queries)

#MySQL #PHP #ThinkPHP 5.1 #Learning ThinkPHP from Scratch #11th iThome Ironman Contest

Intro

Last time I only covered INSERT, UPDATE, and DELETE syntax, but there was one thing I deliberately skipped: SELECT. The reason is that querying actually has a ton of possible combinations, so I wanted to pull it out into its own post. Enough talk — let me show you why this one deserved to be split out.

Querying all columns

First, let’s start with the most basic and simplest syntax: querying every column of data in a table.

Command breakdown: SELECT * FROM [table name];

If you’ve been following along from the last post, you should normally only have one row in the result ↓

Querying

Now things get a little trickier, but this is also the heart of querying: the conditional query. A conditional query usually comes with a where followed by a condition, and depending on the condition you tack on, you’ll get different results.

Before we get hands-on, though, I’d suggest running the following command first to quickly drop in a few sample rows, so we can actually see the effects later:

So at this point our table should have four rows in it ↓

Conditional query

First up, the basic conditional query. Say we want to find only people named 王小明 — the syntax would look like this:

Command breakdown: SELECT * FROM [table name] WHERE [condition];

After running it, you’ll see the result only returns people named 王小明 ↓

Conditional query with AND

If you want the result to be a bit more focused (more specific) — say you only want to find someone who weighs exactly 70.5 kg and happens to be named 王小明 — we can use and to attach another condition and narrow things down:

Command breakdown: SELECT * FROM [table name] WHERE [condition] and [condition];

That way you’ll find the person named 王小明 who also weighs 70.5 kg ↓

Conditional query with OR

When we want to query people named both 陳春嬌 and 王小明 at the same time, we can write it like this:

Command breakdown: SELECT * FROM [table name] WHERE [condition] or [condition];

And if one of them doesn’t exist, it won’t break the query — it’ll just list whatever does match. So if there were no 陳春嬌 but there was a 王小明, it would just list 王小明.

Querying a specific number of rows

What if I only want to grab, say, rows 3 through 4 when querying? That’s where the LIMIT syntax comes in:

Command breakdown: SELECT * FROM [table name] LIMIT [value1],[value2];

Sorting query results

Finally, let’s talk about sorting. Sorting is a super common thing to do, and it mainly comes in two modes:

  • Ascending (ASC)
  • Descending (DESC)

Sorting query results in ascending order

Sorting usually involves one piece of syntax: ORDER BY. The default sort is ascending, so let’s try sorting by height:

Command breakdown: SELECT * FROM [table name] ORDER BY [column];

You can see the heights go from smallest to largest ↓

That said, I personally like to spell it out explicitly to avoid any weird surprises. Here’s the full ascending syntax:

Command breakdown: SELECT * FROM [table name] ORDER BY [column] ASC;

And of course the result is the same ↓

Sorting query results in descending order

So what about descending order? Just swap the ASC from before for DESC:

Command breakdown: SELECT * FROM [table name] ORDER BY [column] DESC;

And now you’ll see it go from largest to smallest.

Wrap-up

Let’s wrap up by rounding up all the SQL commands we covered in this post.

Operation SQL syntax
Query all columns SELECT * FROM [table name];
Conditional query SELECT * FROM [table name] WHERE [condition];
Conditional query AND SELECT * FROM [table name] WHERE [condition] and [condition];
Conditional query OR SELECT * FROM [table name] WHERE [condition] or [condition]
Conditional query (operator) SELECT * FROM [table name] WHERE [column] > [condition];
Query specific number of rows SELECT * FROM [table name] LIMIT [value1],[value2];
Sort results ascending SELECT * FROM [table name] ORDER BY [column] ASC;
Sort results descending SELECT * FROM [table name] ORDER BY [column] DESC;
