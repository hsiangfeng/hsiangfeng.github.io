Deployment gotchas

You’ll probably run into a few snags along the way, so here are the fixes.

CSS / JS files disappear

The main culprit is a .gitignore sitting inside public/static. Delete it, redeploy to Heroku, and you’re good.

After seeding: Call to undefined function mb_substr()

Open composer.json and add "mbstring": "*" under require, then redeploy. The reason is that the Heroku environment doesn’t have this extension, so we use composer.json to tell Heroku we need it.

After adding "mbstring": "*", remember to run composer update — otherwise pushing to Heroku won’t actually change anything.

The captcha vanishes

You’ll notice the captcha just refuses to show up:

And if you visit the captcha route directly, you’ll get this error: Call to undefined function think\captcha\imagecreate().
