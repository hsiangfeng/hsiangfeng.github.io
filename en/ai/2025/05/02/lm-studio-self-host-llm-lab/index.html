Build Your Own AI Language Model Lab — No Help Needed, with LM Studio

#AI #ChatGPT #LLM #LM Studio #Language Models

Intro

Most of us have been bombarded by AI, ChatGPT, and other large language models for a while now, and we’ve all been using web services like ChatGPT or Claude.AI. The upside of these services is that they’re easy to start with and convenient. The downsides? You need an internet connection, there are data privacy concerns, and there are API costs. So is there a way to self-host AI locally? Yep — and this post introduces a really handy tool for that called LM Studio.

What is LM Studio?

LM Studio is a tool built specifically to make running LLMs (large language models) locally easy. The standout feature is that you can run all kinds of different LLM models locally and easily tweak and test them — no need to depend on online services anymore.

The whole interface is in English, but it’s actually pretty simple to operate. So as long as you understand a few basic concepts, even total beginners can spin up their own Chat Bot locally.

Note
This post uses Mac as the demo environment. The Windows interface is basically the same.

Installing LM Studio

Installation is easy. Just head over to the LM Studio website to download the version for your platform.

I won’t go into the install process — it’s just clicking through, nothing complicated.

Using LM Studio

When you first open LM Studio, you’ll basically see this screen:

Don’t worry if it looks unfamiliar at first — let’s start with the left sidebar:

  • Chat: where you chat with the AI — basically the ChatGPT equivalent.
  • Developer: a slightly more advanced section where you can turn your selected model into an API. You can use Python, Node.js, Go, etc. to build with it.
  • My Models: where the models you’ve downloaded live.
  • Discover: where you discover and explore models. You can download all kinds of models here — there are recommended ones, popular ones, and so on.
  • Downloads: shows the progress of the models you’re downloading, plus your download history.

Getting ready to download a model

If you skip ahead and try to chat with the bot before reading the rest of this post, you’ll get this error:

Because we haven’t downloaded any large language model yet.

So click on the Discover icon and a window will pop up.

Before downloading, we need to get familiar with a few things — model parameters.

Model parameters?

In the model list, you’ll see each model has these mysterious letters and numbers after it. Let me pick a few to explain:

  • Qwen3 235B A22B (MoE)
  • Qwen3 4B
  • Gemma 34B QAT
  • DeepSeek R1 Distill (Qwen 7B)
  • Llama 3.3 70B Instruct

Take Qwen3 4B first. Qwen3 is the model name and 4B is the parameter count — meaning this model has 4 billion parameters. The more parameters, the more information the model can learn, and the better it usually performs.

Common codes:

  • B: stands for Billion. So 4B is 4 billion parameters.
  • M: stands for Million. So 4M is 4 million parameters.
  • T: stands for Trillion. So 4T is 4 trillion parameters.

So… what about MoE and QAT?

  • MoE: Mixture of Experts. You can think of it as the model bringing in a bunch of experts to help out, which lets the model perform better on certain tasks. But these models are usually larger because they need to store all the expert sub-models.
  • QAT: Quantization Aware Training. These models are designed to be compressed from the start, taking less memory and running inference faster, while not losing too much accuracy.
  • Instruct: this model has been Fine-Tuned, usually for specific tasks, so it performs better on those tasks.
  • Distill: this model has been distilled — usually a smaller model derived from a larger one by extracting the essential knowledge. It’s smaller and faster, but the trade-off is it might lose some knowledge.

So when we see Qwen3 4B, we know this model is called Qwen3, has 4 billion parameters, and has no MoE/QAT/Instruct/Distill markers — so it’s the original model.

But there’s one really interesting name in the list: Qwen3 235B A22B (MoE). This means the model has 235 billion parameters and is a multi-expert collaboration model, but only 22B of those parameters are active during operation — to save resources. To put it more simply: we have 235 billion books, but each time we only borrow 22 books to read. We don’t have to flip through all 235 billion books every time, which saves resources. So A22B means the model’s active parameter count (Active 22B).

What else should I watch out for?

Before downloading a model, you should also click into the Model Card. There you’ll find the model’s introduction, usage suggestions, and so on.

For example, let me pick DeepSeek R1 Distill (Qwen 7B).

A few key things to look at:

  • DeepSeek R1 Distill (Qwen 7B)
  • Q4_K_M DeepSeek R1 Distill Qwen 7B 4.68B

This model name is a bit unusual — DeepSeek R1 Distill (Qwen 7B). So how do we read it?

  • DeepSeek: model name
  • R1: this is the R1 version
  • Distill: this is a distilled model
  • Qwen 7B: this model was distilled from Qwen 7B

In simple terms: DeepSeek went and learned from Qwen 7B, then extracted Qwen 7B’s knowledge and condensed it into a smaller version. So this model is smaller and faster, but still keeps most of its original brains.

Q4_K_M represents the model’s quantization method.

What is quantization?

If you click into Q4_K_M, you’ll see even more of these mysterious numbers.

  • Q3_K_L
  • Q4_K_M
  • Q6_K
  • Q8_0

So what do these mysterious numbers mean? In short, they represent the quantization method for the model. Breaking down Q4_K_M as an example:

  • Q4: this is the 4-bit ultra-small version — generally smaller means less memory usage
  • K_M: refers to the compression method
    • K_L: less compression, retains the most original model detail, best results
    • K_M: medium compression, fast but still decent performance (commonly recommended version)
    • K_S: most compressed, fastest
    • K_0: closer to the original, but still slightly compressed
    • _0: uses the most basic quantization method, results close to original, but still quantized

There are obviously a lot of details I’m leaving out here — I’m just giving you enough understanding to quickly download what you want.

Download and start using

Now that we’re familiar with the basics, we can actually download a model.

In the Search field, type DeepSeek R1 Distill (Qwen 7B), pick the Q4_K_M version, and click Download.

Wait for the download to finish.

Once it’s done, click Load Model.

After clicking, you’ll jump to the Chat page, and you can see at the top that the Model is loading (the loading bar at the top).

Once it finishes, you can chat with it.

How do you exit or stop the model? Easy — just click Eject at the top.

To remount it, just click “Select a model to load” and a window will pop up for you to pick a model.

That’s it for this intro. The above is the basic usage of LM Studio — letting you quickly build your own AI chatbot.
