Don't Want to Use the Lobster (OpenClaw)? Try Claude Code Channels Instead!

Intro

If you've been keeping up with AI Agent tools, you've probably heard of the lobster (OpenClaw). In short, OpenClaw lets you remotely control an AI Agent on your computer through chat platforms. Claude Code recently launched a similar feature called "Channels".

The Channels concept is really similar to OpenClaw — both let you send messages to a running Claude Code Session via external platforms (Telegram, Discord, etc.) so Claude can handle tasks in the background. The difference is Channels is an official Anthropic feature, so stability and security are more guaranteed — and no need to worry about getting flagged for TOS violations.

So this post walks through what Channels are, how to set them up, and how to actually use them.

What is Claude Code Channels?

In short, a Channel is essentially an MCP Server that pushes external events (messages, notifications, Webhooks, etc.) into your running Claude Code Session — letting Claude react to these events even when you're not at the terminal.

And Channels is bidirectional — Claude can not only receive messages but also reply through the same Channel. For example, you send a message to your Bot on Telegram, Claude processes the request in the terminal, then pushes the result back to Telegram, and you can see the result on your phone directly.

Note: when Claude replies through a Channel, in your terminal you'll only see inbound messages and tool call confirmations (like "sent") — the actual reply content shows up on the other platform (e.g., Telegram or Discord).

But events only arrive while the Session is open, so if you want 24/7 message reception, you need to keep Claude Code running in a background process or a persistent terminal.

Also: Channels is currently in Research Preview. It needs Claude Code v2.1.80 or above and only supports claude.ai login — not Console or API Key auth. For Team or Enterprise plans, the admin needs to enable it first.

What do you need before starting?

Before setup, confirm the following:

  • Claude Code v2.1.80 or above (check with claude --version)
  • A claude.ai account (must log in via /login — API Key not supported)
  • Bun installed (Channel Plugin runs on Bun, check with bun --version)

If these aren't ready, the next steps won't go smoothly.

Installing Bun

Channel Plugins run on Bun, so install it first. If you've already installed it, skip this section.

macOS / Linux / WSL:

bash
curl -fsSL https://bun.sh/install | bash

Windows PowerShell:

powershell
powershell -c "irm bun.sh/install.ps1 | iex"

After installation, verify with:

bash
bun --version

If a version number prints out correctly, install was successful.

Try fakechat first to get a feel

Before officially setting up Telegram or Discord, we can use the official fakechat to get a feel for how Channels work. fakechat is a chat UI running on localhost — no external services or auth needed, perfect for testing.

First, start a Claude Code Session, then run this command to install the fakechat Plugin:

bash
/plugin install fakechat@claude-plugins-official

Note After typing this, restart Claude Code so the Plugin loads correctly — otherwise the subsequent setup commands might not be found.

Install screen

After installation, exit Claude Code, then restart with the --channels parameter:

bash
claude --channels plugin:fakechat@claude-plugins-official

This auto-starts the fakechat server.

fakechat

fakechat

Open your browser and go to http://localhost:8787. You'll see a simple chat interface — try typing a message like:

text
Hello there

fakechat

The message gets pushed to your Claude Code Session. Claude reads the message, processes it, and uses fakechat's reply tool to push the result back to the chat interface — you'll see the result in the browser.

Note If Claude hits a permission prompt during execution, the Session pauses until you manually approve it in the terminal. If you want fully unattended operation, use --dangerously-skip-permissions to skip permission prompts — but only use this in environments you trust.

Setting up a Telegram Channel

After getting a feel for fakechat, let's set up an actual Telegram Channel.

First, open Telegram, search for BotFather, and send /newbot. Follow the prompts to set a display name and a username ending in bot. Once done, BotFather sends you a Token — copy it.

To make sure everyone follows along, here are the steps in detail:

Search @BotFather and create a new Bot. You'll see this initially:

BotFather

Note Make sure it has the verified blue checkmark — that's the real BotFather, not the fake ones I've crossed out below.

BotFather

Then type /newbot to create a new Bot. BotFather first asks "Alright, a new bot. How are we going to call it? Please choose a name for your bot." This name can be anything — you can change it later, so don't sweat it.

Then it asks "Good. Now let's choose a username for your bot. It must end in bot." This username is the unique identifier on Telegram and can't conflict with other Bots, so you might need a few tries to find an available one.

BotFather

Finally you get the HTTP API Token — a long string of letters, numbers, and colons. Save it somewhere (and don't share or leak it).

BotFather

Back in Claude Code, install the Telegram Plugin:

bash
/plugin install telegram@claude-plugins-official

Note After typing this, restart Claude Code so the Plugin loads correctly — otherwise the subsequent setup commands might not be found.

After installation, configure with the Token from BotFather:

bash
/telegram:configure <token>

This stores the Token in the project's .claude/channels/telegram/.env. Alternatively, you can set the TELEGRAM_BOT_TOKEN environment variable in your Shell before starting Claude Code, so you don't need to run configure each time.

After configuring, exit Claude Code and restart with --channels:

bash
claude --channels plugin:telegram@claude-plugins-official

This starts the Telegram Plugin and begins polling messages for your Bot.

Open Telegram, send any message to your Bot, and the Bot will reply with a pairing code.

Telegram Bot

Note If your Bot isn't responding, confirm Claude Code is running with --channels — the Bot can only reply when the Channel is active.

After getting the pairing code, type in Claude Code:

bash
/telegram:access pair <code>

Pairing code

Then lock down access to make sure only your account can send messages:

bash
/telegram:access policy allowlist

Setup done! You can now send commands to Claude Code directly through Telegram.

Setting up a Discord Channel

If you use Discord more, you can set up a Discord Channel instead. Steps are similar to Telegram, with some Discord-specific config.

Note Quick tip — I'd suggest creating your own Discord server first to make follow-up easier.

First, head to the Discord Developer Portal, click "New Application", and set a name.

Find "Bot" in the sidebar.

Click "Reset Token" and copy the Token.

In the Bot settings, scroll down to "Privileged Gateway Intents" and enable "Message Content Intent". This is important — without it, the Bot can't read message content.

Find "OAuth2" in the sidebar.

Go to "URL Generator", check the bot scope, and enable these permissions:

  • View Channels
  • Send Messages
  • Read Message History
  • Send Messages in Threads
  • Attach Files
  • Add Reactions

Permissions

Open the generated URL and add the Bot to your server.

Add to server

Add to server

Back in Claude Code, install the Discord Plugin:

bash
/plugin install discord@claude-plugins-official

Note After typing this, restart Claude Code so the Plugin loads correctly — otherwise the subsequent setup commands might not be found.

After installation, configure the Token:

bash
/discord:configure <token>

Like Telegram, the Token gets stored in .claude/channels/discord/.env, and you can use the DISCORD_BOT_TOKEN env var instead.

After configuring, exit Claude Code and restart with --channels:

bash
claude --channels plugin:discord@claude-plugins-official

From the Discord server you invited the Bot to, send any message to the Bot, and it'll reply with a pairing code.

Conversation

Conversation

Back in Claude Code, complete the pairing:

bash
/discord:access pair <code>

Note I hit an issue where the pairing code kept expiring, so I switched to /discord:access allow <senderId> to add my Discord sender ID directly to the allowlist. If you don't know how to get your Discord ID, see this article: How to easily get your own Discord ID (article in Mandarin)

Then lock down access too:

bash
/discord:access policy allowlist

Done!

If you want to enable multiple Channels at once, separate the Plugins with spaces after --channels:

bash
claude --channels plugin:telegram@claude-plugins-official plugin:discord@claude-plugins-official

Channels security

You might worry whether Channels has security issues — let me briefly explain.

Each approved Channel Plugin maintains a sender Allowlist. Only IDs you've manually added to the Allowlist can push messages — other people's messages get dropped directly. So you don't have to worry about people sending random messages to mess with you.

Telegram and Discord both use a pairing-code mechanism to build the Allowlist:

  1. Send any message to your Bot on Telegram or Discord
  2. Bot replies with a pairing code
  3. In Claude Code, type the pairing code to approve
  4. Your sender ID gets added to the Allowlist

Beyond that, you can use the --channels parameter to control which Channels each Session enables. Even if an MCP Server is in .mcp.json, if it's not in --channels, it can't push messages.

So overall, security is multi-layered: Plugin Allowlist + pairing code verification + Session-level enable control. No need to worry too much.

Channels vs Remote Control vs OpenClaw (the lobster)

You might be curious how Channels differs from the previously-introduced Remote Control and OpenClaw. Quick comparison:

ItemChannelsRemote ControlOpenClaw (the lobster)
Core conceptExternal events pushed into Claude CodePhone/browser directly controls Claude CodeControl your computer through a chat platform
Supported platformsTelegram, Discord (extensible)claude.ai/code, Claude AppVarious chat platforms
Bidirectional communicationSupportedSupportedSupported
Official supportAnthropic officialAnthropic officialThird-party
Use caseWant to push events or messages through chat platformsWant to directly control a Claude Code SessionWant to use a chat platform to control your whole computer

In short:

  • Remote Control suits situations where you want to "directly control" a Claude Code Session — like holding a remote.
  • Channels is more "event-driven" — like pushing CI results, monitoring alerts, or chat messages in for Claude to handle automatically.
  • OpenClaw (the lobster) is a third-party solution — more general-purpose features, but no official stability guarantees.
