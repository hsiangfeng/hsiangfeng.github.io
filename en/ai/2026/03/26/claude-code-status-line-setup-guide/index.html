Claude Code Status Line Setup: Monitor Context Usage and Rate Limit in Real-Time

#Claude Code #AI #AI 開發 #Claude Code 設定 #Status Line #Context #Rate Limit #終端機

Introduction

After using Claude Code for a while you’ve probably hit this — mid-conversation your Context gets compacted, or your Rate Limit silently gets used up, and you only find out when Claude Code tells you “you’ll need to wait a bit”.

Personally I used to run into this all the time. Context would suddenly get compacted, Claude would forget what we’d just discussed, and I’d have to re-explain everything. Pretty wasteful. The root cause is that Claude Code doesn’t show this info on screen by default — you have no idea how much Context is left or how much of your Rate Limit you’ve burned through.

Good news though: Claude Code has a built-in feature called Status Line that puts this info at the bottom of your terminal in real time — current model, Context usage percentage, 5-hour and 7-day Rate Limit usage, and so on. Once it’s set up, no more guessing. Let’s walk through how to configure it.

What is Status Line?

In short, Status Line is a status bar at the bottom of your terminal that shows live info about your current Claude Code session.

The mechanism is straightforward: Claude Code passes the session’s data as JSON via stdin to a script you specify, your script parses it and prints one line of text, and Claude Code displays that line at the bottom of the terminal.

So Claude Code provides the data, and you decide what to display by writing the script.

What information can it display?

The JSON Claude Code passes to the script includes the following fields:

Category Field Description
Model model.display_name Display name, e.g. Opus 4.6 (1M context)
Model model.id Model ID, e.g. claude-opus-4-6
Context context_window.used_percentage Context used percentage
Context context_window.remaining_percentage Context remaining percentage
Context context_window.context_window_size Total Context size
Rate Limit rate_limits.five_hour.used_percentage 5-hour usage percentage
Rate Limit rate_limits.seven_day.used_percentage 7-day usage percentage
Rate Limit rate_limits.five_hour.reset_at 5-hour reset time
Rate Limit rate_limits.seven_day.reset_at 7-day reset time
Cost cost.total_cost_usd Total session cost (USD)
Cost cost.total_duration_ms Total session duration
Git git.branch Current Git branch
Other cwd Current working directory
Other version Claude Code version

Note
The Rate Limit fields only carry data for Claude.ai Pro/Max subscribers. If you log in with an API key, those fields will be empty.

And more importantly, the default script is missing a few things I think are the most useful:

  • Rate Limit usage: This is where you actually trip — by the time you find out the 5-hour bucket is empty, it’s too late
  • Three-stage colour warning: The default only has dim and red, no yellow middle zone. By the time it goes red you’re often already past the point of doing something about it
  • Session cost: For developers using API keys, knowing how much the current session has cost is genuinely useful in real time

Design intent

Before writing the script I thought about which info is “only Status Line can show me”:

Info Already in p10k? Show in Status Line?
Working directory Yes Not needed
Git branch Yes Not needed
Model name No Needed
Context remaining No Needed
Rate Limit No Needed
Session cost No Needed

So my version focuses on these. I’ll also show how to add the directory and Git branch back in later.

For colours, I went with three stages:

  • Green (>= 50%): Safe, nothing to worry about
  • Yellow (21% - 49%): Heads up, start wrapping up or prepping a fresh session
  • Red (<= 20%): Danger, finish what you’re doing fast

The same logic applies to both Context and Rate Limit, so a glance tells you exactly where you stand.

What each segment does

A quick summary of each segment and its colour logic:

Segment Icon Colour Notes
Model name none Purple bold Always shown, visually distinct from the rest
Context bar \uf2db chip Green/Yellow/Red 10-cell progress bar + percentage, the most intuitive view
5h Rate Limit \uf017 clock Green/Yellow/Red Remaining quota in the rolling 5-hour window
7d Rate Limit \uf017 clock Green/Yellow/Red Remaining quota in the rolling 7-day window
Session cost \uf155 dollar Dim grey Quiet, but glance-able anytime

Note
Rate Limit and Session cost fields will have different data depending on how you log in. Pro/Max subscribers get Rate Limit data, API key users get Session cost data. The script uses // empty to check, so empty fields just don’t render — no blank segments to worry about.

Community tools

If you’d rather not write a script at all, the community has a few ready-made options:

  • ccstatusline: Supports multiple themes and Powerline styles, highly customisable
  • claude-statusline: Comes with sane defaults, install and go

These are basically wrappers that generate the script for you, just without the writing step.

Why bother setting up Status Line?

You might be thinking: what’s the point of knowing how much Context is left? Claude Code auto-compacts when it fills up anyway.

The thing is — compacting Context means Claude Code drops part of the previous conversation, and that can cause it to forget instructions or topics you discussed earlier. Then you re-explain, and the back-and-forth eats more time than the savings.

If you can see the remaining Context live, you can be proactive — run /compact when it’s getting close to full, or open a fresh session, to avoid losing context.

Same goes for Rate Limit. If you know the 5-hour window is running out, you can prioritise the important task or pace differently, instead of getting blocked unexpectedly.

Wrap-up

Status Line is a small but genuinely useful feature. Once it’s set up you barely notice it’s there, but when you need it, a glance tells you the current state — no more guessing.

If you’re using Claude Code daily like I am, it’s worth the few minutes to get this dialled in.
