Learning Python From a JavaScript Angle (4) - Types and Variables

#JavaScript #Python #iThome Ironman Contest #Learning Python From a JavaScript Angle

Intro

Okay, we’re finally writing some actual Python. We’ve messed around for three days now, and if I don’t start soon people are going to start unsubscribing. But before we jump into writing code, we’ll kick off with the most basic thing there is: types. And as always, I’ll lean on the syntax differences between Python and JavaScript so we can learn both at once.

Types

Let’s start with a quick look at how types work in Python…

Hey, don’t put it like that. I’d honestly love to slack off for a few more posts, but it’s about time we got to know how Python is actually written. And before you write anything, you’ve got to learn the basics first, right?

So before that, let’s quickly recap which primitive types JavaScript has:

  • String
    • 'this is a string'
    • "this is also a string"
  • Boolean
    • true
    • false
  • Number
    • 100000
    • 3.141592653
  • Undefined
  • Null

Plus the ones added later:

  • Bigint
    • 9007199254740991n
  • Symbol

Everything outside of these we call object types, so you’ll often hear people talk about primitive types versus object types.

If you’re not so solid on JavaScript’s type concepts, I’d suggest reading my earlier post “JavaScript Core Concepts (14) - Operators, Types and Grammar - Primitive and Object Types (article in Mandarin)” for a more detailed explanation.

So what about Python? Python’s types mainly fall into three big categories:

  • Numeric types
  • String types
  • Container types

Pretty interesting, right? It splits straight into three categories, and the container type especially is something special.

Let’s start with the ones closest to JavaScript’s types: the “numeric type” and “string type”. We’ll save container types for another chapter.

I won’t deny that Python and JavaScript do define types a bit differently, so let’s look at the “numeric type” and “string type” first. Even though I say numeric and string types, each of these actually breaks down into more specific types underneath:

  • Numeric types
    • int - integer
    • floats - floating point
    • boolean - boolean
  • String types
    • string - it’s a string, what else do you want?

(Just kidding above.)

The string type is no different from JavaScript’s String type, but the most obvious difference is in the “numeric type”. In JavaScript, whether it’s var num = 1.23456; or var num2 = 123;, we lump them all into the Number type. We can verify that too:

In Python though, there’s a distinction between the int type and the floats type. How do you tell them apart? Just look at whether there’s a decimal point.

If there’s a decimal point, it’s a float. So var num = 1.23456; is actually a float (floats) in Python, and var num2 = 123; is an integer (int).

Variables are basically the most fundamental, most important opening move in any programming language, because we constantly need them to store data. A classic JavaScript example would be storing the data that comes back from an AJAX call.
