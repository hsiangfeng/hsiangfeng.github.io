Intro

Modules

Or the module.exports syntax that Node.js developers reach for all the time:

These days the more famous one is probably ESM (ES6 Modules, a.k.a. JavaScript Modules). If you’re not familiar with ESM, you might want to check out an earlier post of mine, What is ESM (ES6 Modules or JavaScript Modules)? (article in Mandarin) — it goes into a lot more detail.

So here’s my quick summary of what a module is:

Basically, one module equals one file, and that file usually holds a bunch of functions — or features, if you like — that we can use. The point is we can import that file and use it from other files.

For example, say we have five Python files, and all of them need to do addition and say hello:

Doesn’t look like a big deal, right? But picture this: what if we had 100 pages that needed this? You’d be writing the same code 100 times. Sure, you might think “eh, it’s fine, I’ll just copy-paste.” Okay, say you’ve finished pasting all 100 pages. Then the next day your boss tells you to change “Hello” into Chinese — now you’re editing it 100 times again. Maybe you’re thinking “hah, I’ll just use find-and-replace!” Wow, good for you!

Importing a Module

Now for something a bit simpler — the import syntax itself.

We’ve already been using import in the examples above. And just plainly importing a specific file like that is actually one way of importing everything, so you can directly use all the functions and dictionaries inside module.py.

But import has other tricks too. Take the earlier import module — every time you want to use a function from the module, you have to write module.add or module.dic and so on.
