#JavaScript #Python #Learning Python From a JavaScript Angle #13th iThome Ironman Contest

Intro

Next up is something really, really important: handling errors. The technique for this is usually called exception handling.

The Idea Behind Exception Handling

Let’s start with what exception handling even is. When you’re building a system, you’re bound to run into error situations. The user’s network is flaky, they type in the wrong data, you forget to declare a variable, a typhoon snaps the power line mid-input and the power goes out, or the kid next door cuts your network cable while you’re online (a friend actually told me his network cable got cut by a neighborhood kid). If you don’t put some error-handling defenses in place, your system is going to lose your users’ trust the moment it goes live. So handling errors properly really matters.

To avoid these situations we usually reach for exception handling. If you’ve already started writing ES7 syntax, you’re probably very familiar with async and await, and you’ve definitely used try...catch. Here’s a quick snippet:

Throwing (raise)

The clever ones among you probably immediately thought of JavaScript’s throw. Here’s a special case. When we do AJAX create/update/delete operations, the back end might not return the matching HTTP status code. Instead it might hand you a status field to tell you whether this insert came back true or false, so this is when we use throw to kick the current state out to the exception handler.
