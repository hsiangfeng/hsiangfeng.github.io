Learning Python From a JavaScript Angle (5) - Functions

#JavaScript #Python #第 13 屆 iT邦幫忙鐵人賽 #Learn Python From JavaScript

Intro

Last time we covered types and variables, so this time there’s no way around it — we have to talk about functions. Functions are a genuinely painful sticking point for a lot of beginners, so in this post I’ll also share a simple way to think about when you should use a function, and how to practice writing them.

Functions, explained through everyday life

Let’s get into functions. Even though this series is about learning Python from a JavaScript angle, I already wrote a whole post called “A slightly long ramble about JavaScript functions (article in Mandarin)”, so to avoid going too deep on the JavaScript side, anyone curious can read that one. From here I’ll try to keep the focus on Python.

A lot of people aren’t super clear on what a function even is, or when they should use one. So let me describe it simply with an example. Basically…

“When some behaviour repeats a lot, you can wrap it up into a function.”

Yep, that’s it. That easy. That’s the heart of what you’re learning when you first pick up functions.

Okay, okay — I know right about now you’re going to ask “what counts as repeated behaviour?” Let me give you a simple scenario.

Say a week has five days, and from day one to day five, you absolutely do all of these same things in your life:

  • Wake up
  • Brush teeth
  • Exercise
  • Lunch
  • Afternoon nap
  • Afternoon exercise
  • Dinner
  • Evening shower
  • Sleep

  • str()

  • Number()
  • Boolean()

Differences between Python and JavaScript functions

Let’s talk about how Python functions differ from JavaScript functions. JavaScript functions have a few common traits:

  • You can store a function in a variable (anonymous expression)
  • You can call a function before the function statement is created (Hoisting and named function statements)
  • IIFE (Immediately Invoked Function Expression)

lambda

Now let’s talk about storing a function directly inside a variable. To do this in Python, you have to use an operator called lambda, which is what’s known as an anonymous function. As the name suggests, its traits are:

  • No need to declare a name
  • But it can only be one line of code — break it onto a new line and it errors out
