Learning Python From a JavaScript Background (2): The Origin Story

#JavaScript #Python #13th iThome Ironman Contest #Learning Python From a JavaScript Background

Intro

Every programming language has its own little birth story. JavaScript’s is honestly pretty fun (you know, the whole “dog vs hot dog” thing), so what about Python? Is it just as interesting? In this post we’ll take a quick look at where Python came from.

A bit of Python history

In Chinese, Python literally means “python” — the snake. And yes, the animal Python is named after is a snake. But don’t worry, we are absolutely not here to talk about the history of an actual snake.

This isn’t a Discovery Channel contest, it’s the iThome Ironman contest, so today we’re talking about the history of the programming language Python (times a hundred, this is important) — definitely not the history of the snake (unless you really want to learn about snakes, in which case that’s a different story).

Python was actually born pretty early. Compared to JavaScript, Python has about a 3-year head start.

(JavaScript came out in 1994, while Python landed in 1991. Fun aside: one of the languages that inspired JavaScript was actually Python.)

Python’s creator is Guido van Rossum. As for why he made it, I dug around a bit and there seem to be two versions of the story:

  • He needed something to do to pass the time
  • He thought the languages around at the time were a pain to use

As for the name “Python” — it comes from the fact that Guido van Rossum was a fan of Monty Python’s Flying Circus. So he just named it Python.

Bet you didn’t see that coming, right? The whole reason it’s called Python is purely because the author liked this comedy group. You can find this explanation over on Wikipedia too.

What makes Python special

If you’ve already learned, heard about, touched, poked at, or otherwise been near Python in any way, you’ve probably heard people say these two things about it a lot:

  • The syntax is clean and easy to read
  • It’s easy to learn

Basically the reason you hear those two things is that Python’s core design philosophy is all about being concise and readable. So for beginners it really is one of the friendlier languages to start with — to the point where I doubt anyone’s going to send you this meme:

(I have to say, at first I genuinely thought the “from beginner to giving up” meme only existed for C, Java, C# and so on. Turns out even Python has its own version!)

On top of that, Python’s backstory isn’t nearly as dramatic as JavaScript’s. If you’re a frontend developer, you’ve probably been asked the difference between Java and JavaScript more times than you can count — some people honestly have no idea how Java and JavaScript even relate to each other (just like the dog vs hot dog situation). You might even spot a confused recruiter posting a job ad looking for a “JAVA script” engineer (lol, so are you hiring a Java engineer or a JavaScript engineer???).

Wikipedia also calls out Python’s design philosophy specifically. Here’s an excerpt:

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.

Like I mentioned, because Python is so easy to pick up, a lot of online courses and schools recommend it as your very first language.

But the first programming language a friend of mine recommended I learn? C. (Truly a “from beginner to giving up” experience, QQ.)

Anyway, back on topic. At this point you might be wondering: “What can you actually use Python for?”

Basically Python can do a ton of stuff. Besides the famous web-scraping use case, it’s great for data analysis and web development too. For example, the Instagram you know today was built with the Django framework (a web framework written in Python).

Of course, you might also hear people complain that Python’s performance is bad. That mostly comes down to the nature of the language — it’s a dynamic language, so just like JavaScript, the type of a variable is only known at runtime. But realistically, we’ve all used Instagram for years and don’t really feel any lag, so I don’t think it’s a real problem. Beyond that, I personally don’t think performance is the thing to fixate on when deciding whether to learn any language. If learning a language can get you a raise or solve a bunch of problems at work, then picking up one more language is still way more upside than downside.

Oh, right!

Almost forgot to add this — let me throw in one more reason to learn Python.

The main reason is: “you won’t go bald.” (Just kidding.)

And the father of JavaScript?

A sneak peek at Python’s clean syntax

Since we’ll start gradually writing actual Python in later chapters, let’s take a quick look at just how clean it can be.

You can see a bunch of stuff is just gone — variable declarations, curly braces ({}), all of it. Python relies mainly on indentation, so if you’ve written Pug (an HTML template engine) or Sass (a CSS preprocessor), this’ll feel right at home, since both of those lean heavily on indentation too. One thing though: Python is really strict about indentation. Indent one space too few or too many and it’ll throw an error right at you.

So this post was mostly a casual chat to get a feel for Python’s history. Next time, we’ll cover how to actually install Python.
