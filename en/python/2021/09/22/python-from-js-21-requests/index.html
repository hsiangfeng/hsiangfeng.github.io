Learning Python From a JavaScript Angle (21) - Requests

#JavaScript #Python #iThome Ironman Contest #Learning Python From a JavaScript Angle

Intro

In JavaScript, AJAX is one of those fundamentals you really can’t skip — especially these days with front-end/back-end separation being so popular. So I figured it’s worth talking about the Python side too. Can Python do AJAX? In this post we’ll look at Python’s Requests library.

AJAX

Let’s quickly jog our memory on what AJAX even is. AJAX stands for Asynchronous JavaScript and XML. And some of you reading that are probably already thinking “okay, what on earth is this guy on about…”

So let me put it simply. If you’ve ever used React, Vue, or Angular, you’ve definitely run into something like axios, fetch, or XMLHttpRequest — those are all AJAX-style syntax.

This kind of behavior shows up the most in the SPA (Single Page Application) model that’s so popular now. The whole point of AJAX is fetching data asynchronously. What does that mean exactly?

Think back to the old days when switching pages on a website — there’d be a brief flash of a blank white screen, right? With today’s SPA approach, instead you’ll see a “Loading” label or spinner sitting on the screen, and you’ll notice the URL never changes, but the data on the page does. That swap is all happening through AJAX.

That brief white flash I mentioned when switching pages? That’s the classic SSR (Server Side Render) behavior. If you’re not solid on concepts like SPA and SSR, I’d recommend reading my earlier post “A Quick Look at SPA, CSR, SSR, MPA, SSG (article in Mandarin)” for a more detailed rundown.

Okay, I’ve drifted a bit. So why do we even need to understand AJAX here? Mainly because we’re about to do the same thing with Python — Get to grab data from a remote server, Post to add data to a remote server, and so on. So it’s worth a quick recap first.
